# Cancellation Design and Implementation

## Requirements Analysis

In some scenarios, we want to be able to actively cancel a request. Take the common search box case: while the user is typing, the content of the search box keeps changing, and normally we should send a request to the server on every change. But when the user types quickly, we don't want to fire a request on every change. A common solution is to use debounce on the front end, for example sending the request after a 200ms delay. That way, when the user types characters consecutively, as long as the interval between keystrokes is less than 200ms, none of the earlier characters trigger a request.

But there's still an extreme case: the back-end interface is very slow, say it takes more than 1s to respond. In that case, even with a 200ms debounce, when I type slowly (each keystroke interval exceeds 200ms), multiple requests may still go out before the earlier ones have responded. Because the response time of the interface is uncertain, if a request sent earlier takes longer to respond than a request sent later, the later request's response comes back first while the earlier request's response comes back afterward, so the earlier request's response result overwrites the later request's response result, and everything gets messed up. Therefore, in this scenario, in addition to debounce, we also want that when a later request is sent, if an earlier request has not yet responded, we can cancel the earlier request.

In terms of designing axios's cancellation interface, we want the following design:

```typescript
const CancelToken = axios.CancelToken;
const source = CancelToken.source();

axios.get('/user/12345', {
  cancelToken: source.token
}).catch(function (e) {
  if (axios.isCancel(e)) {
    console.log('Request canceled', e.message);
  } else {
    // handle error
  }
});

// cancel the request (the message is optional)
source.cancel('Operation canceled by the user.');
```

We add a `CancelToken` object to `axios`. It has a `source` method that returns a `source` object. `source.token` is passed to the `cancelToken` property of the config object on each request, and then after the request has been sent, we can cancel the request via the `source.cancel` method.

We also support another way of calling it:

``` typescript
const CancelToken = axios.CancelToken;
let cancel;

axios.get('/user/12345', {
  cancelToken: new CancelToken(function executor(c) {
    cancel = c;
  })
});

// cancel the request
cancel();
```

`axios.CancelToken` is a class. We pass an instance of it directly to the `cancelToken` property of the request config. The constructor of `CancelToken` accepts an `executor` method whose parameter is a cancellation function `c`. Inside the execution of the `executor` method we can obtain this cancellation function `c` and assign it to the externally defined `cancel` variable, after which we can cancel the request by calling this `cancel` method.

## Asynchronous-Separation Design Approach

Through the requirements analysis, we know that to cancel a particular request, we need to configure a `cancelToken` for that request and then call a `cancel` method externally.

Sending a request is an asynchronous process that ultimately executes the `xhr.send` method. The `xhr` object provides the [`abort`](https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest/abort) method, which can cancel the request. Because we can't reach the `xhr` object from the outside, we want to execute the `xhr.abort` method when `cancel` is executed.

So now it's as if we insert a piece of code into the `xhr` asynchronous request process, and when we execute the `cancel` function externally, it drives the execution of this piece of code, which then executes the `xhr.abort` method to cancel the request.

We can use a Promise to achieve asynchronous separation: that is, store a Promise object in the `pending` state inside `cancelToken`, and then when we execute the `cancel` method, we can access this Promise object and change it from the `pending` state to the `resolved` state. This way we can implement the cancellation logic inside the `then` function, similar to the following code:

```typescript

if (cancelToken) {
  cancelToken.promise
    .then(reason => {
      request.abort()
      reject(reason)
    })
}
```

## Implementing the CancelToken Class

Next, let's implement this `CancelToken` class. First let's look at the interface definition:

### Interface Definition

`types/index.ts`:

```typescript
export interface AxiosRequestConfig {
  // ...
  cancelToken?: CancelToken
}

export interface CancelToken {
  promise: Promise<string>
  reason?: string
}

export interface Canceler {
  (message?: string): void
}

export interface CancelExecutor {
  (cancel: Canceler): void
}

```

Here, `CancelToken` is the interface definition of the instance type, `Canceler` is the interface definition of the cancellation method, and `CancelExecutor` is the interface definition of the `CancelToken` class constructor parameter.

### Implementation

We create a separate `cancel` directory to manage cancellation-related code, and inside the `cancel` directory we create a `CancelToken.ts` file:

```typescript
import { CancelExecutor } from '../types'

interface ResolvePromise {
  (reason?: string): void
}

export default class CancelToken {
  promise: Promise<string>
  reason?: string

  constructor(executor: CancelExecutor) {
    let resolvePromise: ResolvePromise
    this.promise = new Promise<string>(resolve => {
      resolvePromise = resolve
    })

    executor(message => {
      if (this.reason) {
        return
      }
      this.reason = message
      resolvePromise(this.reason)
    })
  }
}
```
Inside the `CancelToken` constructor, we instantiate a Promise object in the `pending` state, then use a `resolvePromise` variable to point to the `resolve` function. Next we execute the `executor` function, passing in a `cancel` function. Inside the `cancel` function, it calls `resolvePromise` to change the Promise object from the `pending` state to the `resolved` state.

Next we insert a piece of request-cancellation logic into `xhr.ts`.

`core/xhr.ts`:

```typescript
const { /*....*/ cancelToken } = config

if (cancelToken) {
  cancelToken.promise.then(reason => {
    request.abort()
    reject(reason)
  })
}
```

This satisfies the second usage. Next we need to implement the first usage by extending the `CancelToken` static interface.

## Extending CancelToken with Static Interfaces

### Interface Definition

`types/index.ts`:

```typescript
export interface CancelTokenSource {
  token: CancelToken
  cancel: Canceler
}

export interface CancelTokenStatic {
  new(executor: CancelExecutor): CancelToken

  source(): CancelTokenSource
}
```

Here, `CancelTokenSource` serves as the return value type of the `source` static method of the `CancelToken` class, and `CancelTokenStatic` serves as the class type of the `CancelToken` class.

### Implementation

`cancel/CancelToken.ts`:

```typescript
export default class CancelToken {
  // ...

  static source(): CancelTokenSource {
    let cancel!: Canceler
    const token = new CancelToken(c => {
      cancel = c
    })
    return {
      cancel,
      token
    }
  }
}
```

The `source` static method is very simple: define a `cancel` variable, instantiate an object of type `CancelToken`, and then in the `executor` function point `cancel` to the parameter `c`, which is the cancellation function.

This satisfies our first usage. But in the example of the first usage, when catching the request, we use `axios.isCancel` to determine whether the error parameter e was caused by a cancelled request. Next we'll add a layer of wrapping around the cancellation reason, and extend `axios` with static methods.

## Implementing the Cancel Class and Extending axios

### Interface Definition

```typescript
export interface Cancel {
  message?: string
}

export interface CancelStatic {
  new(message?: string): Cancel
}

export interface AxiosStatic extends AxiosInstance {
  create(config?: AxiosRequestConfig): AxiosInstance

  CancelToken: CancelTokenStatic
  Cancel: CancelStatic
  isCancel: (value: any) => boolean
}
```

Here, `Cancel` is the interface definition of the instance type, `CancelStatic` is the interface definition of the class type, and we extend `axios` with several static methods.

### Implementation

I create a `Cancel.ts` file in the `cancel` directory.

```typescript
export default class Cancel {
  message?: string

  constructor(message?: string) {
    this.message = message
  }
}

export function isCancel(value: any): boolean {
  return value instanceof Cancel
}
```

The `Cancel` class is very simple, having a single public property `message`. The `isCancel` method is also very simple: it uses `instanceof` to determine whether the value passed in is a `Cancel` object.

Next we modify the type of `reason` in the `CancelToken` class, changing it into an instance of the `Cancel` type.

First modify the definition part.

`types/index.ts`:

```typescript
export interface CancelToken {
  promise: Promise<Cancel>
  reason?: Cancel
}
```

Then modify the implementation part:

```typescript
import Cancel from './Cancel'

interface ResolvePromise {
  (reason?: Cancel): void
}

export default class CancelToken {
  promise: Promise<Cancel>
  reason?: Cancel

  constructor(executor: CancelExecutor) {
    let resolvePromise: ResolvePromise
    this.promise = new Promise<Cancel>(resolve => {
      resolvePromise = resolve
    })

    executor(message => {
      if (this.reason) {
        return
      }
      this.reason = new Cancel(message)
      resolvePromise(this.reason)
    })
  }
}
```

Next we extend `axios` with some static methods for users to use.

`axios.ts`:

```typescript
import CancelToken from './cancel/CancelToken'
import Cancel, { isCancel } from './cancel/Cancel'

axios.CancelToken = CancelToken
axios.Cancel = Cancel
axios.isCancel = isCancel
```

## Implementing Additional Logic

Beyond this, we also need to implement some additional logic. For example, when a request carries a `cancelToken` that has already been used, we can even skip sending the request entirely and just throw an exception, with the exception message being our cancellation reason. So we need to extend `CancelToken` with a method.

First modify the definition part.

`types/index.ts`:

```typescript
export interface CancelToken {
  promise: Promise<Cancel>
  reason?: Cancel

  throwIfRequested(): void
}
```

Add a `throwIfRequested` method, and next let's implement it:

`cancel/CancelToken.ts`:

```typescript
export default class CancelToken {
  // ...

  throwIfRequested(): void {
    if (this.reason) {
      throw this.reason
    }
  }
}
```

We check whether `this.reason` exists; if it does, it means this `token` has already been used, so we throw an error directly.

Next we add a piece of logic before sending the request.

`core/dispatchRequest.ts`:

```typescript
export default function dispatchRequest(config: AxiosRequestConfig): AxiosPromise {
  throwIfCancellationRequested(config)
  processConfig(config)

  // ...
}

function throwIfCancellationRequested(config: AxiosRequestConfig): void {
  if (config.cancelToken) {
    config.cancelToken.throwIfRequested()
  }
}
```

Before sending the request, we check whether the configured cancelToken has already been used. If it has been used, we don't send the request and throw an exception directly.

## Writing the Demo

In the `examples` directory create a `cancel` directory, and inside the `cancel` directory create `index.html`:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Cancel example</title>
  </head>
  <body>
    <script src="/__build__/cancel.js"></script>
  </body>
</html>
```

Next create `app.ts` as the entry file:

```typescript
import axios, { Canceler } from '../../src/index'

const CancelToken = axios.CancelToken
const source = CancelToken.source()

axios.get('/cancel/get', {
  cancelToken: source.token
}).catch(function(e) {
  if (axios.isCancel(e)) {
    console.log('Request canceled', e.message)
  }
})

setTimeout(() => {
  source.cancel('Operation canceled by the user.')

  axios.post('/cancel/post', { a: 1 }, { cancelToken: source.token }).catch(function(e) {
    if (axios.isCancel(e)) {
      console.log(e.message)
    }
  })
}, 100)

let cancel: Canceler

axios.get('/cancel/get', {
  cancelToken: new CancelToken(c => {
    cancel = c
  })
}).catch(function(e) {
  if (axios.isCancel(e)) {
    console.log('Request canceled')
  }
})

setTimeout(() => {
  cancel()
}, 200)
```

Our demo shows 2 usage patterns, and also demonstrates that if a token has already been used, a subsequent request carrying that token will not be sent.

At this point, we have completed the request cancellation feature of `ts-axios`, cleverly using a Promise to achieve asynchronous separation. By now we have implemented most of the major features of the official `axios` library. In the following chapters, we'll start supplementing and refining the other features of `ts-axios`.
