# Extending the axios.create Static Interface

## Requirements Analysis

So far, our axios has always been a singleton. Once we modify the default axios configuration, it affects every request. We want to provide an `axios.create` static interface that lets us create a new `axios` instance, while also allowing us to pass in a new configuration that gets merged with the default configuration and becomes the new default configuration.

For example:

```typescript
const instance = axios.create({
  transformRequest: [(function(data) {
    return qs.stringify(data)
  }), ...(axios.defaults.transformRequest as AxiosTransformer[])],
  transformResponse: [...(axios.defaults.transformResponse as AxiosTransformer[]), function(data) {
    if (typeof data === 'object') {
      data.b = 2
    }
    return data
  }]
})

instance({
  url: '/config/post',
  method: 'post',
  data: {
    a: 1
  }
})
```

## Extending the Static Method

Since `axios` is gaining a static interface, let's first modify the interface type definition.

`types/index.ts`:

```typescript
export interface AxiosStatic extends AxiosInstance{
  create(config?: AxiosRequestConfig): AxiosInstance
}
```

The `create` function can accept a configuration of type `AxiosRequestConfig` as an extension of the default configuration, or it can be called with no arguments.

Next, let's implement the `axios.create` static method.

`axios.ts`:

```typescript
function createInstance(config: AxiosRequestConfig): AxiosStatic {
  const context = new Axios(config)
  const instance = Axios.prototype.request.bind(context)

  extend(instance, context)

  return instance as AxiosStatic
}
axios.create = function create(config) {
  return createInstance(mergeConfig(defaults, config))
}
```

Internally it calls the `createInstance` function and merges the `config` argument with `defaults` to serve as the new default configuration. Note that here we need the return value type of the `createInstance` function to be `AxiosStatic`.

## Writing the Demo

```typescript
const instance = axios.create({
  transformRequest: [(function(data) {
    return qs.stringify(data)
  }), ...(axios.defaults.transformRequest as AxiosTransformer[])],
  transformResponse: [...(axios.defaults.transformResponse as AxiosTransformer[]), function(data) {
    if (typeof data === 'object') {
      data.b = 2
    }
    return data
  }]
})

instance({
  url: '/config/post',
  method: 'post',
  data: {
    a: 1
  }
}).then((res) => {
  console.log(res.data)
})
```

We made a small modification to the example from the previous lesson: we create a new instance `instance` via the `axios.create` method, passing in `transformRequest` and `transformResponse` configurations that override the default configuration, and then send a request through `instance`. The result is the same as before.

At this point we have implemented the extension of the `axios.create` static interface, and the configuration work for the entire `ts-axios` is wrapped up. The official axios library also supports the ability to cancel requests, which can be cancelled either before the request is sent or after it has been sent but before it responds. In the next chapter we'll implement this feature.
