# HTTP Authorization

## Requirements Analysis

The [Authorization](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Authorization) request header in the HTTP protocol contains credentials that the server uses to verify the identity of the user agent. It is usually sent in subsequent requests after the server returns a 401 Unauthorized status code along with a WWW-Authenticate header.

The axios library also lets you configure an `auth` property in the request configuration. `auth` is an object structure containing two properties, `username` and `password`. Once the user configures these two properties when making a request, we automatically add an `Authorization` property to the HTTP request header, whose value is `Basic <encrypted string>`.
The encrypted string here is the base64-encoded result of `username:password`.

```typescript
axios.post('/more/post', {
  a: 1
}, {
  auth: {
    username: 'Yee',
    password: '123456'
  }
}).then(res => {
  console.log(res)
})
```

## Implementation

First, let's modify the type definitions.

`types/index.ts`:

```typescript
export interface AxiosRequestConfig {
  // ...
  auth?: AxiosBasicCredentials
}

export interface AxiosBasicCredentials {
  username: string
  password: string
}
```

Next, let's modify the merge rules. Since `auth` is also an object format, its merge rule is `deepMergeStrat`.

`core/mergeConfig.ts`:

```typescript
const stratKeysDeepMerge = ['headers', 'auth']
```

Then let's modify the logic before sending the request.

`core/xhr.ts`:

```typescript
const {
  /*...*/
  auth
} = config

if (auth) {
  headers['Authorization'] = 'Basic ' + btoa(auth.username + ':' + auth.password)
}
```

## Writing the Demo

```typescript
axios.post('/more/post', {
  a: 1
}, {
  auth: {
    username: 'Yee',
    password: '123456'
  }
}).then(res => {
  console.log(res)
})
```

In addition, in `server.js` we wrote a small piece of logic for this route endpoint:

```javascript
router.post('/more/post', function(req, res) {
  const auth = req.headers.authorization
  const [type, credentials] = auth.split(' ')
  console.log(atob(credentials))
  const [username, password] = atob(credentials).split(':')
  if (type === 'Basic' && username === 'Yee' && password === '123456') {
    res.json(req.body)
  } else {
    res.end('UnAuthorization')
  }
})
```

Note that here we need to install the third-party library `atob` to decode the base64 string.

At this point, `ts-axios` supports the HTTP authorization feature. Users can configure an `auth` object to automatically add an `Authorization` property to the request header. In the next lesson we'll implement a custom valid status code feature.
