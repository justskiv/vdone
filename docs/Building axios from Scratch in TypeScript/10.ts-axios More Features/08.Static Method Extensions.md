# Static Method Extensions

## Requirements Analysis

The official axios library implements methods such as `axios.all` and `axios.spread`, which are used as follows:

```typescript
function getUserAccount() {
  return axios.get('/user/12345');
}

function getUserPermissions() {
  return axios.get('/user/12345/permissions');
}

axios.all([getUserAccount(), getUserPermissions()])
  .then(axios.spread(function (acct, perms) {
    // Both requests are now complete
  }));
```

In fact, `axios.all` is just a wrapper around `Promise.all`; it returns an array of `Promise`s. The argument to the `then` function should be a function whose parameter is the `Promise resolves` (an array), and here we use the `axios.spread` method. So `axios.spread` takes a function and returns a new function whose structure matches the parameter structure of the `then` function.

Personally, I find these two static methods of `axios` rather useless at present, because `Promise` alone can accomplish both of these needs.

```typescript
function getUserAccount() {
  return axios.get('/user/12345');
}

function getUserPermissions() {
  return axios.get('/user/12345/permissions');
}

Promise.all([getUserAccount(), getUserPermissions()])
  .then(([acct,perms]) {
    // Both requests are now complete
  }));
```
In the `resolve` function of `Promise.all`, we can directly use array destructuring to get the response object corresponding to each request.

But in order to stay consistent with the official axios API, we will also implement these two methods in the `ts-axios` library.

The official axios library also exposes the `Axios` class via `axios.Axios` (there does not seem to be much of a use case for this either, but to stay consistent we will also implement it).

In addition, for the axios instance, the official site also provides a `getUri` method that returns a url based on the passed-in config without sending a request, as shown below:

```typescript
const fakeConfig = {
  baseURL: 'https://www.baidu.com/',
  url: '/user/12345',
  params: {
    idClient: 1,
    idTest: 2,
    testString: 'thisIsATest'
  }
}
console.log(axios.getUri(fakeConfig))
// https://www.baidu.com/user/12345?idClient=1&idTest=2&testString=thisIsATest
```

## Implementation

First, modify the type definitions.

`types/index.ts`:

```typescript
export interface AxiosClassStatic {
  new (config: AxiosRequestConfig): Axios
}

export interface AxiosStatic extends AxiosInstance {
  // ...

  all<T>(promises: Array<T | Promise<T>>): Promise<T[]>

  spread<T, R>(callback: (...args: T[]) => R): (arr: T[]) => R

  Axios: AxiosClassStatic
}

export interface Axios {
  // ...

  getUri(config?: AxiosRequestConfig): string
}
```

Then we go and implement these static methods.

`axios.ts`:

```typescript
axios.all = function all(promises) {
  return Promise.all(promises)
}

axios.spread = function spread(callback) {
  return function wrap(arr) {
    return callback.apply(null, arr)
  }
}

axios.Axios = Axios
```

Finally, we add the instance method `getUri` to Axios.

`core/Axios.ts`:

```typescript
getUri(config?: AxiosRequestConfig): string {
  config = mergeConfig(this.defaults, config)
  return transformURL(config)
}
```

We first merge with the default config, then return a new `url` through the `transformURL` implemented in `dispatchRequest`.

## Writing the demo

```typescript
function getA() {
  return axios.get('/more/A')
}

function getB() {
  return axios.get('/more/B')
}

axios.all([getA(), getB()])
  .then(axios.spread(function(resA, resB) {
    console.log(resA.data)
    console.log(resB.data)
  }))


axios.all([getA(), getB()])
  .then(([resA, resB]) => {
    console.log(resA.data)
    console.log(resB.data)
  })

const fakeConfig = {
  baseURL: 'https://www.baidu.com/',
  url: '/user/12345',
  params: {
    idClient: 1,
    idTest: 2,
    testString: 'thisIsATest'
  }
}
console.log(axios.getUri(fakeConfig))
```

Here we send 2 requests at the same time via `axios.all`, which returns an array of `Promise`s. We can get the results in the callback function passed to `axios.spread`, or get them directly in the callback function passed to `then`. In addition, we can use the `axios.getUri` method to obtain the final request url based on the config without sending a request.
