# Upload and Download Progress Monitoring

## Requirements Analysis

Sometimes, when we upload a file or request a large amount of data, we want to know the progress in real time, and we may even want to display a progress bar based on it.

We want to provide two function properties, `onDownloadProgress` and `onUploadProgress`, in the `axios` request config, so that users can use these two functions to monitor download and upload progress.

```typescript
axios.get('/more/get',{
  onDownloadProgress(progressEvent) {
    // listen for download progress
  }
})

axios.post('/more/post',{
  onUploadProgress(progressEvent) {
    // listen for upload progress
  }
})
```

The `xhr` object provides a [`progress`](https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest/progress_event) event, which we can listen to in order to monitor the download progress of the data. In addition, the [`xhr.uplaod`](https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest/upload) object also provides a [`progress`](https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest/progress_event) event, which we can use to monitor upload progress.

## Code Implementation

First, let's modify the type definition.

`types/index.ts`:

```typescript
export interface AxiosRequestConfig {
  // ...
  onDownloadProgress?: (e: ProgressEvent) => void
  onUploadProgress?: (e: ProgressEvent) => void
}
```

Next, before sending the request, add the properties to the `xhr` object.

`core/xhr.ts`:

```typescript
const {
  /*...*/
  onDownloadProgress,
  onUploadProgress
} = config

if (onDownloadProgress) {
  request.onprogress = onDownloadProgress
}

if (onUploadProgress) {
  request.upload.onprogress = onUploadProgress
}
```

In addition, if the request data is of type `FormData`, we should proactively remove the `Content-Type` field from the request `headers`, letting the browser set the `Content-Type` automatically based on the request data. For example, when we upload a file via `FormData`, the browser will set the `Content-Type` in the request `headers` to `multipart/form-data`.

Let's first add a method to check for `FormData`.

`helpers/util.ts`:

```typescript
export function isFormData(val: any): boolean {
  return typeof val !== 'undefined' && val instanceof FormData
}
```

Then add the related logic.

`core/xhr.ts`:

```typescript
if (isFormData(data)) {
  delete headers['Content-Type']
}
```

We notice that as more requirements pile up inside the `xhr` function, the code is becoming increasingly bloated. We can reorganize the logic and refactor the internal code into a cleaner, more encapsulated structure.

```typescript
export default function xhr(config: AxiosRequestConfig): AxiosPromise {
  return new Promise((resolve, reject) => {
    const {
      data = null,
      url,
      method = 'get',
      headers,
      responseType,
      timeout,
      cancelToken,
      withCredentials,
      xsrfCookieName,
      xsrfHeaderName,
      onDownloadProgress,
      onUploadProgress
    } = config

    const request = new XMLHttpRequest()

    request.open(method.toUpperCase(), url!, true)

    configureRequest()

    addEvents()

    processHeaders()

    processCancel()

    request.send(data)

    function configureRequest(): void {
      if (responseType) {
        request.responseType = responseType
      }

      if (timeout) {
        request.timeout = timeout
      }

      if (withCredentials) {
        request.withCredentials = withCredentials
      }
    }

    function addEvents(): void {
      request.onreadystatechange = function handleLoad() {
        if (request.readyState !== 4) {
          return
        }

        if (request.status === 0) {
          return
        }

        const responseHeaders = parseHeaders(request.getAllResponseHeaders())
        const responseData =
          responseType && responseType !== 'text' ? request.response : request.responseText
        const response: AxiosResponse = {
          data: responseData,
          status: request.status,
          statusText: request.statusText,
          headers: responseHeaders,
          config,
          request
        }
        handleResponse(response)
      }

      request.onerror = function handleError() {
        reject(createError('Network Error', config, null, request))
      }

      request.ontimeout = function handleTimeout() {
        reject(
          createError(`Timeout of ${config.timeout} ms exceeded`, config, 'ECONNABORTED', request)
        )
      }

      if (onDownloadProgress) {
        request.onprogress = onDownloadProgress
      }

      if (onUploadProgress) {
        request.upload.onprogress = onUploadProgress
      }
    }

    function processHeaders(): void {
      if (isFormData(data)) {
        delete headers['Content-Type']
      }

      if ((withCredentials || isURLSameOrigin(url!)) && xsrfCookieName) {
        const xsrfValue = cookie.read(xsrfCookieName)
        if (xsrfValue) {
          headers[xsrfHeaderName!] = xsrfValue
        }
      }

      Object.keys(headers).forEach(name => {
        if (data === null && name.toLowerCase() === 'content-type') {
          delete headers[name]
        } else {
          request.setRequestHeader(name, headers[name])
        }
      })
    }

    function processCancel(): void {
      if (cancelToken) {
        cancelToken.promise.then(reason => {
          request.abort()
          reject(reason)
        })
      }
    }

    function handleResponse(response: AxiosResponse): void {
      if (response.status >= 200 && response.status < 300) {
        resolve(response)
      } else {
        reject(
          createError(
            `Request failed with status code ${response.status}`,
            config,
            null,
            request,
            response
          )
        )
      }
    }
  })
}
```

We divide the whole flow into 7 steps:

- Create a `request` instance.
- Call the `request.open` method to initialize it.
- Call `configureRequest` to configure the `request` object.
- Call `addEvents` to add event handlers to `request`.
- Call `processHeaders` to handle the request `headers`.
- Call `processCancel` to handle the request cancellation logic.
- Call the `request.send` method to send the request.

After splitting it up this way, the whole flow becomes very clear, and when we add new requirements in the future, the code won't become increasingly bloated.

In addition, to display the upload and download progress in a friendly way, we introduce an open-source library, [nprogress](https://github.com/rstacruz/nprogress), which can show a progress bar at the top of the page.

`examples/more/app.ts`:

```typescript
const instance = axios.create()

function calculatePercentage(loaded: number, total: number) {
  return Math.floor(loaded * 1.0) / total
}

function loadProgressBar() {
  const setupStartProgress = () => {
    instance.interceptors.request.use(config => {
      NProgress.start()
      return config
    })
  }

  const setupUpdateProgress = () => {
    const update = (e: ProgressEvent) => {
      console.log(e)
      NProgress.set(calculatePercentage(e.loaded, e.total))
    }
    instance.defaults.onDownloadProgress = update
    instance.defaults.onUploadProgress = update
  }

  const setupStopProgress = () => {
    instance.interceptors.response.use(response => {
      NProgress.done()
      return response
    }, error => {
      NProgress.done()
      return Promise.reject(error)
    })
  }

  setupStartProgress()
  setupUpdateProgress()
  setupStopProgress()
}

loadProgressBar()

const downloadEl = document.getElementById('download')

downloadEl!.addEventListener('click', e => {
  instance.get('https://img.mukewang.com/5cc01a7b0001a33718720632.jpg')
})

const uploadEl = document.getElementById('upload')

uploadEl!.addEventListener('click', e => {
  const data = new FormData()
  const fileEl = document.getElementById('file') as HTMLInputElement
  if (fileEl.files) {
    data.append('file', fileEl.files[0])

    instance.post('/more/upload', data)
  }
})
```

For the `progress` event parameter `e`, there are `e.total` and `e.loaded` properties, representing the total amount of work of the process and the amount of work already done. We can compute the current progress from these two values and then set it via `Nprogess.set`. In addition, we run `NProgress.start()` and `NProgress.done()` by configuring request and response interceptors.

We bind a `click` event to the download button to request an image, and we can see the progress in real time. We also bind a `click` event to the upload button to upload the file we select, and likewise we can see the progress in real time.

On the server side, to handle upload requests, we need to download and install an `express` middleware, `connect-multiparty`, and then use it.

`example/server.js`:

```javascript
const multipart = require('connect-multiparty')
app.use(multipart({
  uploadDir: path.resolve(__dirname, 'upload-file')
}))

router.post('/more/upload', function(req, res) {
  console.log(req.body, req.files)
  res.end('upload success!')
})
```

Here we need to create an empty `upload-file` directory under the `examples` directory to store the uploaded files.

With this middleware, we can handle upload requests and store the uploaded files in the `upload-file` directory.

To make sure the code runs properly, we also need to add `css-loader` and `style-loader` in `examples/webpack.config.js`; don't forget to install them first.
