# withCredentials

## Requirements Analysis

Sometimes we make cross-origin requests, for example a `http://domain-a.com` site sends a request to `http://api.domain-b.com/get`. By default, the browser restricts such cross-origin requests based on the same-origin policy, but we can solve the cross-origin problem using [CORS](https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS) technology.

For same-origin requests, the cookies under the current domain are carried by default. But for cross-origin requests, the cookies under the requested domain are not carried by default. For example, when a `http://domain-a.com` site sends a request to `http://api.domain-b.com/get`, the cookies under the `api.domain-b.com` domain are not carried by default. If we want to carry them (which is needed in many cases), we just need to set the `withCredentials` property of the request's `xhr` object to true.

## Implementation

First, modify the type definition of `AxiosRequestConfig`.

`types/index.ts`:

```typescript
export interface AxiosRequestConfig {
  // ...
  withCredentials?: boolean
}
```

Then modify the logic before the request is sent.

`core/xhr.ts`:

```typescript
const { /*...*/ withCredentials } = config

if (withCredentials) {
  request.withCredentials = true
}
```

## Writing the Demo

Create a `more` directory under the `examples` directory, and create `index.html` under the `cancel` directory:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>More example</title>
  </head>
  <body>
    <script src="/__build__/more.js"></script>
  </body>
</html>
```

Next, create `app.ts` as the entry file:

```typescript
import axios from '../../src/index'

document.cookie = 'a=b'

axios.get('/more/get').then(res => {
  console.log(res)
})

axios.post('http://127.0.0.1:8088/more/server2', { }, {
  withCredentials: true
}).then(res => {
  console.log(res)
})
```

This time, in addition to configuring the API routes in `server.js`, we also created `server2.js`, which starts a cross-origin service.

```javascript
const express = require('express')
const bodyParser = require('body-parser')
const cookieParser = require('cookie-parser')

const app = express()

app.use(bodyParser.json())
app.use(bodyParser.urlencoded({ extended: true }))
app.use(cookieParser())

const router = express.Router()

const cors = {
  'Access-Control-Allow-Origin': 'http://localhost:8080',
  'Access-Control-Allow-Credentials': true,
  'Access-Control-Allow-Methods': 'POST, GET, PUT, DELETE, OPTIONS',
  'Access-Control-Allow-Headers': 'Content-Type'
}

router.post('/more/server2', function(req, res) {
  res.set(cors)
  res.json(req.cookies)
})

router.options('/more/server2', function(req, res) {
  res.set(cors)
  res.end()
})

app.use(router)

const port = 8088
module.exports = app.listen(port)
```

Here we need to install the `cookie-parser` plugin, which is used for the cookies sent with the request.

Through the demo, we can see that for same-origin requests, cookies are carried, while for cross-origin requests, cookies are only carried when we set `withCredentials` to true.

With this, our `withCredentials` feature is complete. In the next lesson, we will implement axios's defense against XSRF.
