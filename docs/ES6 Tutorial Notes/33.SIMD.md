# SIMD

## Overview

SIMD (pronounced `/sim-dee/`) is the abbreviation of "Single Instruction/Multiple Data". It is the interface through which JavaScript operates on the corresponding CPU instructions, and you can think of it as a different mode of computation. Its counterpart is SISD ("Single Instruction/Single Data").
<!-- more -->
SIMD means using one instruction to complete the computation of multiple pieces of data; SISD means using one instruction to complete the computation of a single piece of data, which is JavaScript's default computation mode. Obviously, SIMD is more efficient than SISD, so it is widely used in projects with extremely heavy computational loads, such as 3D graphics rendering and physics simulation.

To understand SIMD, look at the following example.

```javascript
var a = [1, 2, 3, 4];
var b = [5, 6, 7, 8];
var c = [];

c[0] = a[0] + b[0];
c[1] = a[1] + b[1];
c[2] = a[2] + b[2];
c[3] = a[3] + b[3];
c // Array[6, 8, 10, 12]
```

In the code above, the corresponding members of arrays `a` and `b` are added together, and the result is placed into array `c`. Its computation mode processes each array member in turn; there are four array members in total, so four computations are needed.

If the SIMD mode is used, only one computation is enough.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 4);
var b = SIMD.Float32x4(5, 6, 7, 8);
var c = SIMD.Float32x4.add(a, b); // Float32x4[6, 8, 10, 12]
```

In the code above, the respective additions of the four members of arrays `a` and `b` are completed with just one instruction. Therefore, the speed is 4 times faster than the previous approach.

A single SIMD operation can process multiple pieces of data, and these pieces of data are called "lanes". In the code above, four pieces of data are computed at once, so there are four lanes.

SIMD is typically used for vector operations.

```javascript
v + w = 〈v1, …, vn〉+ 〈w1, …, wn〉
      = 〈v1+w1, …, vn+wn〉
```

In the code above, `v` and `w` are two multi-dimensional vectors. Their addition is completed by one instruction under SIMD, rather than n instructions, which greatly improves efficiency. This is very important for operations such as 3D animation, image processing, signal processing, numerical processing, and encryption. For example, Canvas's `getImageData()` reads an image file into a binary array, and SIMD is well suited for processing such arrays.

In general, SIMD is a means of data parallelism, which can speed up some computation-intensive operations. In the future, once combined with WebAssembly, it can allow JavaScript to reach the running speed of binary code.

## Data Types

SIMD provides 12 data types, all with a total length of 128 binary bits.

- Float32x4: four 32-bit floating-point numbers
- Float64x2: two 64-bit floating-point numbers
- Int32x4: four 32-bit integers
- Int16x8: eight 16-bit integers
- Int8x16: sixteen 8-bit integers
- Uint32x4: four unsigned 32-bit integers
- Uint16x8: eight unsigned 16-bit integers
- Uint8x16: sixteen unsigned 8-bit integers
- Bool32x4: four 32-bit boolean values
- Bool16x8: eight 16-bit boolean values
- Bool8x16: sixteen 8-bit boolean values
- Bool64x2: two 64-bit boolean values

Each data type is separated into two parts by the `x` symbol. The part after it indicates the number of lanes, and the part before it indicates the width and type of each lane. For example, `Float32x4` indicates that this value has 4 lanes, each of which is a 32-bit floating-point number.

In each lane, four kinds of data can be placed.

- Floating-point number (float, e.g. 1.0)
- Signed integer (Int, e.g. -1)
- Unsigned integer (Uint, e.g. 1)
- Boolean value (Bool, containing the two values `true` and `false`)

Each SIMD data type is a function method that can accept arguments to generate the corresponding value.

```javascript
var a = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
```

In the code above, the variable `a` is a 128-bit value containing four 32-bit floating-point numbers (i.e. four lanes).

Note that these data type methods are not constructors, and `new` cannot be prepended to them, otherwise an error will be thrown.

```javascript
var v = new SIMD.Float32x4(0, 1, 2, 3);
// TypeError: SIMD.Float32x4 is not a constructor
```

## Static Methods: Mathematical Operations

Each data type has a series of operators that support basic mathematical operations.

### SIMD.%type%.abs(), SIMD.%type%.neg()

The `abs` method accepts a SIMD value as an argument, converts each of its lanes to its absolute value, and returns it as a new SIMD value.

```javascript
var a = SIMD.Float32x4(-1, -2, 0, NaN);
SIMD.Float32x4.abs(a)
// Float32x4[1, 2, 0, NaN]
```

The `neg` method accepts a SIMD value as an argument, converts each of its lanes to its negated value, and returns it as a new SIMD value.

```javascript
var a = SIMD.Float32x4(-1, -2, 3, 0);
SIMD.Float32x4.neg(a)
// Float32x4[1, 2, -3, -0]

var b = SIMD.Float64x2(NaN, Infinity);
SIMD.Float64x2.neg(b)
// Float64x2[NaN, -Infinity]
```

### SIMD.%type%.add(), SIMD.%type%.addSaturate()

The `add` method accepts two SIMD values as arguments, adds each of their lanes together, and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
var b = SIMD.Float32x4(5.0, 10.0, 15.0, 20.0);
var c = SIMD.Float32x4.add(a, b);
```

In the code above, after the addition, the new SIMD value is `(6.0, 12.0, 18.0. 24.0)`.

The `addSaturate` method has the same effect as the `add` method—both add two lanes together—but the handling of overflow differs. For the `add` method, if adding two values causes overflow, the overflowing binary bits are discarded; the `addSaturate` method instead returns the maximum value of that data type.

```javascript
var a = SIMD.Uint16x8(65533, 65534, 65535, 65535, 1, 1, 1, 1);
var b = SIMD.Uint16x8(1, 1, 1, 5000, 1, 1, 1, 1);
SIMD.Uint16x8.addSaturate(a, b);
// Uint16x8[65534, 65535, 65535, 65535, 2, 2, 2, 2]

var c = SIMD.Int16x8(32765, 32766, 32767, 32767, 1, 1, 1, 1);
var d = SIMD.Int16x8(1, 1, 1, 5000, 1, 1, 1, 1);
SIMD.Int16x8.addSaturate(c, d);
// Int16x8[32766, 32767, 32767, 32767, 2, 2, 2, 2]
```

In the code above, the maximum value of `Uint16` is 65535, and the maximum value of `Int16` is 32767. Once overflow occurs, these two values are returned.

Note that the two data types `Uint32x4` and `Int32x4` do not have an `addSaturate` method.

### SIMD.%type%.sub(), SIMD.%type%.subSaturate()

The `sub` method accepts two SIMD values as arguments, subtracts each of their lanes, and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(-1, -2, 3, 4);
var b = SIMD.Float32x4(3, 3, 3, 3);
SIMD.Float32x4.sub(a, b)
// Float32x4[-4, -5, 0, 1]
```

The `subSaturate` method has the same effect as the `sub` method—both subtract two lanes—but the handling of overflow differs. For the `sub` method, if subtracting two values causes overflow, the overflowing binary bits are discarded; the `subSaturate` method instead returns the minimum value of that data type.

```javascript
var a = SIMD.Uint16x8(5, 1, 1, 1, 1, 1, 1, 1);
var b = SIMD.Uint16x8(10, 1, 1, 1, 1, 1, 1, 1);
SIMD.Uint16x8.subSaturate(a, b)
// Uint16x8[0, 0, 0, 0, 0, 0, 0, 0]

var c = SIMD.Int16x8(-100, 0, 0, 0, 0, 0, 0, 0);
var d = SIMD.Int16x8(32767, 0, 0, 0, 0, 0, 0, 0);
SIMD.Int16x8.subSaturate(c, d)
// Int16x8[-32768, 0, 0, 0, 0, 0, 0, 0, 0]
```

In the code above, the minimum value of `Uint16` is `0`, and the minimum value of `Int16` is `-32678`. Once an operation overflows, the minimum value is returned.

### SIMD.%type%.mul(), SIMD.%type%.div(), SIMD.%type%.sqrt()

The `mul` method accepts two SIMD values as arguments, multiplies each of their lanes, and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(-1, -2, 3, 4);
var b = SIMD.Float32x4(3, 3, 3, 3);
SIMD.Float32x4.mul(a, b)
// Float32x4[-3, -6, 9, 12]
```

The `div` method accepts two SIMD values as arguments, divides each of their lanes, and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(2, 2, 2, 2);
var b = SIMD.Float32x4(4, 4, 4, 4);
SIMD.Float32x4.div(a, b)
// Float32x4[0.5, 0.5, 0.5, 0.5]
```

The `sqrt` method accepts a SIMD value as an argument, computes the square root of each lane, and returns the result as a new SIMD value.

```javascript
var b = SIMD.Float64x2(4, 8);
SIMD.Float64x2.sqrt(b)
// Float64x2[2, 2.8284271247461903]
```

### SIMD.%FloatType%.reciprocalApproximation(), SIMD.%type%.reciprocalSqrtApproximation()

The `reciprocalApproximation` method accepts a SIMD value as an argument, computes the reciprocal of each lane (`1 / x`), and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 4);
SIMD.Float32x4.reciprocalApproximation(a);
// Float32x4[1, 0.5, 0.3333333432674408, 0.25]
```

The `reciprocalSqrtApproximation` method accepts a SIMD value as an argument, computes the reciprocal of the square root of each lane (`1 / (x^0.5)`), and returns the result as a new SIMD value.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 4);
SIMD.Float32x4.reciprocalSqrtApproximation(a)
// Float32x4[1, 0.7071067690849304, 0.5773502588272095, 0.5]
```

Note that only floating-point data types have these two methods.

### SIMD.%IntegerType%.shiftLeftByScalar()

The `shiftLeftByScalar` method accepts a SIMD value as an argument, then left-shifts the value of each lane by the specified number of bits, and returns the result as a new SIMD value.

```javascript
var a = SIMD.Int32x4(1, 2, 4, 8);
SIMD.Int32x4.shiftLeftByScalar(a, 1);
// Int32x4[2, 4, 8, 16]
```

If after the left shift the new value exceeds the number of bits of the current data type, the overflowing part is discarded.

```javascript
var ix4 = SIMD.Int32x4(1, 2, 3, 4);
var jx4 = SIMD.Int32x4.shiftLeftByScalar(ix4, 32);
// Int32x4[0, 0, 0, 0]
```

Note that only integer data types have this method.

### SIMD.%IntegerType%.shiftRightByScalar()

The `shiftRightByScalar` method accepts a SIMD value as an argument, then right-shifts the value of each lane by the specified number of bits, and returns a new SIMD value.

```javascript
var a = SIMD.Int32x4(1, 2, 4, -8);
SIMD.Int32x4.shiftRightByScalar(a, 1);
// Int32x4[0, 1, 2, -4]
```

If the original lane value is a signed value, the sign bit remains unchanged and is not affected by the right shift. If it is an unsigned value, then after the right shift the leading bits are padded with `0`.

```javascript
var a = SIMD.Uint32x4(1, 2, 4, -8);
SIMD.Uint32x4.shiftRightByScalar(a, 1);
// Uint32x4[0, 1, 2, 2147483644]
```

In the code above, `-8` shifted right by one bit becomes `2147483644`, because for a 32-bit unsigned integer, the binary form of `-8` is `11111111111111111111111111111000`, and shifting it right by one bit makes it `01111111111111111111111111111100`, which is equivalent to `2147483644`.

Note that only integer data types have this method.

## Static Methods: Lane Processing

### SIMD.%type%.check()

The `check` method is used to check whether a value is a SIMD value of the current type. If it is, the value is returned; otherwise, an error is thrown.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 9);

SIMD.Float32x4.check(a);
// Float32x4[1, 2, 3, 9]

SIMD.Float32x4.check([1,2,3,4]) // throws an error
SIMD.Int32x4.check(a) // throws an error
SIMD.Int32x4.check('hello world') // throws an error
```

### SIMD.%type%.extractLane(), SIMD.%type%.replaceLane()

The `extractLane` method is used to return the value of a given lane. It accepts two arguments: the SIMD value and the lane index.

```javascript
var t = SIMD.Float32x4(1, 2, 3, 4);
SIMD.Float32x4.extractLane(t, 2) // 3
```

The `replaceLane` method is used to replace the value of a specified lane and return a new SIMD value. It accepts three arguments: the original SIMD value, the lane index, and the new lane value.

```javascript
var t = SIMD.Float32x4(1, 2, 3, 4);
SIMD.Float32x4.replaceLane(t, 2, 42)
// Float32x4[1, 2, 42, 4]
```

### SIMD.%type%.load()

The `load` method is used to read data from a binary array and generate a new SIMD value.

```javascript
var a = new Int32Array([1,2,3,4,5,6,7,8]);
SIMD.Int32x4.load(a, 0);
// Int32x4[1, 2, 3, 4]

var b = new Int32Array([1,2,3,4,5,6,7,8]);
SIMD.Int32x4.load(a, 2);
// Int32x4[3, 4, 5, 6]
```

The `load` method accepts two arguments: a binary array and the position from which to start reading (starting from 0). If the position is invalid (for example `-1` or beyond the size of the binary array), an error is thrown.

This method also has three variants `load1()`, `load2()`, and `load3()`, which indicate loading only one lane, two lanes, or three lanes of values starting from the specified position.

```javascript
// format
SIMD.Int32x4.load(tarray, index)
SIMD.Int32x4.load1(tarray, index)
SIMD.Int32x4.load2(tarray, index)
SIMD.Int32x4.load3(tarray, index)

// examples
var a = new Int32Array([1,2,3,4,5,6,7,8]);
SIMD.Int32x4.load1(a, 0);
// Int32x4[1, 0, 0, 0]
SIMD.Int32x4.load2(a, 0);
// Int32x4[1, 2, 0, 0]
SIMD.Int32x4.load3(a, 0);
// Int32x4[1, 2, 3,0]
```

### SIMD.%type%.store()

The `store` method is used to write a SIMD value into a binary array. It accepts three arguments: the binary array, the array position from which to start writing, and the SIMD value. It returns the binary array after the value has been written.

```javascript
var t1 = new Int32Array(8);
var v1 = SIMD.Int32x4(1, 2, 3, 4);
SIMD.Int32x4.store(t1, 0, v1)
// Int32Array[1, 2, 3, 4, 0, 0, 0, 0]

var t2 = new Int32Array(8);
var v2 = SIMD.Int32x4(1, 2, 3, 4);
SIMD.Int32x4.store(t2, 2, v2)
// Int32Array[0, 0, 1, 2, 3, 4, 0, 0]
```

In the code above, `t1` is a binary array, and `v1` is a SIMD value with only four lanes. So after writing into `t1`, only the first four positions have values, and the last four positions are all 0. Whereas `t2` starts writing from position 2, so the first two positions and the last two positions are all 0.

This method also has three variants `store1()`, `store2()`, and `store3()`, which indicate writing only one lane, two lanes, and three lanes of values.

```javascript
var tarray = new Int32Array(8);
var value = SIMD.Int32x4(1, 2, 3, 4);
SIMD.Int32x4.store1(tarray, 0, value);
// Int32Array[1, 0, 0, 0, 0, 0, 0, 0]
```

### SIMD.%type%.splat()

The `splat` method returns a new SIMD value in which all lanes are set to the same pre-given value.

```javascript
SIMD.Float32x4.splat(3);
// Float32x4[3, 3, 3, 3]
SIMD.Float64x2.splat(3);
// Float64x2[3, 3]
```

If the argument is omitted, all integer-type SIMD values are set to `0`, and all floating-point-type SIMD values are set to `NaN`.

### SIMD.%type%.swizzle()

The `swizzle` method returns a new SIMD value, rearranging the lane order of the original SIMD value.

```javascript
var t = SIMD.Float32x4(1, 2, 3, 4);
SIMD.Float32x4.swizzle(t, 1, 2, 0, 3);
// Float32x4[2,3,1,4]
```

In the code above, the first argument of the `swizzle` method is the original SIMD value, and the subsequent arguments correspond to the four lanes of the SIMD value to be returned. It means that the four lanes of the new SIMD are, in order, lane 1, lane 2, lane 0, and lane 3 of the original SIMD value. Since a SIMD value can have up to 16 lanes, the `swizzle` method can accept up to 16 arguments in addition to the first one.

Here is another example.

```javascript
var a = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
// Float32x4[1.0, 2.0, 3.0, 4.0]

var b = SIMD.Float32x4.swizzle(a, 0, 0, 1, 1);
// Float32x4[1.0, 1.0, 2.0, 2.0]

var c = SIMD.Float32x4.swizzle(a, 3, 3, 3, 3);
// Float32x4[4.0, 4.0, 4.0, 4.0]

var d = SIMD.Float32x4.swizzle(a, 3, 2, 1, 0);
// Float32x4[4.0, 3.0, 2.0, 1.0]
```

### SIMD.%type%.shuffle()

The `shuffle` method takes specified lanes from two SIMD values and returns a new SIMD value.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 4);
var b = SIMD.Float32x4(5, 6, 7, 8);

SIMD.Float32x4.shuffle(a, b, 1, 5, 7, 2);
// Float32x4[2, 6, 8, 3]
```

In the code above, `a` and `b` have 8 lanes in total, numbered 0 through 7 in order. `shuffle` takes the corresponding lanes according to the numbers and returns a new SIMD value.

## Static Methods: Comparison Operations

### SIMD.%type%.equal(), SIMD.%type%.notEqual()

The `equal` method is used to compare each lane of two SIMD values `a` and `b`, and obtain a boolean value based on whether the two are exactly equal (`a === b`). Finally, the comparison results of all lanes form a new SIMD value, which is returned as a mask. The `notEqual` method instead compares whether two lanes are not equal (`a !== b`).

```javascript
var a = SIMD.Float32x4(1, 2, 3, 9);
var b = SIMD.Float32x4(1, 4, 7, 9);

SIMD.Float32x4.equal(a,b)
// Bool32x4[true, false, false, true]

SIMD.Float32x4.notEqual(a,b);
// Bool32x4[false, true, true, false]
```

### SIMD.%type%.greaterThan(), SIMD.%type%.greaterThanOrEqual()

The `greatThan` method is used to compare each lane of two SIMD values `a` and `b`. If in that lane `a` is greater, `true` is obtained; otherwise `false` is obtained. Finally, the comparison results of all lanes form a new SIMD value, which is returned as a mask. `greaterThanOrEqual` instead compares whether `a` is greater than or equal to `b`.

```javascript
var a = SIMD.Float32x4(1, 6, 3, 11);
var b = SIMD.Float32x4(1, 4, 7, 9);

SIMD.Float32x4.greaterThan(a, b)
// Bool32x4[false, true, false, true]

SIMD.Float32x4.greaterThanOrEqual(a, b)
// Bool32x4[true, true, false, true]
```

### SIMD.%type%.lessThan(), SIMD.%type%.lessThanOrEqual()

The `lessThan` method is used to compare each lane of two SIMD values `a` and `b`. If in that lane `a` is smaller, `true` is obtained; otherwise `false` is obtained. Finally, the comparison results of all lanes form a new SIMD value, which is returned as a mask. The `lessThanOrEqual` method instead compares whether `a` is equal to `b`.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 11);
var b = SIMD.Float32x4(1, 4, 7, 9);

SIMD.Float32x4.lessThan(a, b)
// Bool32x4[false, true, true, false]

SIMD.Float32x4.lessThanOrEqual(a, b)
// Bool32x4[true, true, true, false]
```

### SIMD.%type%.select()

The `select` method generates a new SIMD value through a mask. It accepts three arguments: the mask and two SIMD values.

```javascript
var a = SIMD.Float32x4(1, 2, 3, 4);
var b = SIMD.Float32x4(5, 6, 7, 8);

var mask = SIMD.Bool32x4(true, false, false, true);

SIMD.Float32x4.select(mask, a, b);
// Float32x4[1, 6, 7, 4]
```

In the code above, the `select` method accepts a mask and two SIMD values as arguments. When the mask corresponding to a lane is `true`, the corresponding lane of the first SIMD value is selected; otherwise the corresponding lane of the second SIMD value is selected.

This method is usually used in combination with comparison operators.

```javascript
var a = SIMD.Float32x4(0, 12, 3, 4);
var b = SIMD.Float32x4(0, 6, 7, 50);

var mask = SIMD.Float32x4.lessThan(a,b);
// Bool32x4[false, false, true, true]

var result = SIMD.Float32x4.select(mask, a, b);
// Float32x4[0, 6, 3, 4]
```

In the code above, a mask is first generated through the `lessThan` method, and then through the `select` method a new SIMD value composed of the smaller value of each lane is generated.

### SIMD.%BooleanType%.allTrue(), SIMD.%BooleanType%.anyTrue()

The `allTrue` method accepts a SIMD value as an argument and then returns a boolean value indicating whether all lanes of that SIMD value are `true`.

```javascript
var a = SIMD.Bool32x4(true, true, true, true);
var b = SIMD.Bool32x4(true, false, true, true);

SIMD.Bool32x4.allTrue(a); // true
SIMD.Bool32x4.allTrue(b); // false
```

The `anyTrue` method instead returns `true` as long as one lane is `true`, and returns `false` otherwise.

```javascript
var a = SIMD.Bool32x4(false, false, false, false);
var b = SIMD.Bool32x4(false, false, true, false);

SIMD.Bool32x4.anyTrue(a); // false
SIMD.Bool32x4.anyTrue(b); // true
```

Note that only the four boolean data types (`Bool32x4`, `Bool16x8`, `Bool8x16`, `Bool64x2`) have these two methods.

These two methods are usually used in combination with comparison operators.

```javascript
var ax4    = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
var bx4    = SIMD.Float32x4(0.0, 6.0, 7.0, 8.0);
var ix4    = SIMD.Float32x4.lessThan(ax4, bx4);
var b1     = SIMD.Int32x4.allTrue(ix4); // false
var b2     = SIMD.Int32x4.anyTrue(ix4); // true
```

### SIMD.%type%.min(), SIMD.%type%.minNum()

The `min` method accepts two SIMD values as arguments, and returns a new SIMD value composed of the smaller value of the corresponding lanes of the two.

```javascript
var a = SIMD.Float32x4(-1, -2, 3, 5.2);
var b = SIMD.Float32x4(0, -4, 6, 5.5);
SIMD.Float32x4.min(a, b);
// Float32x4[-1, -4, 3, 5.2]
```

If one lane's value is `NaN`, then `NaN` is returned preferentially.

```javascript
var c = SIMD.Float64x2(NaN, Infinity)
var d = SIMD.Float64x2(1337, 42);
SIMD.Float64x2.min(c, d);
// Float64x2[NaN, 42]
```

The `minNum` method has exactly the same effect as `min`, with the only difference being that if one lane's value is `NaN`, then the other lane's value is returned preferentially.

```javascript
var ax4 = SIMD.Float32x4(1.0, 2.0, NaN, NaN);
var bx4 = SIMD.Float32x4(2.0, 1.0, 3.0, NaN);
var cx4 = SIMD.Float32x4.min(ax4, bx4);
// Float32x4[1.0, 1.0, NaN, NaN]
var dx4 = SIMD.Float32x4.minNum(ax4, bx4);
// Float32x4[1.0, 1.0, 3.0, NaN]
```

### SIMD.%type%.max(), SIMD.%type%.maxNum()

The `max` method accepts two SIMD values as arguments, and returns a new SIMD value composed of the larger value of the corresponding lanes of the two.

```javascript
var a = SIMD.Float32x4(-1, -2, 3, 5.2);
var b = SIMD.Float32x4(0, -4, 6, 5.5);
SIMD.Float32x4.max(a, b);
// Float32x4[0, -2, 6, 5.5]
```

If one lane's value is `NaN`, then `NaN` is returned preferentially.

```javascript
var c = SIMD.Float64x2(NaN, Infinity)
var d = SIMD.Float64x2(1337, 42);
SIMD.Float64x2.max(c, d)
// Float64x2[NaN, Infinity]
```

The `maxNum` method has exactly the same effect as `max`, with the only difference being that if one lane's value is `NaN`, then the other lane's value is returned preferentially.

```javascript
var c = SIMD.Float64x2(NaN, Infinity)
var d = SIMD.Float64x2(1337, 42);
SIMD.Float64x2.maxNum(c, d)
// Float64x2[1337, Infinity]
```

## Static Methods: Bitwise Operations

### SIMD.%type%.and(), SIMD.%type%.or(), SIMD.%type%.xor(), SIMD.%type%.not()

The `and` method accepts two SIMD values as arguments, and returns a new SIMD value obtained by performing a binary `AND` operation (`&`) on the corresponding lanes of the two.

```javascript
var a = SIMD.Int32x4(1, 2, 4, 8);
var b = SIMD.Int32x4(5, 5, 5, 5);
SIMD.Int32x4.and(a, b)
// Int32x4[1, 0, 4, 0]
```

In the code above, taking lane `0` as an example, the binary form of `1` is `0001`, and the binary form of `5` is `01001`, so after performing the `AND` operation, `0001` is obtained.

The `or` method accepts two SIMD values as arguments, and returns a new SIMD value obtained by performing a binary `OR` operation (`|`) on the corresponding lanes of the two.

```javascript
var a = SIMD.Int32x4(1, 2, 4, 8);
var b = SIMD.Int32x4(5, 5, 5, 5);
SIMD.Int32x4.or(a, b)
// Int32x4[5, 7, 5, 13]
```

The `xor` method accepts two SIMD values as arguments, and returns a new SIMD value obtained by performing a binary "exclusive or" operation (`^`) on the corresponding lanes of the two.

```javascript
var a = SIMD.Int32x4(1, 2, 4, 8);
var b = SIMD.Int32x4(5, 5, 5, 5);
SIMD.Int32x4.xor(a, b)
// Int32x4[4, 7, 1, 13]
```

The `not` method accepts a SIMD value as an argument, and returns a new SIMD value obtained by performing a binary "not" operation (`~`) on each lane.

```javascript
var a = SIMD.Int32x4(1, 2, 4, 8);
SIMD.Int32x4.not(a)
// Int32x4[-2, -3, -5, -9]
```

In the code above, the reason the not operation of `1` yields `-2` is that, internally in the computer, negative numbers are represented in the form of "two's complement". That is, the negative form `-n` of an integer `n` is obtained by inverting every binary bit and then adding 1. Therefore, simply inverting is equivalent to the negative form minus 1; for example, the negative form of `1` is `-1`, and minus 1 yields `-2`.

## Static Methods: Data Type Conversion

SIMD provides the following methods for converting one data type to another.

- `SIMD.%type%.fromFloat32x4()`
- `SIMD.%type%.fromFloat32x4Bits()`
- `SIMD.%type%.fromFloat64x2Bits()`
- `SIMD.%type%.fromInt32x4()`
- `SIMD.%type%.fromInt32x4Bits()`
- `SIMD.%type%.fromInt16x8Bits()`
- `SIMD.%type%.fromInt8x16Bits()`
- `SIMD.%type%.fromUint32x4()`
- `SIMD.%type%.fromUint32x4Bits()`
- `SIMD.%type%.fromUint16x8Bits()`
- `SIMD.%type%.fromUint8x16Bits()`

Methods with the `Bits` suffix copy the binary bits into the new data type unchanged; methods without the suffix perform a data type conversion.

```javascript
var t = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
SIMD.Int32x4.fromFloat32x4(t);
// Int32x4[1, 2, 3, 4]

SIMD.Int32x4.fromFloat32x4Bits(t);
// Int32x4[1065353216, 1073741824, 1077936128, 1082130432]
```

In the code above, `fromFloat32x4` converts floating-point numbers to integers and then stores them into the new data type; `fromFloat32x4Bits` instead copies the binary bits into the new data type unchanged and then interprets them.

Methods with the `Bits` suffix can also be used for copying between unequal numbers of lanes.

```javascript
var t = SIMD.Float32x4(1.0, 2.0, 3.0, 4.0);
SIMD.Int16x8.fromFloat32x4Bits(t);
// Int16x8[0, 16256, 0, 16384, 0, 16448, 0, 16512]
```

In the code above, the original SIMD value `t` has 4 lanes, while the target value has 8 lanes.

If during data conversion the data size of an original lane exceeds the maximum width of the target lane, an error is thrown.

## Example: Computing an Average

In normal mode, computing the average of `n` values requires `n` operations.

```javascript
function average(list) {
  var n = list.length;
  var sum = 0.0;
  for (var i = 0; i < n; i++) {
    sum += list[i];
  }
  return sum / n;
}
```

Using SIMD, the number of computations can be reduced to a quarter of `n`.

```javascript
function average(list) {
  var n = list.length;
  var sum = SIMD.Float32x4.splat(0.0);
  for (var i = 0; i < n; i += 4) {
    sum = SIMD.Float32x4.add(
      sum,
      SIMD.Float32x4.load(list, i)
    );
  }
  var total = SIMD.Float32x4.extractLane(sum, 0) +
              SIMD.Float32x4.extractLane(sum, 1) +
              SIMD.Float32x4.extractLane(sum, 2) +
              SIMD.Float32x4.extractLane(sum, 3);
  return total / n;
}
```

The code above first reads all the values into a SIMD every four positions and immediately accumulates them. Then, it obtains the sum of the four lanes of the accumulated value, and dividing by `n` is all that is needed.
