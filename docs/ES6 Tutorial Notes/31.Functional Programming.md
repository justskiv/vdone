# Functional Programming

The JavaScript language has carried the mark of functional programming ever since its birth. It treats functions as an independent data type, on completely equal footing with other data types. In JavaScript, you can use object-oriented programming, and you can also use functional programming. Some even say that JavaScript is the first functional programming language to be adopted on a large scale in history.
<!-- more -->
The various new features of ES6 make functional programming more convenient and more powerful. This chapter introduces how to do functional programming with ES6.

## Currying

**Currying refers to splitting a multi-argument function into a series of functions, where each split function takes only one argument** (unary).

```javascript
function add (a, b) {
  return a + b;
}

add(1, 1) // 2
```

In the code above, the function `add` takes two arguments, `a` and `b`.

Currying means splitting the function above into two functions, where each function takes only one argument.

```javascript
function add (a) {
  return function (b) {
    return a + b;
  }
}
// Or using the arrow function syntax
const add = x => y => x + y;

const f = add(1);
f(1) // 2
```

In the code above, the function `add` takes only one argument `a` and returns a function `f`. The function `f` also takes only one argument `b`.

## Function Composition

Function composition refers to combining multiple functions into a single function.

```javascript
const compose = f => g => x => f(g(x));

const f = compose (x => x * 4) (x => x + 3);
f(2) // 20
```

In the code above, `compose` is a function composer used to combine two functions into one.

You can see that currying and function composition are closely related. The former is used to split a function into multiple functions, while the latter is used to merge multiple functions into one.

## Argument Flipping

Argument flipping (flip) refers to swapping the order of the first two arguments of a function.

```javascript
var divide = (a, b) => a / b;
var flip = f.flip(divide);

flip(10, 5) // 0.5
flip(1, 10) // 10

var three = (a, b, c) => [a, b, c];
var flip = f.flip(three);
flip(1, 2, 3); // => [2, 1, 3]
```

In the code above, with the normal argument order, 10 divided by 5 equals 2. But the new function obtained after flipping the arguments computes 5 divided by 10, yielding 0.5. If the original function has 3 arguments, only the positions of the first two arguments are swapped.

The code for argument flipping is very simple.

```javascript
let f = {};
f.flip =
  fn =>
    (a, b, ...args) => fn(b, a, ...args.reverse());
```

## Execution Boundary

The execution boundary (until) refers to executing a function until a condition is met.

```javascript
let condition = x => x > 100;
let inc = x => x + 1;
let until = f.until(condition, inc);

until(0) // 101

condition = x => x === 5;
until = f.until(condition, inc);

until(3) // 5
```

In the code above, the condition of the first part is to execute until `x` is greater than 100, so when `x` starts at 0, it keeps executing until 101. The condition of the second part is to execute until it equals 5, so the final value of `x` is 5.

The implementation of the execution boundary is as follows.

```javascript
let f = {};
f.until = (condition, f) =>
  (...args) => {
    var r = f.apply(null, args);
    return condition(r) ? r : f.until(condition, f)(r);
  };
```

The key point of the code above is that if the condition is met, the result is returned; otherwise it keeps executing recursively.

## List Operations

List operations include the following.

- `head`: take the first non-empty member of the list.
- `last`: take the last non-empty member of a finite list.
- `tail`: take the non-empty members other than the "list head".
- `init`: take the non-empty members other than the "list tail".

Below are examples.

```javascript
f.head(5, 27, 3, 1) // 5
f.last(5, 27, 3, 1) // 1
f.tail(5, 27, 3, 1) // [27, 3, 1]
f.init(5, 27, 3, 1) // [5, 27, 3]
```

The implementations of these methods are as follows.

```javascript
let f = {};
f.head = (...xs) => xs[0];
f.last = (...xs) => xs.slice(-1);
f.tail = (...xs) => Array.prototype.slice.call(xs, 1);
f.init = (...xs) => xs.slice(0, -1);
```

## Merge Operations

Merge operations are divided into two kinds: `concat` and `concatMap`. The former combines multiple arrays into one, while the latter first processes the arguments and then combines the processed results into a single array.

```javascript
f.concat([5], [27], [3]) // [5, 27, 3]
f.concatMap(x => 'hi ' + x, 1, [[2]], 3) // ['hi 1', 'hi 2', 'hi 3']
```

The implementation code for these two methods is as follows.

```javascript
let f = {};
f.concat =
  (...xs) => xs.reduce((a, b) => a.concat(b));
f.concatMap =
  (f, ...xs) => f.concat(xs.map(f));
```

## Pairing Operations

Pairing operations are divided into two methods: `zip` and `zipWith`. The `zip` operation pairs up the members of two lists one by one, combining them into a new list. If the two lists are not of equal length, the extra members of the longer list are ignored. The first argument of the `zipWith` operation is a function; it then pairs up the members of the following lists one by one, feeds them into that function, and the return values form a new list.

Below are examples.

```javascript
let a = [0, 1, 2];
let b = [3, 4, 5];
let c = [6, 7, 8];

f.zip(a, b) // [[0, 3], [1, 4], [2, 5]]
f.zipWith((a, b) => a + b, a, b, c) // [9, 12, 15]
```

In the code above, the first argument of the `zipWith` method is a summation function, which pairs up the members of the following three lists one by one and adds them together.

The implementations of these two methods are as follows.

```javascript
let f = {};

f.zip = (...xs) => {
  let r = [];
  let nple = [];
  let length = Math.min.apply(null, xs.map(x => x.length));

  for (var i = 0; i < length; i++) {
    xs.forEach(
      x => nple.push(x[i])
    );

    r.push(nple);
    nple = [];
  }

  return r;
};

f.zipWith = (op, ...xs) =>
  f.zip.apply(null, xs).map(
    (x) => x.reduce(op)
  );
```
