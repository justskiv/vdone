# DOM

## 1. Overview

### 1. DOM

**The DOM is the interface through which JavaScript manipulates web pages**, with the full name "**Document Object Model**". Its **purpose** is to **turn a web page into a JavaScript object, so that you can perform all kinds of operations on it via scripts** (such as adding or removing content).

Based on the DOM model, the browser parses a structured document (such as HTML and XML) into a series of nodes, and these nodes form a tree structure (the DOM Tree). All of the nodes, and the final tree structure, have standardized public interfaces.

The DOM is only an interface specification, and it can be implemented in various languages. So strictly speaking, the DOM is not part of the JavaScript syntax, but DOM manipulation is one of the most common tasks in JavaScript—without the DOM, JavaScript cannot control web pages. On the other hand, JavaScript is also the language most commonly used for DOM manipulation. What follows describes JavaScript's implementation and usage of the DOM standard.



### 2. Nodes

**The smallest unit that makes up the DOM is called a node**. The tree structure of a document (the DOM tree) is composed of various different types of nodes. Each node can be thought of as a leaf of the document tree.

There are seven types of nodes.

- `Document`: the top-level node of the entire document tree
- `DocumentType`: the `doctype` tag (such as ` <!DOCTYPE html> `)
- `Element`: the various HTML tags of a web page (such as ` <body> `, `  <a>  `, etc.)
- `Attribute`: the attributes of a web page element (such as `class="right"`)
- `Text`: the text between tags or contained within a tag
- `Comment`: comments
- `DocumentFragment`: a fragment of a document

The browser provides a native node object `Node`. All seven of the node types above inherit from `Node`, and therefore share some common properties and methods.



### 3. The node tree

All nodes of a document can be abstracted into a tree structure based on their hierarchical level. This tree structure is the DOM tree. It has a top-level node; the next level down are all child nodes of the top-level node, and those child nodes in turn have their own child nodes, deriving level by level into a pyramid structure that, turned upside down, looks like a tree.

The browser natively provides a `document` node, which represents the entire document.

```js
document
// the entire document tree
```

The first level of the document has two nodes: the first is the document type node (` <!doctype html> `), and the second is ` <html> `, the top-level container tag of the HTML page. The latter forms the root node of the tree structure, and all other HTML tag nodes are its descendants.

Apart from the root node, every node has three kinds of hierarchical relationships.

- Parent relationship (parentNode): the immediate node one level up
- Child relationship (childNodes): the immediate nodes one level down
- Sibling relationship (sibling): nodes that share the same parent node

The DOM provides operation interfaces to obtain nodes in these three relationships. For example, the child-node interface includes properties such as `firstChild` (the first child node) and `lastChild` (the last child node), and the sibling interface includes the `nextSibling` (the immediately following sibling node) and `previousSibling` (the immediately preceding sibling node) properties.



## 2. The Node interface

 **All DOM node objects inherit from the Node interface, and share some common properties and methods**. This is the foundation of DOM manipulation.

### 1. Properties

#### 1.1 Node.prototype.nodeType (node type)

The `nodeType` property **returns an integer indicating the type of the node**.

```js
document.nodeType // 9
```

In the code above, the type value of the document node is 9.

The Node object defines several constants that correspond to these type values.

```js
document.nodeType === Node.DOCUMENT_NODE // true
```

In the code above, the `nodeType` property of the document node equals the constant `Node.DOCUMENT_NODE`.

The `nodeType` property values of different nodes, and their corresponding constants, are as follows.

- Document node (document): 9, corresponding to the constant `Node.DOCUMENT_NODE`
- Element node (element): 1, corresponding to the constant `Node.ELEMENT_NODE`
- Attribute node (attr): 2, corresponding to the constant `Node.ATTRIBUTE_NODE`
- Text node (text): 3, corresponding to the constant `Node.TEXT_NODE`
- Document fragment node (DocumentFragment): 11, corresponding to the constant `Node.DOCUMENT_FRAGMENT_NODE`
- Document type node (DocumentType): 10, corresponding to the constant `Node.DOCUMENT_TYPE_NODE`
- Comment node (Comment): 8, corresponding to the constant `Node.COMMENT_NODE`

When determining a node's type, using the `nodeType` property is the common approach.

```js
var node = document.documentElement.firstChild; // gets the <head> node
if (node.nodeType === Node.ELEMENT_NODE) {
  console.log('this node is an element node');
}
```

#### 1.2 Node.prototype.nodeName (node name)

The `nodeName` property **returns the name of the node**.

```js
// HTML code is as follows
// <div id="d1">hello world</div>
var div = document.getElementById('d1');
div.nodeName // "DIV"

document.nodeName // #document
```

In the code above, the `nodeName` property of the element node `  <div>  ` is the uppercase tag name `DIV`.

The `nodeName` property values of different nodes are as follows.

- Document node (document): `#document`
- Element node (element): the uppercase tag name, such as DIV
- Attribute node (attr): the name of the attribute
- Text node (text): `#text`
- Document fragment node (DocumentFragment): `#document-fragment`
- Document type node (DocumentType): the type of the document
- Comment node (Comment): `#comment`

#### 1.3 Node.prototype.nodeValue (text of the current node)

The `nodeValue` property **returns a string representing the text value of the current node itself**; this property is **readable and writable**.

Only **text nodes (text), comment nodes (comment), and attribute nodes (attr) have a text value**, so for these three kinds of nodes `nodeValue` returns a result, while all other types of nodes return `null`. Likewise, only these three kinds of nodes can have their `nodeValue` property set; setting it on other types of nodes has no effect.

```js
// HTML code is as follows
// <div id="d1">hello world</div>
var div = document.getElementById('d1');
div.nodeValue // null
div.firstChild.nodeValue // "hello world"

var textNode = div.firstChild;
console.log(textNode) // text node object "hello world"
console.log(typeof textNode) // object
console.log(textNode.nodeType) // 3
console.log(textNode.nodeName) // #text
console.log(textNode.nodeValue) // 'hello world'

textNode.nodeValue = '123' // modify the text
console.log(textNode.nodeValue) // '123'
```

In the code above, `div` is an element node, so the `nodeValue` property returns `null`. `div.firstChild` is a text node, so it can return a text value.



#### 1.4 Node.prototype.textContent (all text content)

The `textContent` property **returns the text content of the current node and all of its descendant nodes**.

```js
// HTML code is
// <div id="divA">This is <span>some</span> text</div>

document.getElementById('divA').textContent
// This is some text
```

The `textContent` property automatically ignores the HTML tags inside the current node and returns all the text content.

This property is **readable and writable**. Setting its value replaces all the original child nodes with a single new text node. It also has the advantage of automatically escaping HTML tags. This makes it well suited for user-provided content.

```js
document.getElementById('foo').textContent = '<p>GoodBye!</p>'; // write text
```

When inserting text, the code above **treats the `  <p>  ` tag as text** rather than processing it as a tag.

For text nodes (text), comment nodes (comment), and attribute nodes (attr), the value of the `textContent` property is the same as the `nodeValue` property. For other types of nodes, this property concatenates the content of each child node (excluding comment nodes) and returns it. If a node has no child nodes, it returns an empty string.

The `textContent` property of document nodes (document) and document type nodes (doctype) is `null`. To **read the content of the entire document**, you can use `document.documentElement.textContent`.



#### 1.5 Node.prototype.baseURI (get the web page path)

The `baseURI` property **returns a string representing the absolute path of the current web page**. Based on this property, the browser computes the URLs of relative paths on the page. This property is **read-only**.

```js
// the URL of the current page is
// http://www.example.com/index.html
document.baseURI
// "http://www.example.com/index.html"
```

If the page's URL cannot be read, the `baseURI` property returns `null`.

The value of this property is generally determined by the URL of the current page (i.e. the `window.location` property), but you can use the HTML ` <base> ` tag to change its value.

```html
<head>
<base href="http://www.example.com/page.html">
</head>
<script>
    document.baseURI // http://www.example.com/page.html   returns the href value of the base tag
</script>
```

Once set, the `baseURI` property returns the value set by the ` <base> ` tag.



#### 1.6 Node.prototype.ownerDocument (top-level document object)

The `Node.ownerDocument` property **returns the top-level document object that the current node belongs to**, i.e. the `document` object.

```js
var d = p.ownerDocument;
d === document // true
```

The `ownerDocument` property of the `document` object itself returns `null`.



#### 1.7 Node.prototype.nextSibling (the next sibling node)

The `Node.nextSibling` property **returns the first sibling node immediately following the current node**. If there is no sibling node after the current node, it returns `null`.

```js
// HTML code is as follows
// <div id="d1">hello</div><div id="d2">world</div>
var d1 = document.getElementById('d1');
var d2 = document.getElementById('d2');

d1.nextSibling === d2 // true  // if there is whitespace between d1 and d2, a text node containing the whitespace is returned
```

In the code above, `d1.nextSibling` is `d2`, the sibling node immediately following `d1`.

**Note** that this property also includes text nodes and comment nodes (` <!-- comment --> `). So if there is whitespace after the current node, this property returns a text node whose content is the whitespace.

The `nextSibling` property can be used to traverse all child nodes.

```js
var el = document.getElementById('div1').firstChild;

while (el !== null) {
  console.log(el.nodeName);
  el = el.nextSibling;
}
```

The code above traverses all child nodes of the `div1` node.



#### 1.8 Node.prototype.previousSibling (the previous sibling node)

The `previousSibling` property **returns the nearest sibling node preceding the current node**. If there is no sibling node before the current node, it returns `null`.

```js
// HTML code is as follows
// <div id="d1">hello</div><div id="d2">world</div>
var d1 = document.getElementById('d1');
var d2 = document.getElementById('d2');

d2.previousSibling === d1 // true
```

In the code above, `d2.previousSibling` is `d1`, the sibling node preceding `d2`.

**Note** that this property also includes text nodes and comment nodes. So if there is whitespace before the current node, this property returns a text node whose content is the whitespace.



#### 1.9 Node.prototype.parentNode (parent node)

The `parentNode` property **returns the parent node of the current node**. For any node, its parent node can only be one of three types: an element node (element), a document node (document), or a document fragment node (documentfragment).

```js
if (node.parentNode) {
  node.parentNode.removeChild(node); // get node's parent, then remove the child node `node`
}
```

In the code above, the `node.parentNode` property is used to remove the `node` node from the document.

The parent node of both document nodes (document) and document fragment nodes (documentfragment) is `null`. In addition, for nodes that have been created but not yet inserted into the DOM tree, the parent node is also `null`.



#### 1.10 Node.prototype.parentElement (parent element node)

The `parentElement` property **returns the parent element node of the current node**. If the current node has no parent node, or the parent node is not an element node, it returns `null`.

```
if (node.parentElement) {
  node.parentElement.style.color = 'red';
}
```

In the code above, the parent element node's style is set to red.

Since a parent node can only be one of three types—an element node, a document node (document), or a document fragment node (documentfragment)—the `parentElement` property effectively excludes the latter two kinds of parent nodes.



#### 1.11 Node.prototype.firstChild (first child node), Node.prototype.lastChild (last child node)

The `firstChild` property **returns the first child node of the current node**; if the current node has no child nodes, it returns `null`.

```js
// HTML code is as follows
// <p id="p1"><span>First span</span></p>
var p1 = document.getElementById('p1');
p1.firstChild.nodeName // "SPAN"
```

In the code above, the first child node of the `p` element is the `span` element.

**Note** that besides element nodes, `firstChild` **may also return a text node or a comment node**.

```js
// HTML code is as follows
// <p id="p1">
//   <span>First span</span>
//  </p>
var p1 = document.getElementById('p1');
p1.firstChild.nodeName // "#text"
```

In the code above, there is whitespace between the `p` element and the `span` element, which causes `firstChild` to return a text node.

The `lastChild` property **returns the last child node of the current node**; if the current node has no child nodes, it returns `null`. Its usage is the same as the `firstChild` property.

#### 1.12 Node.prototype.childNodes (all child nodes; an array-like object; a live collection)

The `childNodes` property **returns an array-like object** (a `NodeList` collection) **whose members include all the child nodes of the current node**.

```js
var children = document.querySelector('ul').childNodes; // includes element nodes, text nodes, and comment nodes
```

In the code above, `children` is all the child nodes of the `ul` element.

Using this property, you can traverse all the child nodes of a given node.

```js
var div = document.getElementById('div1');
var children = div.childNodes;

for (var i = 0; i < children.length; i++) {
  // ...
}
```

The document node (document) has exactly two child nodes: the document type node (docType) and the HTML root element node.

```js
var children = document.childNodes;
for (var i = 0; i < children.length; i++) {
  console.log(children[i].nodeType);
}
// 10
// 1
```

In the code above, the type of the document node's first child is 10 (i.e. the document type node), and the type of its second child is 1 (i.e. the element node).

**Note** that besides element nodes, the return value of the `childNodes` property **also includes text nodes and comment nodes**. If the current node has no child nodes, it returns an empty `NodeList` collection. Since the `NodeList` object is a **live collection**, any change in the child nodes is immediately reflected in the returned result.

#### 1.13 Node.prototype.isConnected (whether the node is in the document; a boolean)

The `isConnected` property **returns a boolean indicating whether the current node is in the document**.

```js
var test = document.createElement('p');
test.isConnected // false

document.body.appendChild(test);
test.isConnected // true
```

In the code above, the `test` node is a script-generated node. Before it is inserted into the document, the `isConnected` property returns `false`, and after insertion it returns `true`.



### 2. Methods

#### 2.1 Node.prototype.appendChild() (insert a child node at the end)

The `appendChild()` method **accepts a node object as its argument and inserts it as the last child node of the current node**. Its **return value is the child node that was inserted into the document**.

```js
var p = document.createElement('p');
document.body.appendChild(p); // has a return value, the inserted p node
```

The code above creates a new `  <p>  ` node and inserts it at the end of `document.body`.

If the argument node is a node that already exists in the DOM, the `appendChild()` method moves it from its original position to the new position.

```js
var div = document.getElementById('myDiv');// myDiv is an existing element
document.body.appendChild(div); // inserting an existing element is equivalent to moving it
```

In the code above, the inserted node `myDiv` already exists, so the result is that this node is moved from its original position to the end of `document.body`.

If the argument to the `appendChild()` method is a `DocumentFragment` node, then what is inserted is all the child nodes of the `DocumentFragment`, not the `DocumentFragment` node itself. The return value is an empty `DocumentFragment` node.



#### 2.2 Node.prototype.hasChildNodes() (whether the node has child nodes; a boolean)

The `hasChildNodes` method returns a **boolean** **indicating whether the current node has child nodes**.

```js
var foo = document.getElementById('foo');

if (foo.hasChildNodes()) {
  foo.removeChild(foo.childNodes[0]);
}
```

The code above means: if the `foo` node has child nodes, remove its first child node.

**Note** that **child nodes include all types of nodes**, not just element nodes. Even if a node contains only a single space, the `hasChildNodes` method still returns `true`.

There are many ways to determine whether a node has child nodes; here are three of them.

- `node.hasChildNodes()`
- `node.firstChild !== null`
- `node.childNodes && node.childNodes.length > 0`

The `hasChildNodes` method, combined with the `firstChild` and `nextSibling` properties, can be used to **traverse all descendant nodes of the current node**.

```js
function DOMComb(parent, callback) {
  if (parent.hasChildNodes()) {
    for (var node = parent.firstChild; node; node = node.nextSibling) {
      DOMComb(node, callback);
    }
  }
  callback(parent);
}

// usage
DOMComb(document.body, console.log)
```

In the code above, the first argument of the `DOMComb` function is a specified node, and the second argument is a callback function. This callback is applied in turn to the specified node and all of its descendant nodes.

#### 2.3 Node.prototype.cloneNode(Boolean) (clone a node)

The `cloneNode` method is used to **clone a node**. **It accepts a boolean argument indicating whether to clone the child nodes at the same time**. Its return value is a newly cloned node.

```js
var cloneUL = document.querySelector('ul').cloneNode(true);
```

There are some **things to note** when using this method.

(1) Cloning a node copies all of its attributes, but it loses event callback functions added to the node via the `addEventListener` method and `on-` properties (i.e. `node.onclick = fn`).

(2) The node returned by this method is not in the document, i.e. it has no parent node; you must use a method such as `Node.appendChild` to add it to the document.

(3) After cloning a node, the DOM may end up with two web page elements that have the same `id` attribute (i.e. `id="xxx"`); in this case you should modify the `id` attribute of one of them. If the original node has a `name` attribute, you may need to modify that as well.



#### 2.4 Node.prototype.insertBefore() (insert a node at a specified position)

The `insertBefore` method is used to **insert a node at a specified position inside a parent node**.

```js
var insertedNode = parentNode.insertBefore(newNode, referenceNode);
// the new node newNode is inserted before the referenceNode inside parentNode, and the new node is returned
```

The `insertBefore` method accepts two arguments. The first is the node to insert, `newNode`, and the second is a child node `referenceNode` inside the parent node `parentNode`. `newNode` is inserted before the `referenceNode` child node. The return value is the inserted new node, `newNode`.

```js
var p = document.createElement('p');
document.body.insertBefore(p, document.body.firstChild);// insert the p node before the first element inside body
```

In the code above, a new `  <p>  ` node is created and inserted before `document.body.firstChild`, that is, it becomes the first child node of `document.body`.

If the second argument of the `insertBefore` method is `null`, then the new node is inserted at the last position inside the current node, i.e. it becomes the last child node.

```js
var p = document.createElement('p');
document.body.insertBefore(p, null);
```

In the code above, `p` becomes the last child node of `document.body`. This also shows that the second argument of `insertBefore` cannot be omitted.

**Note** that if the node to be inserted is a node that already exists in the current DOM, that node is removed from its original position and inserted at the new position.

Since there is no `insertAfter` method, if you want to insert a new node **after a child node** of the parent, you can simulate it with the `insertBefore` method combined with the `nextSibling` property.

**Insert after a child node**

```js
parent.insertBefore(s1, s2.nextSibling);
```

In the code above, `parent` is the parent node, `s1` is a brand-new node, and `s2` is a child node; this inserts `s1` after `s2`. If `s2` is the last child node of the current node, then `s2.nextSibling` returns `null`, and in that case `s1` is inserted at the end of the current node, becoming its last child node, which is equivalent to being placed immediately after `s2`.

If the node to be inserted is of type `DocumentFragment`, then what is inserted is all the child nodes of the `DocumentFragment`, not the `DocumentFragment` node itself. The return value is an empty `DocumentFragment` node.



#### 2.5 Node.prototype.removeChild() (remove a child node and return it)

The `removeChild` method **accepts a child node as its argument and removes that child node from the current node. The return value is the removed child node**.

```js
var divA = document.getElementById('A');
divA.parentNode.removeChild(divA); // first get the parent node, then call the remove-child method, specifying divA as the child to remove
```

The code above removes the `divA` node. **Note that this method is called on `divA`'s parent node, not on `divA` itself.**

The following is how to **remove all child nodes of the current node**.

```js
var element = document.getElementById('top');
while (element.firstChild) {
  element.removeChild(element.firstChild);
}
```

**The removed node still exists in memory**, but it is no longer part of the DOM. So after a node is removed, you can still use it, for example by inserting it under another node.

If the argument node is not a child of the current node, the `removeChild` method throws an error.



#### 2.6 Node.prototype.replaceChild() (replace a child node and return the replaced one)

The `replaceChild` method is **used to replace one of the current node's child nodes with a new node**. **It returns the replaced child node.**

```js
var replacedNode = parentNode.replaceChild(newChild, oldChild);
```

In the code above, the `replaceChild` method accepts two arguments: the first, `newChild`, is the new node used to replace, and the second, `oldChild`, is the child node to be replaced. The return value is the replaced node, `oldChild`.

```js
var divA = document.getElementById('divA');
var newSpan = document.createElement('span');
newSpan.textContent = 'Hello World!';
divA.parentNode.replaceChild(newSpan, divA);
```

The code above replaces the specified node `divA` with the new node `newSpan`.



#### 2.7 Node.prototype.contains() (whether the argument node is a descendant or the current node; returns a boolean)

The `contains` method **returns a boolean indicating whether the argument node satisfies one of the following three conditions**.

- The argument node is the current node.
- The argument node is a child node of the current node.
- The argument node is a descendant node of the current node.

**Whether the current node contains the argument node**

```js
document.body.contains(node)
```

The code above checks whether the argument node `node` is contained within the current document.

**Note** that passing the current node into the `contains` method returns `true`.

```js
nodeA.contains(nodeA) // true
```



#### 2.8 Node.prototype.compareDocumentPosition() (similar to contains; its return value can be used to work out the positional relationship)

The usage of the `compareDocumentPosition` method **is exactly the same as the `contains` method; it returns a six-bit binary value representing the relationship between the argument node and the current node**.

| Binary value | Decimal value | Meaning                                              |
| :----------- | :------------ | :-------------------------------------------------- |
| 000000       | 0             | The two nodes are the same                          |
| 000001       | 1             | The two nodes are not in the same document (i.e. one node is not in the current document) |
| 000010       | 2             | The argument node precedes the current node         |
| 000100       | 4             | The argument node follows the current node          |
| 001000       | 8             | The argument node contains the current node         |
| 010000       | 16            | The current node contains the argument node         |
| 100000       | 32            | For internal browser use                            |

```js
// HTML code is as follows
// <div id="mydiv">
//   <form><input id="test" /></form>
// </div>

var div = document.getElementById('mydiv');
var input = document.getElementById('test');

div.compareDocumentPosition(input) // 20: current node contains argument node (16) + argument node follows current node (4)
input.compareDocumentPosition(div) // 10: argument node contains current node (8) + argument node precedes current node (2)
```

In the code above, the `div` node contains the `input` node (binary `010000`), and the `input` node follows the `div` node (binary `000100`), so the first `compareDocumentPosition` method returns `20` (binary `010100`, i.e. `010000 + 000100`), and the second `compareDocumentPosition` method returns `10` (binary `001010`).

Since the meaning of the `compareDocumentPosition` return value is defined on each bit, if you want to check for a particular meaning, you need to use bitwise operators.

```js
var head = document.head;
var body = document.body;
if (head.compareDocumentPosition(body) & 4) { // 4 means the argument node follows the current node
  console.log('document structure is correct');
} else {
  console.log('<body> cannot come before <head>');
}
```

In the code above, the return value of `compareDocumentPosition` is bitwise-ANDed (`&`) with `4` (also called a mask) to get a boolean indicating whether `<body>` is before `<head>`.



#### 2.9 Node.prototype.isEqualNode() (whether two nodes are equal); Node.prototype.isSameNode() (whether two nodes are the same)

The `isEqualNode` method **returns a boolean used to check whether two nodes are equal. Equal nodes are nodes that have the same type, the same attributes, and the same child nodes.**

```js
var p1 = document.createElement('p');
var p2 = document.createElement('p');

p1.isEqualNode(p2) // true
```

The `isSameNode` method **returns a boolean indicating whether two nodes are the same node**.

```js
var p1 = document.createElement('p');
var p2 = document.createElement('p');

p1.isSameNode(p2) // false
p1.isSameNode(p1) // true
```

#### 2.10 Node.prototype.normalize() ("normalizes" the current node and its descendant nodes)

The `normalize` method **"normalizes" the current node and its descendant nodes**. In a "normalized" DOM tree, there is no empty text node, nor two adjacent text nodes.

Note 1: An "empty text node" does not include a text node made up of whitespace characters (spaces, line breaks, etc.).

Note 2: Causes of two or more adjacent text nodes include:

1. Inserting and splitting text nodes via scripts calling the relevant DOM interfaces.
2. Excessively long text nodes in HTML being automatically split by the browser into multiple adjacent text nodes.

```js
var wrapper = document.createElement("div");

wrapper.appendChild(document.createTextNode("Part 1 "));
wrapper.appendChild(document.createTextNode("Part 2 "));

// at this point (before normalization), wrapper.childNodes.length === 2
// wrapper.childNodes[0].textContent === "Part 1 "
// wrapper.childNodes[1].textContent === "Part 2 "

wrapper.normalize();
// now (after normalization), wrapper.childNodes.length === 1
// wrapper.childNodes[0].textContent === "Part 1 Part 2"
```

Before the `normalize` method is used, the `wrapper` node above has two adjacent text child nodes. After the `normalize` method is used, the two text child nodes are merged into one.

This method is the inverse of `Text.splitText`; see the chapter "The Text Node Object" for more details.

#### 2.11 Node.prototype.getRootNode() (get the root node, document)

The `getRootNode()` method **returns the root node `document` of the document the current node belongs to, which has the same effect as the `ownerDocument` property**.

```js
document.body.firstChild.getRootNode() === document
// true
document.body.firstChild.getRootNode() === document.body.firstChild.ownerDocument
// true
```

This method can be used on the `document` node itself, which differs from `document.ownerDocument`.

```js
document.getRootNode() // document
document.ownerDocument // null
```



## 3. The NodeList interface and the HTMLCollection interface (collections of nodes)

Nodes are individual objects, but sometimes you need a data structure that can hold multiple nodes. The DOM provides two kinds of **node collections** for holding multiple nodes: `NodeList` and `HTMLCollection`.

Both kinds of collections are interface specifications. Many DOM properties and methods return a `NodeList` instance or an `HTMLCollection` instance. The main difference is that **`NodeList` can contain various types of nodes, whereas `HTMLCollection` can only contain HTML element nodes**.



### 1. The NodeList interface (a collection of nodes of various kinds)

#### 1.1 Overview

A `NodeList` instance is an **array-like object whose members are node objects**. You can obtain a `NodeList` instance through the following methods.

- **The `Node.childNodes` property**
- **Node search methods such as `document.querySelectorAll()`**

```js
document.body.childNodes instanceof NodeList // true

/*
<ul>
    <li></li>
    <li></li>
    <li></li>
</ul>*/

var $li = document.querySelectorAll('li');
console.log($li) // NodeList(3) [li, li, li]
console.log($li instanceof NodeList) // true

```

A `NodeList` instance is very much like an array; **you can use its `length` property and `forEach` method**. However, it is not an array, so you cannot use array-specific methods such as `pop` or `push`.

```js
var children = document.body.childNodes;

Array.isArray(children) // false

children.length // 34
children.forEach(console.log)
```

In the code above, the NodeList instance `children` is not an array, but it does have a `length` property and a `forEach` method.

If you want to use array methods on a `NodeList` instance, **you can convert it into a real array**.

```js
var children = document.body.childNodes;
var nodeArr = Array.prototype.slice.call(children);
```

Besides using the `forEach` method to traverse a NodeList instance, **you can also use a `for` loop**.

```js
var children = document.body.childNodes;

for (var i = 0; i < children.length; i++) {
  var item = children[i];
}
```

Note that a NodeList instance **may be a live collection or a static collection**. A live collection is a living collection: removing or adding a relevant node in the DOM is immediately reflected in the NodeList instance. **Currently, only `Node.childNodes` returns a live collection; all other NodeLists are static collections.**

```js
var children = document.body.childNodes;
children.length // 18
document.body.appendChild(document.createElement('p'));
children.length // 19
```

In the code above, after a child node is added to the document, the `length` property of the NodeList instance `children` increases by 1.



#### 1.2 NodeList.prototype.length

The `length` property returns the number of nodes contained in the NodeList instance.

```js
document.querySelectorAll('xxx').length
// 0
```

In the code above, `document.querySelectorAll` returns a NodeList collection. For HTML tags that do not exist, the `length` property returns `0`.



#### 1.3 NodeList.prototype.forEach()

The `forEach` method is used to traverse all members of a NodeList. It accepts a callback function as its argument, and executes this callback once per iteration; its usage is exactly the same as the `forEach` method of array instances.

```js
var children = document.body.childNodes;
children.forEach(function f(item, i, list) { // arguments: current member, current index, NodeList instance
  // ...
}, this); // forEach second argument: binds `this` inside the callback; can be omitted
```

In the code above, the three arguments of the callback `f` are, in order, **the current member, its position, and the current NodeList instance**. The second argument of the `forEach` method is used to bind `this` inside the callback; this argument can be omitted.



#### 1.4 NodeList.prototype.item() (accepts an index argument and returns the member, a node, at that index)

The `item` method **accepts an integer as its argument representing the position of a member, and returns the member at that position.**

```js
document.body.childNodes.item(0)
```

In the code above, `item(0)` returns the first member.

If the argument value is greater than the actual length, or the index is invalid (such as a negative number), the `item` method returns `null`. If the argument is omitted, the `item` method throws an error.

All array-like objects can use the bracket operator to retrieve members. **In general, you use the bracket operator rather than the `item` method.**

```js
document.body.childNodes[0]
```



#### 1.5 NodeList.prototype.keys(), NodeList.prototype.values(), NodeList.prototype.entries() (iterator objects; use with for...of to iterate over the information of each member)

These three methods all return an ES6 iterator object, which you can iterate over with a `for...of` loop to get the information of each member. The difference is that `keys()` returns an iterator of **keys**, `values()` returns an iterator of **values**, and `entries()` returns an iterator **of arrays containing both the key and the value**.

```js
var children = document.body.childNodes;

for (var key of children.keys()) {
  console.log(key); // returns the key
}
// 0
// 1
// 2
// ...

for (var value of children.values()) {
  console.log(value); // returns the value (i.e. the node name)
}
// #text
// <script>
// ...

for (var entry of children.entries()) {
  console.log(entry); // returns an array of key and value
}
// Array [ 0, #text ]
// Array [ 1, <script> ]
// ...
```



### 2. The HTMLCollection interface (a collection of HTML element nodes)

#### 2.1 Overview

`HTMLCollection` is a **collection of node objects that can only contain element nodes** (element), and cannot contain other types of nodes. Its return value is an **array-like object**, but unlike the `NodeList` interface, `HTMLCollection` **has no `forEach` method; you can only traverse it with a `for` loop.**

What returns an `HTMLCollection` instance is mainly some collection properties of the `Document` object, such as `document.links`, `document.forms`, `document.images`, etc.

```js
document.links instanceof HTMLCollection // true
```

`HTMLCollection` instances are all **live collections**; changes to nodes are reflected in the collection in real time.

If an element node has an `id` or `name` attribute, then on the `HTMLCollection` instance you can reference that element node using the `id` attribute or the `name` attribute. If there is no corresponding node, it returns `null`.

```js
// HTML code is as follows
// <img id="pic" src="http://example.com/foo.jpg">

var pic = document.getElementById('pic');
document.images.pic === pic // true
```

In the code above, `document.images` is an `HTMLCollection` instance, and you can retrieve the `  <img>  ` element from the `HTMLCollection` instance using its `id` attribute value.



###### Comparison of the data returned by the various JS selectors

```html
<ul>
    <li id="li1"></li>
    <li class="li2"></li>
    <li name='liName'></li>
 </ul>

<script>
    var li1 = document.getElementById('li1');
    var li2 = document.querySelector('#li1');

    var li3 = document.getElementsByClassName('li2');
    var li4 = document.getElementsByTagName('li');

    var li5 = document.getElementsByName('liName');
    var li6 = document.querySelectorAll('li');

    console.log(li1) // <li id="li1"></li>
    console.log(li2) // <li id="li1"></li>

    console.log(li3) // HTMLCollection [li.li2]
    console.log(li4) // HTMLCollection(3) [li#li1, li.li2, li, li1: li#li1, liName: li]

    console.log(li5) // NodeList [li]
    console.log(li6) // NodeList(3) [li#li1, li.li2, li]

</script>
```





#### 2.2 HTMLCollection.prototype.length

The `length` property returns the number of members contained in the `HTMLCollection` instance.

```js
document.links.length // 18
```



#### 2.3 HTMLCollection.prototype.item() (same as bracket access)

The `item` method accepts an integer as its argument representing the position of a member, and returns the member at that position.

```js
var c = document.images;
var img0 = c.item(0);
```

In the code above, `item(0)` returns the member at position 0. Since the **bracket operator does the same thing and is more convenient to use, in general you always use the bracket operator**.

If the argument value exceeds the number of members or is invalid (such as less than 0), the `item` method returns `null`.



#### 2.4 HTMLCollection.prototype.namedItem() (the argument is an id or name; returns the element node)

The **argument of the `namedItem` method is a string representing the value of an `id` attribute or `name` attribute, and it returns the corresponding element node**. If there is no corresponding node, it returns `null`.

```js
// HTML code is as follows
// <img id="pic" src="http://example.com/foo.jpg">

var pic = document.getElementById('pic');
document.images.namedItem('pic') === pic // true
```



## 4. The ParentNode interface (the parent-node interface) and the ChildNode interface (the child-node interface)

 Besides inheriting from the Node interface, node objects also inherit from other interfaces. The `ParentNode` interface **indicates that the current node is a parent node, and provides some methods for handling child nodes**. The `ChildNode` interface **indicates that the current node is a child node, and provides some related methods.**



### 1. The ParentNode interface

**If the current node is a parent node, it inherits the `ParentNode` interface**. Since only **element nodes** (element), **document nodes** (document), and **document fragment nodes** (documentFragment) can have child nodes, only these three kinds of nodes inherit the `ParentNode` interface.

#### 1.1 Properties

#### (1) ParentNode.children (returns an HTMLCollection instance of all element child nodes of the current node)

The `children` property **returns an `HTMLCollection` instance whose members are all the element child nodes of the current node**. This property is **read-only**.

The following is an example of traversing all the element child nodes of a node.

```js
for (var i = 0; i < el.children.length; i++) {
  // ...
}
```

Note that the `children` property **includes only element child nodes**, not other types of child nodes (such as text child nodes). If there are no element-type child nodes, the `length` property of the returned `HTMLCollection` instance is `0`.

In addition, `HTMLCollection` is a live collection, and it reflects any changes to the DOM in real time.



#### (2) ParentNode.firstElementChild (the first element child node)

The `firstElementChild` property **returns the first element child node of the current node**. If there are no element child nodes, it returns `null`.

```js
document.firstElementChild.nodeName
// "HTML"
```

In the code above, the first element child node of the `document` node is `<html>`.



#### (3) ParentNode.lastElementChild (the last element child node)

The `lastElementChild` property **returns the last element child node of the current node**; if there are no element child nodes, it returns `null`.

```js
document.lastElementChild.nodeName
// "HTML"
```

In the code above, the last element child node of the `document` node is `<html>` (because `document` contains only this one element child node).



#### (4) ParentNode.childElementCount (the number of all element child nodes)

The `childElementCount` property **returns an integer indicating the number of all element child nodes of the current node**. If it contains no element child nodes, it returns `0`.

```js
document.body.childElementCount // 13
```



#### 1.2 Methods

##### ParentNode.append() (insert child nodes at the end), ParentNode.prepend() (insert child nodes at the beginning)

The `append` method **appends one or more child nodes to the current node, at the position after the last element child node**.

This method can add not only element child nodes but also text child nodes.

```js
var parent = document.body;

// add an element child node
var p = document.createElement('p');
parent.append(p);

// add a text child node
parent.append('Hello');

// add multiple element child nodes
var p1 = document.createElement('p');
var p2 = document.createElement('p');
parent.append(p1, p2);

// add an element child node and a text child node
var p = document.createElement('p');
parent.append('Hello', p);
```

Note that this method **has no return value**.

The `prepend` method **appends one or more child nodes to the current node, at the position before the first element child node**. Its usage is exactly the same as the `append` method, and it likewise **has no return value**.



### 2. The ChildNode interface

**If a node has a parent node, then that node inherits the `ChildNode` interface.**

#### 2.1 ChildNode.remove() (remove the current node)

The `remove` method is used to **remove the current node** from its parent node.

```js
el.remove()
```

The code above removes the `el` node from the DOM.



#### 2.2 ChildNode.before() (insert sibling nodes before), ChildNode.after() (insert sibling nodes after)

The `before` method is used to **insert one or more sibling nodes before the current node**. Both share the same parent node.

Note that this method can insert not only element nodes but also text nodes.

```js
var p = document.createElement('p');
var p1 = document.createElement('p');

// insert an element node
el.before(p);

// insert a text node
el.before('Hello');

// insert multiple element nodes
el.before(p, p1);

// insert an element node and a text node
el.before(p, 'Hello');
```

The `after` method is used to **insert one or more sibling nodes after the current node**, and both share the same parent node. Its usage is exactly the same as the `before` method.



#### 2.3 ChildNode.replaceWith() (replace the current node with the argument node)

The `replaceWith` method **replaces the current node with the argument node.** The argument can be an element node or a text node.

```js
var span = document.createElement('span');
el.replaceWith(span);
```

In the code above, the `el` node is replaced by the `span` node.



## 5. The Document node

### 1. Overview

The `document` node object **represents the entire document**; every web page has its own `document` object. The `window.document` property points to this object. As soon as the browser begins loading the HTML document, this object exists and can be used directly.

There are different ways to obtain the `document` object.

- For a normal web page, use `document` or `window.document` directly.
- For a page inside an `iframe`, use the `contentDocument` property of the `iframe` node.
- For a document returned by an Ajax operation, use the `responseXML` property of the `XMLHttpRequest` object.
- The `ownerDocument` property of an internal node.

The `document` object **inherits the `EventTarget` interface, the `Node` interface, and the `ParentNode` interface**. This means the methods of these interfaces can all be called on the `document` object. Beyond that, the `document` object also has many of its own properties and methods.



### 2. Properties

#### 2.1 Shortcut properties (shortcuts pointing to a particular node inside the document)

**The following properties are shortcuts that point to a particular node inside the document.**

##### (1) document.defaultView (returns the window object)

The `document.defaultView` property **returns the `window` object that the `document` object belongs to**. If the current document does not belong to a `window` object, this property returns `null`.

```js
document.defaultView === window // true
```



##### (2) document.doctype (points to `<!DOCTYPE html>`)

For an HTML document, the `document` object **generally has two child nodes**. The first child node is `document.doctype`, which points to the ` <DOCTYPE> ` node, i.e. the document type node (Document Type Declaration, abbreviated DTD). The document type node for HTML is generally written as ` <!DOCTYPE html> `. If the page does not declare a DTD, this property returns `null`.

```js
var doctype = document.doctype;
doctype // "<!DOCTYPE html>"
doctype.name // "html"
```

`document.firstChild` usually returns this node.



##### (3) document.documentElement (returns the root element node of the current document, i.e. the \<html\> node)

The `document.documentElement` property **returns the root element node (root) of the current document**. It is usually the second child node of the `document` node, immediately following the `document.doctype` node. For an HTML page, this property is generally the ` <html> ` node.



##### (4) document.body (points to \<body\>), document.head (points to \<head\>)

The `document.body` property points to the ` <body> ` node, and the `document.head` property points to the ` <head> ` node.

These two properties **always exist**; if the ` <head> ` or ` <body> ` is omitted in the page's source code, the browser creates them automatically. In addition, these two properties are **writable**; rewriting their values is equivalent to removing all child nodes.



##### (5) document.scrollingElement (returns the scrolling element of the document; \<html\> in standard mode, \<body\> in quirks mode)

The `document.scrollingElement` property **returns the scrolling element of the document**. That is, when the document scrolls as a whole, which element is actually scrolling.

**In standard mode**, this property **returns the document's root element `document.documentElement` (i.e. ` <html> `)**.

**In quirks mode**, it **returns the ` <body> ` element**; if that element does not exist, it returns `null`.

```js
// scroll the page to the top of the browser
document.scrollingElement.scrollTop = 0;
```



##### (6) document.activeElement (returns the focused element in the document, usually a form element; if none, returns <body\> or null)

The `document.activeElement` property **returns the DOM element that currently has focus**. Typically, this property returns a form element such as ` <input> `, ` <textarea> `, or ` <select> `; if there is no currently focused element, it returns the ` <body> ` element or `null`.



##### (7) document.fullscreenElement (returns the DOM element currently displayed in fullscreen)

The `document.fullscreenElement` property **returns the DOM element currently displayed in fullscreen**. If not in fullscreen, this property returns `null`.

```js
if (document.fullscreenElement.nodeName == 'VIDEO') {
  console.log('playing video in fullscreen');
}
```

In the code above, `document.fullscreenElement` lets you know whether the `  <video>  ` element is in fullscreen, allowing you to determine the user's behavior.



#### 2.2 Node collection properties (collections of specific elements inside the document; return HTMLCollection instances)

The following properties **return an `HTMLCollection` instance** representing a collection of specific elements inside the document. These collections are all **live**: any change in the original nodes is immediately reflected in the collection.

##### (1) document.links (returns all <a\> and <area\> elements that have an href attribute)

The `document.links` property **returns all `  <a>  ` and ` <area> ` nodes in the current document that have an `href` attribute set.**

```js
// print all the links in the document
var links = document.links;
for(var i = 0; i < links.length; i++) {
  console.log(links[i]);
}
```



##### (2) document.forms (returns all <form\> form nodes)

The `document.forms` property **returns all ` <form> ` form nodes**.

```js
var selectForm = document.forms[0];
```

The code above gets the first form in the document.

Besides using a positional index, the `id` and `name` attributes can also be used to reference a form.

```js
/* HTML code is as follows
  <form name="foo" id="bar"></form>
*/
document.forms[0] === document.forms.foo // true
document.forms.bar === document.forms.foo // true
```



##### (3) document.images (returns all `  <img>  ` image nodes on the page)

The `document.images` property **returns all `  <img>  ` image nodes on the page**.

```js
var imglist = document.images;

for(var i = 0; i < imglist.length; i++) {
  if (imglist[i].src === 'banner.gif') {
    // ...
  }
}
```

The code above searches for a particular image among all `img` tags.



##### (4) document.embeds, document.plugins (return all ` <embed> ` embed nodes)

The `document.embeds` property and the `document.plugins` property both **return all ` <embed> ` nodes**.



##### (5) document.scripts (returns all `  <script>  ` nodes)

The `document.scripts` property **returns all `  <script>  ` nodes**.

```js
var scripts = document.scripts;
if (scripts.length !== 0 ) {
  console.log('the current page has scripts');
}
```



##### (6) document.styleSheets (returns the collection of stylesheets embedded in or imported by the document)

The `document.styleSheets` property **returns the collection of stylesheets embedded in or imported by the document**; see the chapter "CSS Manipulation" for details.



##### (7) Summary (except for the styleSheets property, all of the others return HTMLCollection instances)

Except for `document.styleSheets`, all of the collection properties above return `HTMLCollection` instances.

```js
document.links instanceof HTMLCollection // true
document.images instanceof HTMLCollection // true
document.forms instanceof HTMLCollection // true
document.embeds instanceof HTMLCollection // true
document.scripts instanceof HTMLCollection // true
```

An `HTMLCollection` instance is an **array-like object**, so these properties all have a `length` property, and **you can use the bracket operator to reference their members**. If a member has an `id` or `name` attribute, you can also use the values of these two attributes to reference that member on the `HTMLCollection` instance.

```js
// HTML code is as follows
// <form name="myForm">
document.myForm === document.forms.myForm // true
```



#### 2.3 Static document information properties

The following properties **return information about the document**.

##### (1) document.documentURI, document.URL (return the URL string)

Both the `document.documentURI` property and the `document.URL` property **return a string representing the URL of the current document**. The difference is that they are inherited from different interfaces: `documentURI` is inherited from the `Document` interface and can be used for all documents, while `URL` is inherited from the `HTMLDocument` interface and can only be used for HTML documents.

```js
document.URL
// http://www.example.com/about

document.documentURI === document.URL
// true
```

If **the document's anchor (`#anchor`) changes, both of these properties change accordingly**.



##### (2) document.domain (returns the domain name)

The `document.domain` property **returns the domain name of the current document, without the protocol and port**. For example, if the page's URL is `http://www.example.com:80/hello.html`, then the `document.domain` property equals `www.example.com`. If the domain name cannot be obtained, this property returns `null`.

`document.domain` is basically a read-only property, with one exception. A page on a subdomain can set `document.domain` to the corresponding parent domain. For example, if the current domain is `a.sub.example.com`, then the `document.domain` property can be set to `sub.example.com` or to `example.com`. After this change, two pages with the same `document.domain` can read each other's resources, such as Cookies that have been set.

In addition, setting `document.domain` causes the port to be changed to `null`. So if you use setting `document.domain` to communicate, both pages must set this value to ensure the ports are the same.



##### (3) document.location (provides URL-related information and operation methods)

The `Location` object is a native object provided by the browser that offers URL-related information and operation methods. You can obtain this object through the `window.location` and `document.location` properties.

For a detailed introduction to this object, see the "Location Object" chapter in the "Browser Model" part.



##### (4) document.lastModified (returns the time the current document was last modified)

The `document.lastModified` property returns a **string** representing the **time the current document was last modified**. The returned value and its date format differ between browsers.

```js
document.lastModified
// "03/07/2018 11:18:27"
```

Note that the value of the `document.lastModified` property is a string, so it cannot be compared directly. You must use the `Date.parse` method to convert it into a `Date` instance before comparing two pages.

```js
var lastVisitedDate = Date.parse('01/01/2018');
if (Date.parse(document.lastModified) > lastVisitedDate) {
  console.log('the page has changed');
}
```

If the page has content generated by JavaScript, the `document.lastModified` property always returns the current time.



##### (5) document.title (returns the title of the current document)

The `document.title` property **returns the title of the current document**. By default, it returns the value of the ` <title> ` node. However, this property is writable; once modified, it returns the modified value.

```js
document.title = 'New Title';
document.title // "New Title"
```



##### (6) document.characterSet (returns the encoding of the current document, such as UTF-8, etc.)

The `document.characterSet` property **returns the encoding of the current document**, such as `UTF-8`, `ISO-8859-1`, etc.



##### (7) document.referrer (where the visitor of the current document came from, e.g. arrived via a search engine)

The `document.referrer` property **returns a string indicating where the visitor of the current document came from**.

```js
document.referrer
// "https://example.com/path"
```

If the source cannot be obtained, or the user typed the URL directly instead of clicking through from another page, `document.referrer` returns an empty string.

The value of `document.referrer` always matches the `Referer` field of the HTTP headers. However, `document.referrer` is spelled with two `r`s, whereas the `Referer` field in the headers has only one `r`.



##### (8) document.dir (returns the text direction; rtl for right-to-left, ltr for left-to-right)

`document.dir` returns **a string indicating the text direction**. It **has only two possible** values: `rtl` means text flows from right to left, as Arabic does; `ltr` means text flows from left to right, which most scripts—including English and Chinese—use.

> Note: The first time this property is called, it may return an empty string "".



##### (9) document.compatMode (returns the mode in which the browser handles the document; backward-compatible mode vs. strict mode)

The `compatMode` property returns the mode in which the browser handles the document; the possible values are `BackCompat` (backward-compatible mode) and `CSS1Compat` (strict mode).

Generally speaking, if the first line of the page code sets an explicit `DOCTYPE` (such as `<!DOCTYPE html>`), the value of `document.compatMode` is `CSS1Compat`.



#### 2.4 Document state properties

##### (1) document.hidden (whether the current page is visible; returns a boolean)

The `document.hidden` property **returns a boolean indicating whether the current page is visible**. If the **window is minimized or the browser switches Tabs, the page becomes invisible, causing `document.hidden` to return `true`.**

This property was introduced by the Page Visibility API and is generally used together with that API.



##### (2) document.visibilityState (returns the document's visibility state: visible, hidden, prerender, unloaded)

`document.visibilityState` **returns the document's visibility state**.

Its value has **four states**.

> - `visible`: the page is visible. Note that the page may be partially visible, i.e. it is not the focused window and is partly obscured by other windows.
> - `hidden`: the page is not visible, possibly because the window is minimized or the browser has switched to another Tab.
> - `prerender`: the page is in the process of being rendered; to the user, the page is not visible.
> - `unloaded`: the page has been unloaded from memory.

**This property can be used to prevent loading certain resources while the page is loading, or to stop some page functionality when the page is not visible.**



##### (3) document.readyState (returns the current state of the document: loading, interactive, complete)

The `document.readyState` property **returns the current state of the document**; there are **three possible** values.

- `loading`: the HTML code is being loaded (parsing not yet complete)
- `interactive`: the external resources are being loaded
- `complete`: loading is complete

The process by which this property changes is as follows.

1. The browser begins parsing the HTML document, and the `document.readyState` property equals `loading`.
2. When the browser encounters a `  <script>  ` element in the HTML document that does not have an `async` or `defer` attribute, it pauses parsing and begins executing the script; at this point the `document.readyState` property still equals `loading`.
3. When the HTML document has finished parsing, the `document.readyState` property becomes `interactive`.
4. The browser waits for external resources such as images, stylesheets, and font files to finish loading; once they have all finished loading, the `document.readyState` property becomes `complete`.

The code below is used to check whether the page has loaded successfully.

```js
// basic check
if (document.readyState === 'complete') {
  // ...
}

// polling check
var interval = setInterval(function() {
  if (document.readyState === 'complete') {
    clearInterval(interval);
    // ...
  }
}, 100);
```

In addition, **each state change triggers a `readystatechange` event.**



#### 2.5 document.cookie (used to manipulate browser Cookies)

The `document.cookie` property is used to manipulate browser Cookies; see the "Cookie" chapter in the "Browser Model" part for details.



#### 2.6 document.designMode (controls whether the current document is editable; on, off)

The `document.designMode` property **controls whether the current document is editable. This property has only two values, `on` and `off`, with a default of `off`**. Once set to `on`, the user can edit the content of the entire document.

The code below turns on the `designMode` property of the document inside an `iframe` element, turning it into a WYSIWYG editor.

```js
// HTML code is as follows
// <iframe id="editor" src="about:blank"></iframe>
var editor = document.getElementById('editor');
editor.contentDocument.designMode = 'on';
```



#### 2.7 document.implementation (returns a `DOMImplementation` object)

The `document.implementation` property **returns a `DOMImplementation` object**. This object has three methods, **mainly used to create new Document objects independent of the current document**.

- `DOMImplementation.createDocument()`: creates an XML document.
- `DOMImplementation.createHTMLDocument()`: creates an HTML document.
- `DOMImplementation.createDocumentType()`: creates a DocumentType object.

The following is an example of creating an HTML document.

```js
var doc = document.implementation.createHTMLDocument('Title');
var p = doc.createElement('p');
p.innerHTML = 'hello world';
doc.body.appendChild(p);

document.replaceChild(
  doc.documentElement,
  document.documentElement
);
```

In the code above, the first step creates a new HTML document `doc`, then replaces `document.documentElement` with its root element `document.documentElement`. This makes all the content of the current document disappear, becoming `hello world`.



### 3. Methods

#### 3.1 document.open(), document.close() (open and close the document's writable state)

The `document.open` method **clears all content of the current document, putting it into a writable state for the `document.write` method to write content into**.

The `document.close` method is used to **close the document opened by `document.open()`**.

```js
document.open();
document.write('hello world');
document.close();
```



#### 3.2 document.write() (write content to the current document), document.writeln() (write content with a line break)

The `document.write` method is **used to write content to the current document**.

During the page's initial rendering phase, as long as the page has not closed writing (i.e. `document.close()` has not been executed), the content written by `document.write` is appended after the existing content.

```js
// the page displays "helloworld"
document.open();
document.write('hello');
document.write('world');
document.close();
```

Note that `document.write` is parsed as HTML code, not escaped.

```js
document.write('<p>hello world</p>');
```

In the code above, `document.write` interprets `  <p>  ` as an HTML tag.

If the page has already finished parsing (after the `DOMContentLoaded` event has occurred), calling the `write` method first calls the `open` method, erasing all content of the current document, and then writes.

```js
document.addEventListener('DOMContentLoaded', function (event) {
  document.write('<p>Hello World!</p>');
});

// equivalent to
document.addEventListener('DOMContentLoaded', function (event) {
  document.open();
  document.write('<p>Hello World!</p>');
  document.close();
});
```

If the `write` method is called during the page's rendering process, the `open` method is not called automatically. (You can think of it as: the `open` method has been called, but the `close` method has not yet been called.)

```js
<html>
<body>
hello
<script type="text/javascript">
  document.write("world")
</script>
</body>
</html>
```

Opening the page above in a browser displays `hello world`.

`document.write` is a method that existed before JavaScript was standardized; there are now perfectly standards-compliant ways to write content to a document (such as assigning to the `innerHTML` property). So **except in certain special cases, you should avoid using the `document.write` method as much as possible.**

The `document.writeln` method is exactly the same as the `write` method, except that it adds a line break at the end of the output content.

```js
document.write(1);
document.write(2);
// 12

document.writeln(1);
document.writeln(2);
// 1
// 2
//
```

Note that the line break added by the `writeln` method is an ASCII line break, which has no effect when rendered as an HTML page—that is, no line break appears on the page. Line breaks on a page must be written explicitly with ` <br> `.



#### 3.3 document.querySelector() (returns the element node matching the selector), document.querySelectorAll() (returns a `NodeList` object containing all nodes matching the given selector)

The `document.querySelector` method accepts a CSS selector as its argument and **returns the element node matching that selector**. If multiple nodes satisfy the match condition, it returns the first matching node. If no matching node is found, it returns `null`.

```js
var el1 = document.querySelector('.myclass');
var el2 = document.querySelector('#myParent > [ng-click]');
```

The `document.querySelectorAll` method is used similarly to `querySelector`, except that it **returns a `NodeList` object containing all nodes matching the given selector**.

```js
elementList = document.querySelectorAll('.myclass');
```

The argument to these two methods can be multiple comma-separated CSS selectors, returning element nodes that match any one of the selectors, which is consistent with CSS selector rules.

```js
var matches = document.querySelectorAll('div.note, div.alert');
```

The code above returns `div` elements whose `class` attribute is `note` or `alert`.

Both methods **support complex CSS selectors**.

```js
// select elements whose data-foo-bar attribute equals someval
document.querySelectorAll('[data-foo-bar="someval"]');

// select all elements in the myForm form that fail validation
document.querySelectorAll('#myForm :invalid');

// select div elements, except those whose class contains ignore
document.querySelectorAll('DIV:not(.ignore)');

// select all three kinds of elements: div, a, script
document.querySelectorAll('DIV, A, SCRIPT');
```

However, they **do not support CSS pseudo-element selectors** (such as `:first-line` and `:first-letter`) or **pseudo-class selectors** (such as `:link` and `:visited`), i.e. they cannot select pseudo-elements or pseudo-classes.

If the argument to the `querySelectorAll` method is the string `*`, it returns all element nodes in the document. In addition, the result returned by `querySelectorAll` is not a live collection and does not reflect changes to element nodes in real time.

Finally, besides being defined on the `document` object, these two methods are also defined on element nodes, i.e. **they can also be called on element nodes**.



#### 3.4 document.getElementsByTagName() (searches by HTML tag name and returns the matching elements)

The `document.getElementsByTagName` method **searches by HTML tag name and returns the matching elements**. Its return value is an array-like object (**an `HTMLCollection` instance**) that reflects changes to the HTML document in real time. If there are no matching elements, it returns an empty collection.

```js
var paras = document.getElementsByTagName('p');
paras instanceof HTMLCollection // true
```

The code above returns all `p` element nodes in the current document.

HTML tag names are case-insensitive, so the `getElementsByTagName` method is also case-insensitive. In addition, the order of the members in the returned result is the order in which they appear in the document.

If you pass `*`, you can return all HTML elements in the document.

```js
var allElements = document.getElementsByTagName('*');
```

**Note** that element nodes themselves also define a `getElementsByTagName` method, which returns the matching elements among that element's descendants. That is, this method can be called not only on the `document` object but also on any element node.

```js
var firstPara = document.getElementsByTagName('p')[0];
var spans = firstPara.getElementsByTagName('span');
```

The code above selects all `span` elements inside the first `p` element.



#### 3.5 document.getElementsByClassName() (returns elements whose class matches)

The `document.getElementsByClassName` method returns an array-like object (**an `HTMLCollection` instance**) that **includes all elements whose `class` name matches the specified condition; changes to the elements are reflected in the returned result in real time**.

```js
var elements = document.getElementsByClassName(names);
```

**Since `class` is a reserved word, JavaScript uniformly uses `className` to represent the CSS `class`.**

The argument can be multiple `class`es separated by spaces.

```js
var elements = document.getElementsByClassName('foo bar');
```

The code above returns **elements that have both the `foo` and `bar` classes**; the order of `foo` and `bar` does not matter.

Note that in normal mode, CSS `class`es are case-sensitive. (In `quirks mode`, they are case-insensitive.)

Like the `getElementsByTagName` method, the `getElementsByClassName` method can be called not only on the `document` object but also on any element node.

```js
// called on a non-document object
var elements = rootElement.getElementsByClassName(names);
```



#### 3.6 document.getElementsByName() (returns elements with the corresponding name value)

The `document.getElementsByName` method is used to select HTML elements that have a `name` attribute (such as ` <form> `, ` <radio> `, `  <img>  `, ` <frame> `, ` <embed> `, ` <object> `, etc.), returning an array-like object (**a `NodeList` instance**), since there may be more than one element with the same `name` attribute.

```js
// the form is <form name="x"></form>
var forms = document.getElementsByName('x');
forms[0].tagName // "FORM"
```



#### 3.7 document.getElementById() (returns the element node matching the specified `id` attribute)

The `document.getElementById` method **returns the element node matching the specified `id` attribute**. If no matching node is found, it returns `null`.

```js
var elem = document.getElementById('para1');
```

Note that the argument to this method is case-sensitive. For example, if a node's `id` attribute is `main`, then `document.getElementById('Main')` returns `null`.

**Both the `document.getElementById` method and the `document.querySelector` method can obtain an element node,** the difference being that the argument to `document.querySelector` uses CSS selector syntax, while the argument to `document.getElementById` is the element's `id` attribute.

```js
document.getElementById('myElement')
document.querySelector('#myElement')
```

In the code above, both methods can select the element whose `id` is `myElement`, but `document.getElementById()` is much more efficient than `document.querySelector()`.

In addition, **this method can only be used on the `document` object, not on other element nodes**.



#### 3.8 document.elementFromPoint(x, y) (returns the topmost element node at the specified coordinates on the page), document.elementsFromPoint(x, y) (returns an array whose members are all the elements at the specified coordinates)

The `document.elementFromPoint` method **returns the topmost element node at the specified position on the page**.

```js
var element = document.elementFromPoint(50, 50);
```

The code above selects the topmost HTML element at the coordinate position `(50, 50)`.

The **two arguments** of the `elementFromPoint` method are, in order, **the x and y coordinates relative to the top-left corner of the current viewport**, in pixels. If the HTML element at that position cannot be returned (such as the scrollbar of a text box), it returns its parent element (such as the text box). If the coordinate values are meaningless (such as negative values or values exceeding the viewport size), it returns `null`.

`document.elementsFromPoint()` **returns an array whose members are all the elements at the specified coordinates (relative to the viewport)**.

```js
var elements = document.elementsFromPoint(x, y);
```

**Note: The coordinates of both methods are relative to the viewport, which means that when the scrollbar scrolls, the selected element may change accordingly.**



#### 3.9 document.createElement() (creates an element node and returns it)

The `document.createElement` method is **used to create an element node and return that node.**

```js
var newDiv = document.createElement('div');
```

The argument to the `createElement` method is the element's tag name, i.e. the `tagName` property of the element node, which is case-insensitive for HTML pages—that is, passing `div` or `DIV` returns the same kind of node. If the argument contains angle brackets (i.e. `<` and `>`), it throws an error.

```js
document.createElement('<div>');
// DOMException: The tag name provided ('<div>') is not a valid name
```

**Note** that the argument to `document.createElement` can be a custom tag name.

```js
document.createElement('foo');
```



#### 3.10 document.createTextNode() (creates a text node and returns it)

The `document.createTextNode` method is **used to create a text node (a `Text` instance) and return that node**. Its **argument is the content of the text node**.

```js
var newDiv = document.createElement('div');
var newContent = document.createTextNode('Hello');
newDiv.appendChild(newContent);
```

The code above creates a new `div` node and a text node, then inserts the text node into the `div` node.

**This method ensures that the returned node is rendered by the browser as text rather than as HTML code. Therefore, it can be used to display user input, avoiding XSS attacks.**

```js
var div = document.createElement('div');
div.appendChild(document.createTextNode('<span>Foo & bar</span>'));
console.log(div.innerHTML)
// &lt;span&gt;Foo &amp; bar&lt;/span&gt;
```

In the code above, the `createTextNode` method escapes the greater-than and less-than signs, ensuring that even if the user's input contains malicious code, it is displayed correctly.

Note that this method does not escape single quotes and double quotes, so it cannot be used to assign values to HTML attributes.

```js
function escapeHtml(str) {
  var div = document.createElement('div');
  div.appendChild(document.createTextNode(str));
  return div.innerHTML;
};

var userWebsite = '" onmouseover="alert(\'derp\')" "';
var profileLink = '<a href="' + escapeHtml(userWebsite) + '">Bob</a>';
var div = document.getElementById('target');
div.innerHTML = profileLink;
// <a href="" onmouseover="alert('derp')" "">Bob</a>
```

In the code above, because the `createTextNode` method does not escape double quotes, the `onmouseover` handler ends up having code injected into it.



#### 3.11 document.createAttribute() (creates an attribute node and returns it)

The `document.createAttribute` method **creates a new attribute node (an `Attr` instance) and returns it**.

```js
var attribute = document.createAttribute(name);
```

The argument `name` of the `document.createAttribute` method is the name of the attribute.

```js
var node = document.getElementById('div1');

var a = document.createAttribute('my_attrib');
a.value = 'newVal';

node.setAttributeNode(a);
// or
node.setAttribute('my_attrib', 'newVal');
```

The code above inserts a `my_attrib` attribute with a value of `newVal` into the `div1` node.



#### 3.12 document.createComment() (creates a comment node and returns it)

The `document.createComment` method **creates a new comment node and returns that node.**

```js
var CommentNode = document.createComment(data);
```

The argument of the `document.createComment` method is a string that becomes the content of the comment node.



#### 3.13 document.createDocumentFragment() (creates an empty document fragment object)

The `document.createDocumentFragment` method **creates an empty document fragment object (a `DocumentFragment` instance)**.

```js
var docFragment = document.createDocumentFragment();
```

A `DocumentFragment` **is a DOM fragment that exists in memory, does not belong to the current document, and is often used to build a relatively complex DOM structure and then insert it into the current document**. The advantage of this is that, because the `DocumentFragment` does not belong to the current document, any change to it does not trigger a re-render of the page, **giving better performance than modifying the current document's DOM directly.**

```js
var docfrag = document.createDocumentFragment();

[1, 2, 3, 4].forEach(function (e) {
  var li = document.createElement('li');
  li.textContent = e;
  docfrag.appendChild(li);
});

var element  = document.getElementById('ul');
element.appendChild(docfrag);
```

In the code above, the document fragment `docfrag` contains four ` <li> ` nodes, and these child nodes are inserted into the current document all at once.



#### 3.14 document.createEvent() (creates an event object, an Event instance)

The `document.createEvent` method **creates an event object (an `Event` instance)** that can be used by the `element.dispatchEvent` method to trigger a specified event.

```js
var event = document.createEvent(type);
```

The **argument of the `document.createEvent` method is the event type, such as `UIEvents`, `MouseEvents`, `MutationEvents`, or `HTMLEvents`.**

```js
var event = document.createEvent('Event');
event.initEvent('build', true, true);
document.addEventListener('build', function (e) {
  console.log(e.type); // "build"
}, false);
document.dispatchEvent(event);
```

The code above creates a new event instance named `build`, then triggers that event.





#### 3.15 document.addEventListener() (add an event listener), document.removeEventListener() (remove an event listener), document.dispatchEvent() (trigger an event)

These three methods are used to handle events on the `document` node. They are all inherited from the `EventTarget` interface; for a detailed introduction see the chapter "The EventTarget Interface".

```js
// add an event listener function
document.addEventListener('click', listener, false);

// remove an event listener function
document.removeEventListener('click', listener, false);

// trigger an event
var event = new Event('click');
document.dispatchEvent(event);
```



#### 3.16 document.hasFocus() (determine whether any element in the current document is activated or focused)

The `document.hasFocus` method **returns a boolean indicating whether any element in the current document is activated or has focus.**

```js
var focused = document.hasFocus();
```

**Note that a document with focus must be active, but the converse does not hold: an active document does not necessarily have focus. For example, when a user clicks a button that opens a new window from the current window, the new window is active but does not have focus.**



#### 3.17 document.adoptNode() (adopt an external node), document.importNode() (import/copy an external node)

The `document.adoptNode` method **removes a node and its child nodes from the document or `DocumentFragment` they originally belonged to, makes them belong to the current `document` object, and returns the new node after insertion**. The `ownerDocument` property of the inserted node object becomes the current `document` object, and its `parentNode` property is `null`.

```js
var node = document.adoptNode(externalNode); // externalNode is the external node
document.appendChild(node);
```

Note that the `document.adoptNode` method only **changes the node's ownership**; it does not insert this node into the new document tree. So you still need to use the `appendChild` method or the `insertBefore` method to insert the new node into the current document tree.

The `document.importNode` method, on the other hand, **copies a node and its child nodes from the document or `DocumentFragment` they originally belong to, making them belong to the current `document` object**. The `ownerDocument` property of the copied node object becomes the current `document` object, and its `parentNode` property is `null`.

```js
var node = document.importNode(externalNode, deep); // externalNode is the external node; deep indicates whether to do a deep copy
```

The **first argument of the `document.importNode` method is the external node**, and the **second argument is a boolean indicating whether to do a deep copy or a shallow copy of the external node**; the default is a shallow copy (false). Although the second argument is optional, it is recommended to always keep this argument and set it to `true`.

Note that the `document.importNode` method only copies the external node; at this point the node's parent node is `null`. The next step is to insert this node into the current document tree.

```js
var iframe = document.getElementsByTagName('iframe')[0];
var oldNode = iframe.contentWindow.document.getElementById('myNode');
var newNode = document.importNode(oldNode, true);
document.getElementById("container").appendChild(newNode);
```

**The code above copies a specified node `myNode` from the `iframe` window and inserts it into the current document.**



#### 3.18 document.createNodeIterator() (returns a child-node iterator, a NodeFilter instance)

The `document.createNodeIterator` method **returns a child-node iterator (a NodeFilter instance)**.

```js
var nodeIterator = document.createNodeIterator(
  document.body, // the root node to traverse
  NodeFilter.SHOW_ELEMENT // the type of nodes to traverse
);
```

The code above returns an iterator over the child nodes of the ` <body> ` element.

The **first argument of the `document.createNodeIterator` method is the root node to traverse**, and the **second argument is the type of nodes to traverse**, here specified as element nodes (`NodeFilter.SHOW_ELEMENT`). The syntax for several main node types is as follows.

- **All nodes: NodeFilter.SHOW_ALL**
- **Element nodes: NodeFilter.SHOW_ELEMENT**
- **Text nodes: NodeFilter.SHOW_TEXT**
- **Comment nodes: NodeFilter.SHOW_COMMENT**

The `document.createNodeIterator` method returns an "iterator" object (a `NodeFilter` instance). Its `nextNode()` and `previousNode()` methods can be used to traverse all child nodes.

**Each call to the `nextNode()` / `previousNode()` method of the NodeFilter instance moves the current pointer to the next / previous node.**

```js
var nodeIterator = document.createNodeIterator(document.body);
var pars = [];
var currentNode;

while (currentNode = nodeIterator.nextNode()) {
    // in js every operator has a return value, which is the result of the operation. here it both assigns to currentNode and automatically calls Boolean() to perform a boolean evaluation.
    // each call to the nextNode() method of the NodeFilter instance returns the next node

  pars.push(currentNode);
}
```

In the code above, the iterator's `nextNode` method is used to read all the child nodes of the root node into an array one by one. The `nextNode` method first returns the node where the iterator's internal pointer is located, then moves the pointer to the next node. After all members have been traversed, it returns `null`. The `previousNode` method, on the other hand, first moves the pointer to the previous node and then returns that node.

```js
var nodeIterator = document.createNodeIterator(
  document.body,
  NodeFilter.SHOW_ELEMENT
);

var currentNode = nodeIterator.nextNode();
var previousNode = nodeIterator.previousNode();

currentNode === previousNode // true
```

In the code above, `currentNode` and `previousNode` both point to the same node.

Note that the **first node returned by the iterator is always the root node**.

```js
pars[0] === document.body // true
```



#### 3.19 document.createTreeWalker() (returns a DOM subtree walker, a TreeWalker instance)

The `document.createTreeWalker` method **returns a DOM subtree walker**. It is basically similar to the `document.createNodeIterator` method, the difference being that it returns a `TreeWalker` instance, whereas the latter returns a `NodeIterator` instance. In addition, its **first node is not the root node**.

The **first argument of the `document.createTreeWalker` method is the root node to traverse**, and the **second argument specifies the type of nodes to traverse** (the same as the second argument of the `document.createNodeIterator` method).

**A second difference from the `document.createNodeIterator` method is that the `TreeWalker` instance has a `currentNode` property.**

```js
var treeWalker = document.createTreeWalker(
  document.body, // the root node to traverse
  NodeFilter.SHOW_ELEMENT // the type of nodes to traverse
);

var nodeList = [];
// each call to nextNode() changes the value of the currentNode property to the next node
while(treeWalker.nextNode()) {
  nodeList.push(treeWalker.currentNode); // the TreeWalker instance has a currentNode property
}
```

The code above traverses all element nodes under the ` <body> ` node and inserts them into the `nodeList` array.



#### 3.20 document.execCommand(), document.queryCommandSupported(), document.queryCommandEnabled()

##### (1) document.execCommand() (executes a command; implements features such as copying text, and is heavily used by rich-text editors)

Concept 1: **When an HTML document is switched to design mode, `document` exposes the `execCommand` method, which allows running commands to manipulate the elements of an [editable content area](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Global_attributes/contenteditable).**

Concept 2: **The execCommand method executes a command on the current document, the current selection, or a given range. It is commonly used when processing HTML data.**



Concept 3: If the `document.designMode` property is set to `on`, then the entire document is editable by the user; if an element's `contenteditable` attribute is set to `true`, then that element is editable. In both of these cases, you can use the `document.execCommand()` method to change the style of content—for example, `document.execCommand('bold')` makes the font bold.

```js
document.execCommand(command, showDefaultUI, input)
```

This method accepts three arguments.

- `command`: a string representing the style to apply.
- `showDefaultUI`: a boolean indicating whether to use the default user interface; it is recommended to always set it to `false`.
- `input`: a string representing the auxiliary content for the style. For example, when creating a hyperlink, this argument is the URL to link to. If the second argument is set to `true`, the browser pops up a prompt box asking the user to enter this argument in the prompt. However, not all browsers support this, so for compatibility you still need to implement your own way of obtaining this argument.

```js
var url = window.prompt('Please enter a URL');

if (url) {
  document.execCommand('createlink', false, url);
}
```

In the code above, the user is first prompted to enter the URL to link to, and then the hyperlink is created manually. Note that the second argument is `false`, indicating that no prompt box should pop up automatically at this point.

The return value of `document.execCommand()` is a boolean. If it is `false`, it means the method could not take effect.

In most cases, this method only takes effect on the selected content. If there are multiple editable content areas, it only takes effect on the element that currently has focus.

There are many kinds of style changes that the `document.execCommand()` method can perform; here are some of them: bold, insertLineBreak, selectAll, createLink, insertOrderedList, subscript, delete, insertUnorderedList, superscript, formatBlock, insertParagraph, undo, forwardDelete, insertText, unlink, insertImage, italic, unselect, insertHTML, redo. ##### (3) document.queryCommandEnabled() (whether a particular execCommand command is currently available)

The `document.queryCommandEnabled()` method returns a boolean indicating whether a particular `document.execCommand()` command is currently available. For example, the `bold` command is only available when there is selected text; if no text is selected, it is unavailable.

```js
// HTML code is
// <input type="button" value="Copy" onclick="doCopy()">

function doCopy(){
  // whether the browser supports the copy command (copy the selected content to the clipboard)
  if (document.queryCommandSupported('copy')) {
    copyText('Hello');
  }else{
    console.log('the browser does not support it');
  }
}

function copyText(text) {
  var input = document.createElement('textarea');
  document.body.appendChild(input);
  input.value = text;
  input.focus();
  input.select();

  // whether there is currently selected text
  if (document.queryCommandEnabled('copy')) {
    var success = document.execCommand('copy');
    input.remove();
    console.log('Copy Ok');
  } else {
    console.log('queryCommandEnabled is false');
  }
}
```

In the code above, it first determines whether the browser supports the `copy` command (which allows the selected content of an editable area to be copied to the clipboard); if it does, it creates a temporary text box, writes the content "Hello" into it, and selects it. Then it determines whether the selection was successful, and if so, copies "Hello" to the clipboard and removes that temporary text box.

## 6. The Element node

**The `Element` node object corresponds to an HTML element of the web page. Every HTML element is transformed into an `Element` node object (hereafter referred to as an element node) in the DOM tree.**

The `nodeType` property of an element node is always `1`.

```js
var p = document.querySelector('p');
p.nodeName // "P"
p.nodeType // 1  node type; 1 means an element node
```

The `Element` object inherits the `Node` interface, so **the properties and methods of `Node` all exist on the `Element` object**. In addition, **the element nodes corresponding to different HTML elements are not the same; the browser uses different constructors to generate different element nodes**. For example, the node object of an `  <a>  ` element is generated by the `HTMLAnchorElement` constructor, and the node object of a ` <button> ` element is generated by the `HTMLButtonElement` constructor. Therefore, an element node is not a single kind of object but a group of objects; besides inheriting the properties and methods of `Element`, these objects also have the properties and methods of their respective constructors.



**An element node has the properties and methods of its respective constructor, inherits the properties and methods of `Element`, and at the same time inherits the properties and methods of `Node`.**



### 1. Instance properties

#### 1.1 Properties related to element characteristics

##### (1) Element.id (returns the element's id attribute; readable and writable)

The `Element.id` property **returns the `id` attribute of the specified element; this property is readable and writable.**

```js
// HTML code is <p id="foo">
var p = document.querySelector('p');
p.id // "foo"
```

Note that the value of the `id` attribute is **case-sensitive**, i.e. the browser correctly distinguishes the `id` attributes of the two elements `  <p id="foo">  ` and `  <p id="FOO">  `, but it is best not to name them this way.



##### (2) Element.tagName (returns the element's uppercase tag name)

The `Element.tagName` property **returns the uppercase tag name of the specified element, which is equal to the value of the `nodeName` property.**

```js
// HTML code is
// <span id="myspan">Hello</span>
var span = document.getElementById('myspan');
span.id // "myspan"
span.tagName // "SPAN"
```



##### (3) Element.dir (the element's text direction; readable and writable; ltr, rtl)

The `Element.dir` property is **used to read and write the text direction of the current element**, which may be left-to-right (`"ltr"`) or right-to-left (`"rtl"`).



##### (4) Element.accessKey (the shortcut key assigned to the current element; readable and writable)

The `Element.accessKey` property is **used to read and write the shortcut key assigned to the current element**.

```js
// HTML code is as follows
// <button accesskey="h" id="btn">Click</button>
var btn = document.getElementById('btn');
btn.accessKey // "h"
```

In the code above, the shortcut key of the `btn` element is `h`, and pressing `Alt + h` transfers focus to it.



##### (5) Element.draggable (whether the current element is draggable; a boolean; readable and writable)

The `Element.draggable` property **returns a boolean indicating whether the current element is draggable. This property is readable and writable.**

```html
<p draggable="true" ondragstart="drag(this)">This is a movable paragraph.</p>
<script>
    function drag(el) {
      console.log(el.draggable) // true
    }
</script>
```



##### (6) Element.lang (returns the language setting of the current element; readable and writable)

The `Element.lang` property **returns the language setting of the current element. This property is readable and writable.**

```js
// HTML code is as follows
// <html lang="en">
document.documentElement.lang // "en"
```



##### (7) Element.tabIndex (the order of the current element during Tab-key traversal; an integer; readable and writable)

The `Element.tabIndex` property **returns an integer indicating the order of the current element during Tab-key traversal. This property is readable and writable.**

If the `tabIndex` property value **is negative (usually `-1`), then the Tab key does not traverse to that element**. If it is a positive integer, traversal proceeds in order from smallest to largest. If two elements have the same positive integer `tabIndex` value, they are traversed in the order they appear. After traversing all elements with a positive integer `tabIndex`, it then traverses all elements whose `tabIndex` equals `0`, or whose attribute value is invalid, or which have no `tabIndex` attribute, in the order they appear in the page.



##### (8) Element.title (the current element's HTML `title` attribute; readable and writable)

The `Element.title` property is **used to read and write the current element's HTML `title` attribute. This property is usually used to specify the text tooltip that pops up on mouse hover.**



#### 1.2 Properties related to element state

##### (1) Element.hidden (the current element's hidden attribute, which controls visibility; a boolean; readable and writable)

The `Element.hidden` property **returns a boolean representing the current element's `hidden` attribute, which is used to control whether the current element is visible. This property is readable and writable.**

```js
var btn = document.getElementById('btn');
var mydiv = document.getElementById('mydiv');

btn.addEventListener('click', function () {
  mydiv.hidden = !mydiv.hidden;
}, false);
```

**Note that this property is independent of CSS settings.** A CSS setting for this element's visibility is not reflected by `Element.hidden`. That is, this property cannot be used to determine the actual visibility of the current element.

**CSS settings take precedence over `Element.hidden`.** If CSS specifies that the element is invisible (`display: none`) or visible (`display: hidden`), then `Element.hidden` cannot change the element's actual visibility. In other words, **this property is only effective when CSS has not explicitly set the visibility of the current element.**



##### (2) Element.contentEditable (the editability of the element's content; a string; readable and writable),

##### 	Element.isContentEditable (whether the element's content is editable; a boolean; read-only)

An HTML element can have a `contentEditable` attribute set to make the element's content editable.

```html
<div contenteditable>123</div>
```

In the code above, the `  <div>  ` element has a `contenteditable` attribute, so the user can edit the content of this block on the web page.

The `Element.contentEditable` property returns a string indicating whether the `contenteditable` attribute is set; there are three possible values. This property is writable.

- `"true"`: the element's content is editable
- `"false"`: the element's content is not editable
- `"inherit"`: whether the element is editable is inherited from the parent element's setting

The `Element.isContentEditable` property returns a boolean, also indicating whether the `contenteditable` attribute is set. This property is read-only.



#### 1.3 Element.attributes (an array-like object whose members are all the HTML attribute nodes of the element)

The `Element.attributes` property **returns an array-like object whose members are all the attribute nodes of the current element node**; see the chapter "Manipulating Attributes" for details.

```js
var p = document.querySelector('p');
var attrs = p.attributes;

for (var i = attrs.length - 1; i >= 0; i--) {
  console.log(attrs[i].name + '->' + attrs[i].value);
}
```

The code above traverses all the attributes of the `p` element.



#### 1.4 Element.className (the element's class), Element.classList (an array-like object whose members are the individual classes, with its own methods)

The `className` property is **used to read and write the `class` attribute of the current element node.** Its value is a string in which each `class` is separated by a space.

The `classList` property **returns an array-like object in which each `class` of the current element node is a member.**

```js
// HTML code <div class="one two three" id="myDiv"></div>
var div = document.getElementById('myDiv');

div.className
// "one two three"

div.classList
// {
//   0: "one"
//   1: "two"
//   2: "three"
//   length: 3
// }
```

In the code above, the `className` property returns a space-separated string, while the `classList` property points to an array-like object whose `length` property (read-only) returns the number of `class`es on the current element.

###### The `classList` object has the following methods.

- **`add()`: add a class.**
- **`remove()`: remove a class.**
- **`contains()`: check whether the current element contains a particular class.**
- **`toggle()`: add or remove a particular class on the current element.**
- **`item()`: return the class at the specified index position.**
- **`toString()`: convert the list of classes into a string.**

```js
var div = document.getElementById('myDiv');

div.classList.add('myCssClass');
div.classList.add('foo', 'bar');
div.classList.remove('myCssClass');
div.classList.toggle('myCssClass'); // if myCssClass does not exist, add it; otherwise remove it
div.classList.contains('myCssClass'); // returns true or false
div.classList.item(0); // returns the first class
div.classList.toString();
```

Below is a comparison of how `className` and `classList` are written when adding and removing a class.

```js
var foo = document.getElementById('foo');

// add a class
foo.className += 'bold';
foo.classList.add('bold');

// remove a class
foo.classList.remove('bold');
foo.className = foo.className.replace(/^bold$/, '');
```

**The `toggle` method can accept a boolean as its second argument.** If it is `true`, the class is added; if it is `false`, the class is removed.

```js
el.classList.toggle('abc', boolValue);

// equivalent to
if (boolValue) {
  el.classList.add('abc');
} else {
  el.classList.remove('abc');
}
```



#### 1.5 Element.dataset (returns an object for reading and writing the element's data- attributes)

A web page element can have custom `data-` attributes used to add data.

```js
<div data-timestamp="1522907809292"></div>
```

In the code above, the `  <div>  ` element has a custom `data-timestamp` attribute used to add a timestamp to the element.

The `Element.dataset` property **returns an object from which you can read and write `data-` attributes.**

```js
// <article
//   id="foo"
//   data-columns="3"
//   data-index-number="12314"
//   data-parent="cars">
//   ...
// </article>
var article = document.getElementById('foo');
article.dataset.columns // "3"
article.dataset.indexNumber // "12314"
article.dataset.parent // "cars"
```

Note that all of the properties on `dataset` return strings.

In HTML code, the attribute name of a `data-` attribute can only contain English letters, digits, hyphens (`-`), dots (`.`), colons (`:`), and underscores (`_`). The **rules** for converting them into the corresponding JavaScript `dataset` property name are as follows.

- **The leading `data-` is omitted.**
- **If a hyphen is followed by an English letter, the hyphen is removed and that letter becomes uppercase.**
- **Other characters are unchanged.**

Therefore, `data-abc-def` corresponds to `dataset.abcDef`, and `data-abc-1` corresponds to `dataset["abc-1"]`.

Besides using `dataset` to read and write `data-` attributes, you can **also use `Element.getAttribute()` and `Element.setAttribute()` to read and write these attributes via their full attribute names.**

```js
var mydiv = document.getElementById('mydiv');

mydiv.dataset.foo = 'bar';
mydiv.getAttribute('data-foo') // "bar"
```



#### 1.6 Element.innerHTML (returns a string of all the HTML code contained in the element; readable and writable)

The `Element.innerHTML` property **returns a string equivalent to all the HTML code contained in the element**. This property is **readable and writable** and is commonly used to set the content of a node. It can rewrite the content of all element nodes, including the ` <HTML> ` and ` <body> ` elements.

Setting the `innerHTML` property to empty is equivalent to deleting all the nodes it contains.

```js
el.innerHTML = '';
```

The code above is equivalent to turning the `el` node into an empty node; all the nodes `el` originally contained are deleted.

Note that when reading the property value, if a text node contains `&`, the less-than sign (`<`), or the greater-than sign (`>`), the `innerHTML` property converts them into entity form `&amp;`, `&lt;`, `&gt;`. If you want to get the original text, it is recommended to use the `element.textContent` property.

```js
// HTML code is <p id="para"> 5 > 3 < 6 &</p>
document.getElementById('para').innerHTML
// 5 &gt; 3 &lt; &amp;
```

When writing, if the inserted **text contains HTML tags, they are parsed into node objects and inserted into the DOM**. Note that if the text contains a `  <script>  ` tag, although a `script` node can be created, the inserted code is not executed.

```js
var name = "<script>alert('haha')</script>"; // inserting a string containing js code; the js does not execute after insertion
el.innerHTML = name;
```

The code above inserts the script as content, but the script does not execute. However, `innerHTML` still has security risks.

```js
var name = "<img src=x onerror=alert(1)>";
el.innerHTML = name;
```

In the code above, the `alert` method does execute. **Therefore, for security reasons, if you are inserting text, it is best to use the `textContent` property instead of `innerHTML`.**



#### 1.7 Element.outerHTML (returns a string containing the HTML code of the element itself and all its child elements; readable and writable)

The `Element.outerHTML` property **returns a string representing all the HTML code of the current element node, including the element itself and all its child elements.**

```js
// HTML code is as follows
// <div id="d"><p>Hello</p></div>
var d = document.getElementById('d');
d.outerHTML
// '<div id="d"><p>Hello</p></div>'
```

The `outerHTML` property is **readable and writable**, and **assigning to it is equivalent to replacing the current element.**

```js
// HTML code is as follows
// <div id="container"><div id="d">Hello</div></div>
var container = document.getElementById('container');
var d = document.getElementById('d');
container.firstChild.nodeName // "DIV"
d.nodeName // "DIV"

d.outerHTML = '<p>Hello</p>'; // the div is replaced by a p tag
container.firstChild.nodeName // "P"
d.nodeName // "DIV"  but the variable d still points to the original div element, which still exists in memory
```

In the code above, the variable `d` represents a child node. After its `outerHTML` property is reassigned, the inner `div` element no longer exists and has been replaced by a `p` element. However, **the variable `d` still points to the original `div` element, which means the replaced `DIV` element still exists in memory.**

Note that if a **node has no parent node, setting the `outerHTML` property throws an error**.

```js
var div = document.createElement('div');
div.outerHTML = '<p>test</p>';
// DOMException: This element has no parent node.
```

In the code above, the `div` element has no parent node, so setting the `outerHTML` property throws an error.



#### 1.8 Element.clientHeight (element height), Element.clientWidth (element width)

The `Element.clientHeight` property returns an integer representing **the CSS height of the element node** (in pixels). **It only works for block-level elements; for inline elements it returns `0`**. If a block-level element has no CSS height set, it returns the actual height.

Besides the element's own height, it also **includes the `padding` portion**, but not `border` or `margin`. If there is a horizontal scrollbar, the height of the horizontal scrollbar must also be subtracted. Note that this value is always an integer; if it is a decimal, it is rounded.

The `Element.clientWidth` property returns **the CSS width of the element node, and likewise only works for block-level elements**; it also includes only the element's own width and `padding`, and if there is a vertical scrollbar, the width of the vertical scrollbar must be subtracted.

The `clientHeight` property of `document.documentElement` returns the height of the current viewport (i.e. the height of the browser window), which equals the `window.innerHeight` property minus the height of the horizontal scrollbar (if any). The height of `document.body` is the actual height of the page. Generally, `document.body.clientHeight` is greater than `document.documentElement.clientHeight`.

```js
// viewport height, not including the invisible part
document.documentElement.clientHeight

// total page height
document.body.clientHeight
```



#### 1.9 Element.clientLeft (left border width), Element.clientTop (top border width)

The `Element.clientLeft` property equals the width of the element node's **left border** (in pixels), not including the left-side `padding` and `margin`. If no left border is set, or the element is an inline element (`display: inline`), this property returns `0`. This property always returns an integer; if it is a decimal, it is rounded.

The `Element.clientTop` property equals the width of the page element's **top border** (in pixels); its other characteristics are the same as `clientLeft`.



#### 1.10 Element.scrollHeight (the element's total height), Element.scrollWidth (the element's total width)

The `Element.scrollHeight` property returns an integer (decimals are rounded) representing the current **element's total height** (in pixels), **including the overflowing, currently invisible part. It includes `padding`**, but not `border`, `margin`, or the height of the horizontal scrollbar (if there is one); it also includes the height of pseudo-elements (`::before` or `::after`).

The `Element.scrollWidth` property represents the current **element's total width** (in pixels), and is otherwise similar to the `scrollHeight` property. Both of these properties are read-only.

**The total height of the entire page** can be read from `document.documentElement` or `document.body`.

```js
// returns the total height of the page
document.documentElement.scrollHeight
document.body.scrollHeight
```

Note that if the content of an element node overflows, the `scrollHeight` property still returns the element's total height even if the overflowing content is hidden.

```js
// HTML code is as follows
// <div id="myDiv" style="height: 200px; overflow: hidden;">...<div>
document.getElementById('myDiv').scrollHeight // 356
```

In the code above, even though the CSS height of the `myDiv` element is only 200 pixels and the overflowing part is invisible, `scrollHeight` still returns the element's original height.



#### 1.11 Element.scrollLeft (the number of pixels the element's horizontal scrollbar has scrolled to the right), Element.scrollTop (the number of pixels the element's vertical scrollbar has scrolled down)

The `Element.scrollLeft` property represents **the number of pixels the current element's horizontal scrollbar has scrolled to the right**, and the `Element.scrollTop` property represents **the number of pixels the current element's vertical scrollbar has scrolled down**. For page elements that have no scrollbar, these two properties are always equal to 0.

To check the horizontal and vertical scroll distances of the entire page, read them from the `document.documentElement` element.

```js
document.documentElement.scrollLeft
document.documentElement.scrollTop
```

Both of these properties are **readable and writable**; setting their value causes the browser to automatically scroll the current element to the corresponding position.



#### 1.12 Element.offsetParent (returns the nearest ancestor element of the current element whose CSS `position` property is not `static`)

The `Element.offsetParent` property **returns the nearest ancestor element of the current element whose CSS `position` property is not `static`**.

> `position: static;` is the default, i.e. the style with no positioning.

```js
<div style="position: absolute;">
  <p>
    <span>Hello</span>
  </p>
</div>
```

In the code above, the `offsetParent` property of the `span` element is the `div` element.

**This property is mainly used to determine the reference base for calculating a child element's positional offset**; `Element.offsetTop` and `Element.offsetLeft` are computed relative to the `offsetParent` element.

If the element is invisible (its `display` property is `none`), or its position is fixed (its `position` property is `fixed`), then the `offsetParent` property returns `null`.

```js
<div style="position: absolute;">
  <p>
    <span style="display: none;">Hello</span>
  </p>
</div>
```

In the code above, the `offsetParent` property of the `span` element is `null`.

If the `position` property of all the ancestor nodes of an element is `static`, then the `Element.offsetParent` property points to the ` <body> ` element.



#### 1.13 Element.offsetHeight (the element's CSS vertical height), Element.offsetWidth (the element's CSS horizontal width)

The `Element.offsetHeight` property returns an integer representing **the element's CSS vertical height** (in pixels), **including the element's own height, padding, and border, as well as the height of the horizontal scrollbar (**if a scrollbar exists).

The `Element.offsetWidth` property represents **the element's CSS horizontal width** (in pixels), and is otherwise the same as `Element.offsetHeight`.

Both of these properties are **read-only properties** that differ from `Element.clientHeight` and `Element.clientWidth` only by the addition of the border's height or width. If the element's CSS is set to invisible (such as `display: none;`), they return `0`.



#### 1.14 Element.offsetLeft (horizontal offset), Element.offsetTop (vertical offset)

`Element.offsetLeft` returns the current **element's top-left corner horizontal offset relative to the `Element.offsetParent` node**, and `Element.offsetTop` returns the **vertical offset**, in pixels. Usually, these two values refer to the offset relative to the parent node.

The code below can calculate the coordinates of an element's top-left corner relative to the entire page.

```js
function getElementPosition(e) {
  var x = 0;
  var y = 0;
  while (e !== null)  {
    x += e.offsetLeft;
    y += e.offsetTop;
    e = e.offsetParent;
  }
  return {x: x, y: y};
}
```



#### 1.15 Element.style (returns a CSSStyleDeclaration instance, used to manipulate CSS)

Every element node has a `style` used to read and write **the element's inline style information**; for a detailed introduction, see the chapter "CSS Manipulation".



#### 1.16 Element.children (an array-like object of all child elements), Element.childElementCount (the number of child elements)

The `Element.children` property **returns an array-like object (an `HTMLCollection` instance) that includes all the child elements of the current element node**. If the current element has no child elements, the returned object contains zero members.

```js
if (para.children.length) {
  var children = para.children;
    for (var i = 0; i < children.length; i++) {
      // ...
    }
}
```

The code above traverses all the child elements of the `para` element.

The difference between this property and the `Node.childNodes` property is that it includes only element-type child nodes, not other types of child nodes.

The `Element.childElementCount` property returns **the number of child element nodes** contained in the current element node, which is the same value as `Element.children.length`.



#### 1.17 Element.firstElementChild (the first child element), Element.lastElementChild (the last child element)

The `Element.firstElementChild` property returns the first element child node of the current element, and `Element.lastElementChild` returns the last element child node.

If there are no element child nodes, both of these properties return `null`.



#### 1.18 Element.nextElementSibling (the next sibling element node), Element.previousElementSibling (the previous sibling element node)

The `Element.nextElementSibling` property returns the **next sibling element node** of the current element node, or `null` if there is none.

```js
// HTML code is as follows
// <div id="div-01">Here is div-01</div>
// <div id="div-02">Here is div-02</div>
var el = document.getElementById('div-01');
el.nextElementSibling
// <div id="div-02">Here is div-02</div>
```

The `Element.previousElementSibling` property returns the **previous sibling element node** of the current element node, or `null` if there is none.



### 2. Instance methods

#### 2.1 Methods related to tag attributes

Element nodes provide six methods for manipulating attributes.

##### (1) `getAttribute()`: read the value of an attribute

##### (2) `getAttributeNames()`: return all the attribute names of the current element

##### (3) `setAttribute()`: write an attribute value

##### (4) `hasAttribute()`: whether an attribute exists

##### (5) `hasAttributes()`: whether the current element has attributes

##### (6) `removeAttribute()`: delete an attribute

For an introduction to these methods, see the chapter "**Manipulating Attributes**".



#### 2.2 Element.querySelector() (returns the first matching child element)

The `Element.querySelector` method **accepts a CSS selector as its argument and returns the first matching child element of the parent element**. If no matching child element is found, it returns `null`.

```js
var content = document.getElementById('content');
var el = content.querySelector('p');
```

The code above returns the first `p` element of the `content` node.

The `Element.querySelector` method **can accept any complex CSS selector**.

```js
document.body.querySelector("style[type='text/css'], style:not([type])");
```

Note that this method cannot select pseudo-elements.

It can accept multiple selectors separated by commas.

```js
element.querySelector('div, p')
```

The code above returns the first `div` or `p` child element of `element`.

It is worth **noting** that when the browser executes the `querySelector` method, it first searches the given CSS selector globally and then filters out which of the results are child elements of the current element. As a result, there can be some counterintuitive outcomes. Below is a snippet of HTML code.

```html
<div>
<p>111</p>
<blockquote id="outer">
  <p>Hello</p>
  <div id="inner">
    <p>World</p>
  </div>
</blockquote>
</div>
```

Then querying as below actually returns the first `p` element, not the second.

```js
var outer = document.getElementById('outer');
outer.querySelector('div p')
// <p>Hello</p>
// the counterintuitive result occurs because it searches globally and finds the div outside of outer, then selects the first p among outer's child elements

outer.querySelector('div').querySelector('p')
// <p>World</p> this way you can get the inner p element
```



#### 2.3 Element.querySelectorAll() (returns a NodeList instance containing all matching child elements)

The `Element.querySelectorAll` method accepts a CSS selector as its argument and **returns a `NodeList` instance containing all matching child elements**.

> The returned NodeList instance is an **array-like object**

```js
var el = document.querySelector('#test');
var matches = el.querySelectorAll('div.highlighted > p');
```

The execution mechanism of this method is the same as the `querySelector` method: it also **searches globally first, then filters out the current element's child elements**. Therefore, the selector actually applies to the entire document.

It can also accept multiple CSS selectors separated by commas. If the selector contains a pseudo-element selector, it always returns an empty `NodeList` instance.



#### 2.4 Element.getElementsByClassName() (get all child elements by class)

The `Element.getElementsByClassName` method **returns an `HTMLCollection` instance whose members are all the child element nodes of the current element node that have the specified class**. This method is used similarly to the `document.getElementsByClassName` method, except that the search scope is not the entire document but the current element node.

```js
element.getElementsByClassName('red test');
```

Note that the argument to this method is case-sensitive.

Since an `HTMLCollection` instance is a **live collection**, any change to the `document` object is immediately reflected in the instance, so the code below does not work as expected.

```js
// HTML code is as follows
// <div id="example">
//   <p class="foo"></p>
//   <p class="foo"></p>
// </div>
var element = document.getElementById('example');
var matches = element.getElementsByClassName('foo');

for (var i = 0; i< matches.length; i++) {
  matches[i].classList.remove('foo'); // once foo is removed, the matches collection changes immediately
  matches.item(i).classList.add('bar');
}
// after execution, the HTML code is as follows
// <div id="example">
//   <p></p>
//   <p class="foo bar"></p>
// </div>
```

In the code above, the first member of the `matches` collection, **once its `foo` class is removed, immediately disappears from `matches`**, leading to the result above.



#### 2.5 Element.getElementsByTagName() (get child elements by tag name)

The `Element.getElementsByTagName` method **returns an `HTMLCollection` instance whose members are all the child element nodes of the current node that match the specified tag name**. This method is used similarly to the `document.getElementsByClassName` method, except that the search scope is not the entire document but the current element node.

```js
var table = document.getElementById('forecast-table');
var cells = table.getElementsByTagName('td');
```

Note that the argument to this method is case-insensitive.



#### 2.6 Element.closest() (accepts a CSS selector argument and returns the nearest ancestor node)

The `Element.closest` method **accepts a CSS selector as its argument and returns the nearest ancestor node of the current node (including the current node itself) that matches the selector**. If no node matches the CSS selector, it returns `null`.

```js
// HTML code is as follows
// <article>
//   <div id="div-01">Here is div-01
//     <div id="div-02">Here is div-02
//       <div id="div-03">Here is div-03</div>
//     </div>
//   </div>
// </article>

var div03 = document.getElementById('div-03');

// the nearest ancestor node of div-03
div03.closest("#div-02") // div-02

div03.closest("div") // div-03 closest includes the current node itself
div03.closest("div div") // div-03
div03.closest("div>div") // div-03

div03.closest("article > div") //div-01
div03.closest(":not(div)") // article
```

In the code above, since the `closest` method also takes the current node into account, the second `closest` method returns `div-03`.



#### 2.7 Element.matches() (whether the current element matches the given CSS selector)

The `Element.matches` method **returns a boolean indicating whether the current element matches the given CSS selector.**

```js
if (el.matches('.someClass')) {
  console.log('Match!');
}
```



#### 2.8 Event-related methods

The following three methods are related to events on `Element` nodes. These methods are all inherited from the `EventTarget` interface; **see the relevant chapter for details**.

##### (1) `Element.addEventListener()`: add an event callback function

##### (2) `Element.removeEventListener()`: remove an event listener function

##### (3) `Element.dispatchEvent()`: trigger an event

```js
element.addEventListener('click', listener, false);
element.removeEventListener('click', listener, false);

var event = new Event('click');
element.dispatchEvent(event);
```



#### 2.9 Element.scrollIntoView() (scroll the element into the browser's visible area)

The `Element.scrollIntoView` method **scrolls the current element into the visible area of the browser window**, similar to the effect of setting `window.location.hash`.

```js
el.scrollIntoView(); // equivalent to el.scrollIntoView(true)
el.scrollIntoView(false);
```

This method can accept a boolean as its argument. If it is `true`, the top of the element aligns with the top of the visible part of the current area (provided the current area is scrollable); if it is `false`, the bottom of the element aligns with the bottom of the visible part of the current area (provided the current area is scrollable). If this argument is not provided, it defaults to `true`.



#### 2.10 Element.getBoundingClientRect() (returns an object with the element's size and position information)

The `Element.getBoundingClientRect` method **returns an object providing the size, position, and other information of the current element node—basically all the information of the CSS box model**.

```js
var rect = obj.getBoundingClientRect();
```

In the code above, the `rect` object returned by the `getBoundingClientRect` method has the following properties (all **read-only**).

- `x`: the x coordinate of the element's top-left corner relative to the viewport
- `y`: the y coordinate of the element's top-left corner relative to the viewport
- `height`: the element's height
- `width`: the element's width
- `left`: the x coordinate of the element's top-left corner relative to the viewport, equal to the `x` property
- `right`: the x coordinate of the element's right edge relative to the viewport (equals `x + width`)
- `top`: the y coordinate of the element's top relative to the viewport, equal to the `y` property
- `bottom`: the y coordinate of the element's bottom relative to the viewport (equals `y + height`)

Since the element's **position relative to the viewport changes as the page scrolls**, the four properties representing position are not fixed. If you want to get the absolute position, you can add `window.scrollX` to the `left` property and `window.scrollY` to the `top` property.

Note that all properties of the `getBoundingClientRect` method count the border (the `border` property) as part of the element. That is, they are all calculated from the points on the outer edge of the border. Therefore, **`width` and `height` include the element itself + `padding` + `border`.**

In addition, all of the properties above are inherited from the prototype, so `Object.keys` returns an empty array—this is also something to note.

```js
var rect = document.body.getBoundingClientRect();
Object.keys(rect) // []
```

In the code above, the `rect` object has no own properties, and the `Object.keys` method only returns an object's own properties, so it returns an empty array.



#### 2.11 Element.getClientRects() (all the rectangles the element forms on the page; an array-like object)

The `Element.getClientRects` method **returns an array-like object containing all the rectangles the current element forms on the page** (which is why `Rect` in the method name is plural). Each rectangle has six properties—`bottom`, `height`, `left`, `right`, `top`, and `width`—representing their four coordinates relative to the viewport, as well as their own height and width.

For box elements (such as `  <div>  ` and `  <p>  `), the object returned by this method has only one member, the element itself. For inline elements (such as ` <span> `, `  <a>  `, ` <em> `), the number of members in the object returned by this method depends on how many lines the element occupies on the page. This is the main difference between it and the `Element.getBoundingClientRect()` method, which always returns a single rectangle for inline elements.

```js
<span id="inline">Hello World Hello World Hello World</span>
```

The code above is an inline element ` <span> `; if it occupies three lines on the page, the object returned by the `getClientRects` method has three members, and if it occupies one line on the page, the object returned by the `getClientRects` method has only one member.

```js
var el = document.getElementById('inline');
el.getClientRects().length // 3
el.getClientRects()[0].left // 8
el.getClientRects()[0].right // 113.908203125
el.getClientRects()[0].bottom // 31.200000762939453
el.getClientRects()[0].height // 23.200000762939453
el.getClientRects()[0].width // 105.908203125
```

This method is mainly used to determine whether an inline element wraps, and the positional offset of each line of an inline element.

Note that if an inline element contains line breaks, this method takes the line breaks into account.

```js
<span id="inline">
  Hello World
  Hello World
  Hello World
</span>
```

In the code above, there are three line breaks inside the ` <span> ` node; even though HTML ignores line breaks and displays them as a single line, the `getClientRects()` method still returns three members. If the line width is set very narrow and the ` <span> ` element above is displayed as 6 lines, then it returns six members.



#### 2.12 Element.insertAdjacentElement() (insert a new node at a specified position relative to the element)

The `Element.insertAdjacentElement` method **inserts a new node at a specified position relative to the current element**. This method returns the inserted node, or `null` if the insertion fails.

```js
element.insertAdjacentElement(position, element);
```

The `Element.insertAdjacentElement` method accepts two arguments in total: the first is a string representing the insertion position, and the second is the node to be inserted. The first argument can only take the following values.

- `beforebegin`: before the current element
- `afterbegin`: before the first child node inside the current element
- `beforeend`: after the last child node inside the current element
- `afterend`: after the current element

Note that the two values `beforebegin` and `afterend` only take effect when the current node has a parent node. If the current node was created by a script and has no parent node, the insertion fails.

```js
var p1 = document.createElement('p')
var p2 = document.createElement('p')
p1.insertAdjacentElement('afterend', p2) // null
```

In the code above, `p1` has no parent node, so inserting `p2` after it fails.

If the inserted node is a node that already exists in a document, it is removed from its original position and placed at the new position.



#### 2.13 Element.insertAdjacentHTML() (insert an HTML string at a specified position), Element.insertAdjacentText() (insert text at a specified position)

The `Element.insertAdjacentHTML` method is **used to parse an HTML string into a DOM structure and insert it at a specified position relative to the current node.**

```js
element.insertAdjacentHTML(position, text);
```

This method accepts two arguments: the first is a string representing the specified position, and the second is the HTML string to be parsed. The first argument can only be set to one of the following four values.

- `beforebegin`: before the current element
- `afterbegin`: before the first child node inside the current element
- `beforeend`: after the last child node inside the current element
- `afterend`: after the current element

```js
// HTML code: <div id="one">one</div>
var d1 = document.getElementById('one');
d1.insertAdjacentHTML('afterend', '<div id="two">two</div>');
// HTML code after execution:
// <div id="one">one</div><div id="two">two</div>
```

This method only inserts nodes into the existing DOM structure, which makes it much faster than the `innerHTML` method.

Note that this method does not escape the HTML string, which means it cannot be used to insert user-input content, otherwise there is a security risk.

The `Element.insertAdjacentText` method inserts a text node at a specified position relative to the current node; its usage is exactly the same as the `Element.insertAdjacentHTML` method.

```js
// HTML code: <div id="one">one</div>
var d1 = document.getElementById('one');
d1.insertAdjacentText('afterend', 'two');
// HTML code after execution:
// <div id="one">one</div>two
```



#### 2.14 Element.remove() (remove the element)

The `Element.remove` method is **inherited from the ChildNode interface and is used to remove the current element node from its parent node.**

```js
var el = document.getElementById('mydiv');
el.remove();
```

The code above removes the `el` node from the DOM tree.



#### 2.15 Element.focus() (transfer page focus to the element), Element.blur() (remove focus)

The `Element.focus` method is **used to transfer the focus of the current page to a specified element**.

```js
document.getElementById('my-span').focus();
```

This method can accept an object as its argument. The `preventScroll` property of the argument object is a boolean specifying **whether to keep the current element at its original position** rather than scrolling it into the visible area.

```js
function getFocus() {
  document.getElementById('btn').focus({preventScroll:false});
}
```

The code above makes the `btn` element gain focus and **scroll into the visible area**.

Finally, the `document.activeElement` property gives you the element that currently has focus.

The `Element.blur` method is **used to remove focus from the current element**.



#### 2.16 Element.click() (simulate a click event)

The `Element.click` method is used to **simulate a mouse click on the current element, which is equivalent to triggering a `click` event.**



## 7. Manipulating element attributes

An HTML element consists of a tag name and a number of key-value pairs; these key-value pairs are called "attributes".

```html
<a id="test" href="http://www.example.com">
  link
</a>
```

In the code above, the `a` element has two attributes: the `id` attribute and the `href` attribute.

An attribute is itself an object (an `Attr` object), but in practice this object is rarely used. Generally, attributes are manipulated through the element node object (the `HTMlElement` object). This chapter explains how to manipulate these attributes.



### 1. The Element.attributes property (returns an array-like live object containing all the attributes of the tag)

An element object has an `attributes` property that returns an **array-like live object whose members are all the attribute node objects of the element's tag**; real-time changes to the attributes are reflected in this node object. Other types of node objects also have an `attributes` property, but they all return `null`, so this property can be regarded as exclusive to element objects.

An individual attribute can be referenced by index or by attribute name.

```js
// <a id="test" href="http://www.example.com">link</a>
// document.getElementsByTagName('a')[0].attributes returns a NamedNodeMap array-like object

NamedNodeMap {0: id, 1: href, id: id, href: href, length: 2}

// both of these access methods return the attribute node object for id
// NamedNodeMap[0]
// NamedNodeMap.id

{
    baseURI: "file:///C:/Users/dell/Desktop/test.html"
    childNodes: NodeList []
    firstChild: null
    isConnected: false
    lastChild: null
    localName: "id"
    name: "id" // the attribute is 'id'
    namespaceURI: null
    nextSibling: null
    nodeName: "id"
    nodeType: 2
    nodeValue: "test"
    ownerDocument: document
    ownerElement: a#test
    parentElement: null
    parentNode: null
    prefix: null
    previousSibling: null
    specified: true
    textContent: "test"
    value: "test" // the value of id
}
```



```js
// HTML code is as follows
// <body bgcolor="yellow" onload="">
document.body.attributes[0]
document.body.attributes.bgcolor
document.body.attributes['ONLOAD']
```

Note that all three methods in the code above **return attribute node objects, not attribute values**.

**An attribute node object has `name` and `value` properties** corresponding to the attribute's name and value, which are equivalent to the `nodeName` property and the `nodeValue` property.

```js
// HTML code is
// <div id="mydiv">
var n = document.getElementById('mydiv');

n.attributes[0].name // "id"
n.attributes[0].nodeName // "id"

n.attributes[0].value // "mydiv"
n.attributes[0].nodeValue // "mydiv"
```

The code below can traverse all the attributes of an element node.

```js
var para = document.getElementsByTagName('p')[0];
var result = document.getElementById('result');

if (para.hasAttributes()) {
  var attrs = para.attributes;
  var output = '';
  for(var i = attrs.length - 1; i >= 0; i--) {
    output += attrs[i].name + '->' + attrs[i].value;
  }
  result.textContent = output;
} else {
  result.textContent = 'No attributes to show';
}
```



### 2. The standard attributes of an element

The standard attributes of an HTML element (i.e. **attributes defined in the standard**) **automatically become properties of the element node object**.

```js
// <a id="aId" href="http://www.example.com" class="aClass">link</a>

var a = document.getElementById('test');

// a is the node object; id and href automatically become properties of the a node
a.id // "aId"
a.href // "http://www.example.com/"
a.className // "aClass"
```

In the code above, the `id` and `href` attributes of the `a` element tag automatically become properties of the node object.

These properties are all writable.

```js
var img = document.getElementById('myImage');
img.src = 'http://www.example.com/image.jpg';
```

The code above immediately replaces the `src` property of the `img` object, i.e. it displays a different image.

This way of modifying attributes is often used to add form attributes.

```js
var f = document.forms[0];
f.action = 'submit.php';
f.method = 'POST';
```

The code above adds a submission URL and submission method to a form.

Note that **although this usage can read and write attributes, it cannot delete them**; the `delete` operator does not work here.

The attribute names of HTML elements are case-insensitive, but the property names of JavaScript objects are case-sensitive. The conversion rule is: when converting to a JavaScript property name, lowercase is always used. **If the attribute name consists of multiple words, camelCase is used**, i.e. starting from the second word, the first letter of each word is uppercase, such as `onClick`.

Some HTML attribute names are JavaScript reserved words, so they must be renamed when converted to JavaScript properties. The main two are as follows.

- The `for` attribute becomes `htmlFor`
- The `class` attribute becomes `className`

In addition, HTML attribute values are generally strings, but JavaScript properties automatically convert types. For example, the string `true` is converted to a boolean, the value of `onClick` is converted to a function, and the value of the `style` attribute is converted to a `CSSStyleDeclaration` object. Therefore, you can assign values of various types to these properties.



### 3. The standard methods for manipulating attributes

#### 3.1 Overview

**Element nodes provide six methods for manipulating attributes.**

- `getAttribute()`
- `getAttributeNames()`
- `setAttribute()`
- `hasAttribute()`
- `hasAttributes()`
- `removeAttribute()`

There are a few things to note here.

(1) Applicability

These six methods **apply to all attributes (including user-defined attributes)**.

(2) Return value

`getAttribute()` only returns strings; it does not return values of other types.

(3) Attribute names

These methods only accept the standard names of attributes; you do not need to rename reserved words—for example, `for` and `class` can be used directly. In addition, these methods are case-insensitive with respect to attribute names.

```js
var image = document.images[0];
image.setAttribute('class', 'myImage');
```

In the code above, the `setAttribute` method uses `class` directly as the attribute name, without writing it as `className`.



#### 3.2 Element.getAttribute() (returns the value of the element's specified attribute, as a string)

The `Element.getAttribute` method **returns the specified attribute of the current element node**. If the specified attribute does not exist, it returns `null`.

```js
// HTML code is
// <div id="div1" align="left">
var div = document.getElementById('div1');
div.getAttribute('align') // "left"
```



#### 3.3 Element.getAttributeNames() (returns an array whose members are the names of all the element's attributes)

`Element.getAttributeNames()` **returns an array whose members are the names of all the attributes of the current element**. If the current element has no attributes, it returns an empty array. The same result can also be obtained using the `Element.attributes` property; the only difference is that the latter returns an array-like object.

```js
var mydiv = document.getElementById('mydiv');

mydiv.getAttributeNames().forEach(function (key) {
  var value = mydiv.getAttribute(key);
  console.log(key, value);
})
```

The code above is used to traverse all the attributes of a node.



#### 3.4 Element.setAttribute() (set, i.e. add, an attribute on the element)

The `Element.setAttribute` method is used to **add an attribute to the current element node**. If an attribute with the same name already exists, it is equivalent to editing the existing attribute. This method has no return value.

```js
// HTML code is
// <button>Hello World</button>
var b = document.querySelector('button');
b.setAttribute('name', 'myButton');
b.setAttribute('disabled', true); // attribute values are always strings; other types are converted to strings, here it becomes the string 'true'
```

In the code above, the `name` attribute of the `button` element is set to `myButton`, and the `disabled` attribute is set to `true`.

There are two things to note here. First, attribute values are always strings, and values of other types are automatically converted to strings—for example, the boolean `true` becomes the string `true`. Second, the `disable` attribute in the example above is a boolean attribute; for a ` <button> ` element, this attribute does not need a value—as long as it is set, it always takes effect—so the `setAttribute` method can set the `disabled` attribute to any value.



#### 3.5 Element.hasAttribute() (whether it contains the specified attribute)

The `Element.hasAttribute` method **returns a boolean indicating whether the current element node contains the specified attribute**.

```js
var d = document.getElementById('div1');

if (d.hasAttribute('align')) {
  d.setAttribute('align', 'center');
}
```

The code above checks whether the `div` node has an `align` attribute. If it does, it sets it to center alignment.



#### 3.6 Element.hasAttributes() (whether it has any attributes)

The `Element.hasAttributes` method **returns a boolean indicating whether the current element has any attributes; if it has no attributes, it returns `false`, otherwise it returns `true`.**

```js
var foo = document.getElementById('foo');
foo.hasAttributes() // true
```



#### 3.7 Element.removeAttribute() (remove the specified attribute)

The `Element.removeAttribute` method **removes the specified attribute**. This method **has no return value**.

```js
// HTML code is
// <div id="div1" align="left" width="200px">
document.getElementById('div1').removeAttribute('align');
// the HTML code is now
// <div id="div1" width="200px">
```



### 4. The dataset property (get data-* custom attributes)

Sometimes you need to attach data to an HTML element for use by JavaScript scripts. One solution is to use **custom attributes**.

```html
<div id="mydiv" foo="bar">
```

The code above **defines a custom `foo` attribute** on the `div` element, and you can then read and write this attribute with `getAttribute()` and `setAttribute()`.

```js
var n = document.getElementById('mydiv');
n.getAttribute('foo') // bar
n.setAttribute('foo', 'baz')
```

**Although this approach achieves the goal, it makes the HTML element's attributes non-standard, causing the page code to fail validation.**

A better solution is to use **the standard-provided `data-*` attributes**.

```js
<div id="mydiv" data-foo="bar">
```

Then, use the element node object's `dataset` property, which points to an object that can be used to manipulate the HTML element tag's `data-*` attributes.

```js
var n = document.getElementById('mydiv');
n.dataset.foo // bar  omit data-, just access foo to get the value of data-foo
n.dataset.foo = 'baz'
```

In the code above, `data-foo` is read and written through `dataset.foo`.

**To delete a `data-*` attribute, you can use the `delete` command directly.**

```js
delete document.getElementById('myDiv').dataset.foo;
```

Besides the `dataset` property, you can also use methods such as `getAttribute('data-foo')`, `removeAttribute('data-foo')`, `setAttribute('data-foo')`, and `hasAttribute('data-foo')` to manipulate `data-*` attributes.

Note that there are restrictions on the attribute name after `data-`: it can only contain letters, digits, hyphens (`-`), dots (`.`), colons (`:`), and underscores (`_`). Moreover, the attribute name **should not use uppercase letters from `A` to `Z`**—for example, you cannot have an attribute name like `data-helloWorld`; it must be written as `data-hello-world`.

**After `data-`, you cannot use camelCase; you can use kebab-case.**

When converting to a `dataset` key, if a hyphen is followed by a lowercase letter, the hyphen is removed, that lowercase letter is converted to uppercase, and other characters are unchanged. Conversely, when a `dataset` key is converted to an attribute name, all uppercase letters are converted to a hyphen plus the lowercase form of that letter, and other characters are unchanged. For example, `dataset.helloWorld` is converted to `data-hello-world`.

**When accessing via dataset, any kebab-case attribute must be accessed using camelCase**



## 8. The Text node and the DocumentFragment node

### 1. The concept of the Text node

**A text node (`Text`) represents the text content of an element node (`Element`) and an attribute node (`Attribute`). If a node contains only a piece of text, then it has a text child node representing the node's text content.**

We usually obtain a text node using properties of the parent node such as `firstChild` and `nextSibling`, or create a text node using the `createTextNode` method of the `Document` node.

```js
// get a text node
var textNode = document.querySelector('p').firstChild;

// create a text node
var textNode = document.createTextNode('Hi');
document.querySelector('div').appendChild(textNode);
```

The browser natively provides a **`Text` constructor**. It returns a text node instance. Its argument is the text content of that text node.

```js
// empty string
var text1 = new Text(); // returns a text instance

// non-empty string
var text2 = new Text('This is a text node'); // the argument is the text content of the text node
```

Note that since a space is also a character, even a single space forms a text node. For example, ` ` contains a space, and its child node is a text node.

Besides **inheriting the `Node` interface, the text node also inherits the `CharacterData` interface**. For the properties and methods of the `Node` interface, please refer to the chapter "The Node Interface"; they are not repeated here. Most of the following properties and methods come from the `CharacterData` interface.



### 2. Properties of the Text node

#### 2.1 data (set or read the content of the text node)

The `data` property is equivalent to the `nodeValue` property and is used to **set or read the content of a text node**.

```js
// read the text content
document.querySelector('p').firstChild.data
// equivalent to
document.querySelector('p').firstChild.nodeValue

// set the text content
document.querySelector('p').firstChild.data = 'Hello World';
```



#### 2.2 wholeText (returns the current text node together with adjacent text nodes as a whole)

The `wholeText` property **returns the current text node together with adjacent text nodes as a whole**. In most cases, the return value of the `wholeText` property is the same as the `data` property and the `textContent` property. However, in certain special cases there are differences.

For example, suppose the HTML code is as follows.

```js
<p id="para">A <em>B</em> C</p>
```

At this point, the `wholeText` property and the `data` property of the text node return the same value.

```js
var el = document.getElementById('para');
el.firstChild.wholeText // "A "
el.firstChild.data // "A "
```

However, once the ` <em> ` node is removed, the `wholeText` property and the `data` property differ, because at this point the `  <p>  ` node actually contains two adjacent text nodes.

```js
el.removeChild(para.childNodes[1]);
el.firstChild.wholeText // "A C"
el.firstChild.data // "A "
```



#### 2.3 length (text length)

The `length` property **returns the text length of the current text node**.

```js
(new Text('Hello')).length // 5
```



#### 2.4 nextElementSibling (the next sibling element node), previousElementSibling (the previous sibling element node)

The `nextElementSibling` property **returns the sibling element node immediately following the current text node**. If no element node can be obtained, it returns `null`.

```js
// HTML is
// <div>Hello <em>World</em></div>
var tn = document.querySelector('div').firstChild;
tn.nextElementSibling
// <em>World</em>
```

The `previousElementSibling` property **returns the nearest sibling element node before the current text node**. If no element node can be obtained, it returns `null`.



### 3. Methods of the Text node

#### 3.1 appendData(), deleteData(), insertData(), replaceData(), substringData()

The following 5 methods are all **methods for editing the text content of a `Text` node**.

- `appendData()`: **append a string at the end** of the `Text` node.
- `deleteData()`: **delete a substring inside the `Text` node**; the first argument is the starting position of the substring, and the second argument is the length of the substring.
- `insertData()`: **insert a string into the `Text` node**; the first argument is the insertion position, and the second argument is the substring to insert.
- `replaceData()`: used to **replace text**; the first argument is the starting position of the replacement, the second argument is the length to be replaced, and the third argument is the new string to add.
- `substringData()`: used to **get a substring**; the first argument is the starting position of the substring in the `Text` node, and the second argument is the length of the substring.

```js
// HTML code is
// <p>Hello World</p>
var pElementText = document.querySelector('p').firstChild;

pElementText.appendData('!');
// the page displays Hello World!
pElementText.deleteData(7, 5);
// the page displays Hello W
pElementText.insertData(7, 'Hello ');
// the page displays Hello WHello
pElementText.replaceData(7, 5, 'World');
// the page displays Hello WWorld
pElementText.substringData(7, 10);
// the page display is unchanged; returns "World "
```



#### 3.2 remove() (remove the Text node)

The `remove` method is used to remove the current `Text` node.

```js
// HTML code is
// <p>Hello World</p>
document.querySelector('p').firstChild.remove()
// the HTML code is now
// <p></p>
```



#### 3.3 splitText() (split a Text node into two)

The `splitText` method **splits a `Text` node into two adjacent `Text` nodes. Its argument is the split position (zero-based)**, ending before the character at that position. If the split position does not exist, it throws an error.

After splitting, this method **returns the string after the split position**, while the original `Text` node becomes one containing only the string before the split position.

```js
// html code is <p id="p">foobar</p>
var p = document.getElementById('p');
var textnode = p.firstChild;

var newText = textnode.splitText(3);
newText // "bar"
textnode // "foo"
```

**The parent element node's `normalize` method can merge two adjacent `Text` nodes.**

Continuing the example above, the text node's `splitText` method splits one `Text` node into two, and the parent element's `normalize` method can perform the inverse operation, merging them.

```js
p.childNodes.length // 2

// merge the two adjacent Text nodes
p.normalize();
p.childNodes.length // 1
```



### 4. The DocumentFragment document fragment node

The `DocumentFragment` node represents a **fragment of a document and is itself a complete DOM tree structure. It has no parent node**, so `parentNode` returns `null`, but **it can have any number of child nodes inserted. It does not belong to the current document, and manipulating a `DocumentFragment` node is much faster than manipulating the DOM tree directly.**

It is generally used to build a DOM structure and then insert it into the current document. The `document.createDocumentFragment` method, as well as the browser's native `DocumentFragment` constructor, can create an empty `DocumentFragment` node. You then use other DOM methods to add child nodes to it.

```js
var docFrag = document.createDocumentFragment();
// equivalent to
var docFrag = new DocumentFragment();

var li = document.createElement('li');
li.textContent = 'Hello World';
docFrag.appendChild(li);

document.querySelector('ul').appendChild(docFrag);
```

The code above creates a `DocumentFragment` node, then adds an `li` node into it, and finally moves the `DocumentFragment` node into the original document.

**Note** that the `DocumentFragment` **node itself cannot be inserted into the current document**. When it is used as an argument to methods such as `appendChild()`, `insertBefore()`, and `replaceChild()`, it is **its child nodes that are inserted into the current document, not the node itself**. Once a `DocumentFragment` node is added into the current document, it becomes an empty node (its `textContent` property is an empty string) and can be used again. If you want to preserve the content of a `DocumentFragment` node, you can use the `cloneNode` method.

```js
document
  .querySelector('ul')
  .appendChild(docFrag.cloneNode(true));
```

Adding a `DocumentFragment` node into the current document this way does not empty the `DocumentFragment` node.

Below is an example that uses a `DocumentFragment` to reverse the order of all child nodes of a specified node.

```js
function reverse(n) {
  var f = document.createDocumentFragment();
  while(n.lastChild) f.appendChild(n.lastChild);
  n.appendChild(f);
}
```

The `DocumentFragment` node object **has no properties or methods of its own; they are all inherited from the `Node` node and the `ParentNode` interface**. That is, the `DocumentFragment` node has the following four properties beyond the `Node` node.

- `children`: returns a live `HTMLCollection` collection object that includes all the child element nodes of the current `DocumentFragment` object.
- `firstElementChild`: returns the first child element node of the current `DocumentFragment` object, or `null` if there is none.
- `lastElementChild`: returns the last child element node of the current `DocumentFragment` object, or `null` if there is none.
- `childElementCount`: returns the number of all child elements of the current `DocumentFragment` object.





## 8. CSS Manipulation

 CSS and JavaScript are two fields with clearly divided responsibilities: the former is responsible for the visual appearance of the page, and the latter is responsible for interacting with the user's behavior. However, since they both belong to the front end of web development, there are inevitably intersections and mutual cooperation between them. This chapter explains how to **manipulate CSS through JavaScript**.



### 1. The style attribute of HTML elements

The simplest way to manipulate CSS styles is to use the `getAttribute()`, `setAttribute()`, and `removeAttribute()` methods of a page element node to directly read, write, or delete the page element's `style` attribute.

```js
div.setAttribute(
  'style',
  'background-color:red;' + 'border:1px solid black;'
);
```

The code above is equivalent to the following HTML code.

```js
<div style="background-color:red; border:1px solid black;" />
```

**`style` can not only be read and written as a string—it is itself also an object** that implements the CSSStyleDeclaration interface (see the introduction below), and **you can read and write individual properties directly**.

```js
e.style.fontSize = '18px';
e.style.color = 'black';
```



### 2. The CSSStyleDeclaration interface

#### 2.1 Introduction - used to manipulate an element's style; readable and writable

The CSSStyleDeclaration interface is **used to manipulate an element's style**. This interface is implemented in three places.

- The `style` property of an element node (`Element.style`)
- The `style` property of a `CSSStyle` instance
- The return value of `window.getComputedStyle()`

The CSSStyleDeclaration interface **can read and write CSS style properties directly**, except that hyphens need to be changed to camelCase.

```js
var divStyle = document.querySelector('div').style;

// readable and writable
divStyle.backgroundColor = 'red';
divStyle.border = '1px solid black';
divStyle.width = '100px';
divStyle.height = '100px';
divStyle.fontSize = '10em';

// note, the styles read are only inline styles; you cannot read styles from a stylesheet
divStyle.backgroundColor // red
divStyle.border // 1px solid black
divStyle.height // 100px
divStyle.width // 100px
```

In the code above, the value of the `style` property is a CSSStyleDeclaration instance. **The properties this object contains correspond one-to-one with CSS rules**, but their names need to be rewritten—for example, `background-color` is written as `backgroundColor`. The rewriting rule is to remove the hyphen from the CSS property name and then capitalize the first letter after the hyphen. If the CSS property name is a JavaScript reserved word, the rule name must be prefixed with the string `css`—for example, **`float` is written as `cssFloat`**.

**Rewriting rule: kebab-case to camelCase, and prefix reserved words with `css`**

Note that the property values of this object are all strings; when setting them you must include the unit, but not the semicolon at the end of the rule. For example, `divStyle.width` cannot be written as `100`; it must be written as `100px`.

In addition, **`Element.style` returns only the inline styles, not all of the element's styles**. Styles set via a stylesheet, or styles inherited from a parent element, cannot be obtained through this property. **All of an element's styles must be obtained via `window.getComputedStyle()`.**



#### 2.2 CSSStyleDeclaration instance properties

##### (1) CSSStyleDeclaration.cssText (used to read and write all inline styles)

The `CSSStyleDeclaration.cssText` property is **used to read and write all the style declaration text of the current rule**.

```js
var divStyle = document.querySelector('div').style;

divStyle.cssText = 'background-color: red;' // overwrites the entire original inline style
  + 'border: 1px solid black;'
  + 'height: 100px;'
  + 'width: 100px;';
```

Note that the value of the `cssText` property does not require rewriting the CSS property names.

**The simplest way to delete all inline styles of an element is to set `cssText` to an empty string.**

```js
divStyle.cssText = '';
```



##### (2) CSSStyleDeclaration.length (the number of inline styles)

The `CSSStyleDeclaration.length` property **returns an integer indicating how many style declarations the current rule contains.**

```js
// HTML code is as follows
// <div id="myDiv"
//   style="height: 1px;width: 100%;background-color: #CA1;"
// ></div>
var myDiv = document.getElementById('myDiv');
var divStyle = myDiv.style;
divStyle.length // 3
```

In the code above, the inline style of the `myDiv` element contains 3 style rules in total.



##### (3) CSSStyleDeclaration.parentRule (returns the style block that the current rule belongs to)

The `CSSStyleDeclaration.parentRule` property **returns the style block that the current rule belongs to** (a CSSRule instance). If there is no style block it belongs to, this property returns `null`.

This property is read-only and is only meaningful when using the CSSRule interface.

```js
var declaration = document.styleSheets[0].rules[0].style;
declaration.parentRule === document.styleSheets[0].rules[0]
// true
```



#### 2.3 CSSStyleDeclaration instance methods

##### (1) CSSStyleDeclaration.getPropertyPriority() (whether the specified style has the important priority set)

The `CSSStyleDeclaration.getPropertyPriority` method **accepts a CSS style property name as its argument and returns a string indicating whether the `important` priority is set. If it is, it returns `important`; otherwise it returns an empty string.**

```js
// HTML code is
// <div id="myDiv" style="margin: 10px!important; color: red;"/>
var style = document.getElementById('myDiv').style;
style.margin // "10px"
style.getPropertyPriority('margin') // "important"
style.getPropertyPriority('color') // ""
```

In the code above, the `margin` property has `important` priority, while the `color` property does not.



##### (2) CSSStyleDeclaration.getPropertyValue() (returns the value of the specified style)

The `CSSStyleDeclaration.getPropertyValue` method **accepts a CSS style property name as its argument and returns a string representing the value of that property.**

```js
// HTML code is
// <div id="myDiv" style="margin: 10px!important; color: red;"/>
var style = document.getElementById('myDiv').style;
style.margin // "10px"
style.getPropertyValue("margin") // "10px"
```



##### (3) CSSStyleDeclaration.item() (returns the CSS property name at the specified position)

The `CSSStyleDeclaration.item` method **accepts an integer as its argument and returns the CSS property name at that position**.

```js
// HTML code is
// <div id="myDiv" style="color: red; background-color: white;"/>
var style = document.getElementById('myDiv').style;
style.item(0) // "color"
style.item(1) // "background-color"
```

In the code above, the CSS property name at position `0` is `color`, and the CSS property name at position `1` is `background-color`.

If no argument is provided, this method throws an error. If the argument value exceeds the actual number of properties, this method returns an empty string value.



##### (4) CSSStyleDeclaration.removeProperty() (removes the specified CSS property and returns its original value)

The `CSSStyleDeclaration.removeProperty` method **accepts a property name as its argument, removes that property from the CSS rule, and returns the property's original value.**

```js
// HTML code is
// <div id="myDiv" style="color: red; background-color: white;">
//   111
// </div>
var style = document.getElementById('myDiv').style;
style.removeProperty('color') // 'red'
// the HTML code becomes
// <div id="myDiv" style="background-color: white;">
```

In the code above, after the `color` property is removed, the font color changes from red to the default color.



##### (5) CSSStyleDeclaration.setProperty() (set a new CSS property)

The `CSSStyleDeclaration.setProperty` method is used to **set a new CSS property**. This method has no return value.

This method can accept three arguments.

- First argument: the property name; this argument is required.
- Second argument: the property value; this argument is optional. If omitted, the value defaults to an empty string.
- Third argument: the priority; this argument is optional. If set, the only valid value is `important`, representing `!important` in the CSS rule.

```js
// HTML code is
// <div id="myDiv" style="color: red; background-color: white;">
//   111
// </div>
var style = document.getElementById('myDiv').style;
style.setProperty('border', '1px solid blue');
```

After the code above runs, the `myDiv` element gets a blue border.



### 3. Detecting CSS modules (determining whether the browser supports a particular style)

The CSS specification evolves so quickly that new modules keep appearing. Different versions of different browsers vary in their support for CSS modules. Sometimes **you need to know whether the current browser supports a particular module, which is called "CSS module detection".**

**Determining whether the browser supports a particular style**

A fairly generally applicable method is to determine whether a particular property value of an element's `style` object is a string.

```js
typeof element.style.animationName === 'string';
typeof element.style.transform === 'string';
```

If the CSS property does exist, it returns a string. Even if the property is not actually set, it returns an empty string. If the property does not exist, it returns `undefined`.

```js
document.body.style['maxWidth'] // ""
document.body.style['maximumWidth'] // undefined
```

The code above shows that this browser supports the `max-width` property but not the `maximum-width` property.

Note that regardless of whether the CSS property name is written with or without a hyphen, the `style` property reflects whether the property exists.

```js
document.body.style['backgroundColor'] // ""
document.body.style['background-color'] // ""
```

In addition, when using this, you need to take into account the CSS prefixes of different browsers.

```js
var content = document.getElementById('content');
typeof content.style['webkitAnimation'] === 'string'
```

**This detection method can be written as a function.**

```js
function isPropertySupported(property) {
  if (property in document.body.style) return true;
  var prefixes = ['Moz', 'Webkit', 'O', 'ms', 'Khtml'];
  var prefProperty = property.charAt(0).toUpperCase() + property.substr(1);

  for(var i = 0; i < prefixes.length; i++){
    if((prefixes[i] + prefProperty) in document.body.style) return true;
  }

  return false;
}

isPropertySupported('background-clip')
// true
```



### 4. The native CSS object

**The browser natively provides a CSS object that offers some utility methods for JavaScript to manipulate CSS.**

This object currently has two static methods.

#### 4.1 CSS.escape() (escape special characters in a CSS selector)

The `CSS.escape` method is **used to escape special characters in a CSS selector.**

```js
<div id="foo#bar">
```

In the code above, the element's `id` attribute contains a `#` sign, a character that has special meaning in a CSS selector. You cannot write it directly as `document.querySelector('#foo#bar')`; you can only write it as `document.querySelector('#foo\\#bar')`. The reason a double backslash must be used here is that the single-quoted string itself escapes the backslash once.

The `CSS.escape` method is used to escape such special characters.

```js
document.querySelector('#' + CSS.escape('foo#bar'))
//CSS.escape('foo#bar') escapes to foo\#bar
```



#### 4.2 CSS.supports() (whether the current environment supports a particular CSS rule)

The `CSS.supports` method **returns a boolean indicating whether the current environment supports a particular CSS rule**.

There are two ways to write its arguments: one is where the first argument is the property name and the second argument is the property value; the other is where the entire argument is a complete line of CSS.

```js
// first way
CSS.supports('transform-origin', '5px') // true

// second way
CSS.supports('display: table-cell') // true
```

Note that in the second way, the argument must not end with a semicolon, otherwise the result is inaccurate.

```js
CSS.supports('display: table-cell;') // false
```



### 5. window.getComputedStyle() (returns the final styles of an element)

Inline styles have the highest priority, and changing an inline style is usually reflected immediately. However, a page element's final styles are computed by combining various rules. Therefore, if you want to get an element's actual styles, reading only the inline styles is not enough; you need to obtain the style rules the browser ultimately computes.

The `window.getComputedStyle` method is used to **return the final style rules computed by the browser**. It accepts an element node object as its argument and returns a CSSStyleDeclaration instance containing the final style information of the specified node. By "final style information" we mean the result after stacking the various CSS rules.

```js
var div = document.querySelector('div');
var styleObj = window.getComputedStyle(div);
styleObj.backgroundColor
```

In the code above, the background color obtained is the `div` element's true background color.

Note that the CSSStyleDeclaration instance is a **live object**: any modification to the styles is reflected on this instance in real time. In addition, this instance is **read-only**.

The `getComputedStyle` method can also accept a second argument representing the current element's pseudo-element (such as `:before`, `:after`, `:first-line`, `:first-letter`, etc.).

```js
var result = window.getComputedStyle(div, ':before');
```

The following example shows how to get an element's height.

```js
var elem = document.getElementById('elem-container');
var styleObj = window.getComputedStyle(elem, null)
var height = styleObj.height; // gets the element's final rendered height; most reliable
// equivalent to
var height = styleObj['height'];
var height = styleObj.getPropertyValue('height');
```

The `height` property obtained in the code above is the height the browser ultimately rendered, which is more reliable than heights obtained by other methods. Since `styleObj` is a CSSStyleDeclaration instance, you can use the various CSSStyleDeclaration instance properties and methods.

There are a few things to note.

- The CSS values returned by a CSSStyleDeclaration instance are all in absolute units. For example, **lengths are all in pixels** (the return value includes a `px` suffix), and **colors are in `rgb(#, #, #)` or `rgba(#, #, #, #)` format**.
- The **shorthand forms** of CSS rules **do not work**. For example, to read the value of the `margin` property, you cannot read it directly; you can only read properties such as `marginLeft` and `marginTop`. Likewise, the `font` property cannot be read directly; you can only read individual properties such as `font-size`.
- To read a raw CSS property name, use the bracket operator, such as `styleObj['z-index']`; to read a camelCase CSS property name, you can read it directly, such as `styleObj.zIndex`.
- The `cssText` property of the CSSStyleDeclaration instance returned by this method does not work; it returns `undefined`.



### 6. Getting the styles of CSS pseudo-elements

CSS pseudo-elements are elements added to the DOM via CSS, mainly generated through the `:before` and `:after` selectors, and then their content is specified with the `content` property.

Below is a snippet of HTML code.

```html
<div id="test">Test content</div>
```

The way to add the `:before` pseudo-element in CSS is as follows.

```css
#test:before {
  content: 'Before ';
  color: #FF0;
}
```

**A node element's `style` object cannot read or write the styles of pseudo-elements; in this case you need to use `window.getComputedStyle()`**. To get pseudo-elements in JavaScript, you can use the following method.

```js
var test = document.querySelector('#test');

var result = window.getComputedStyle(test, ':before').content;
var color = window.getComputedStyle(test, ':before').color;
```

In addition, **you can also use the `getPropertyValue` method of a CSSStyleDeclaration instance** to get the properties of pseudo-elements.

```js
var result = window.getComputedStyle(test, ':before')
  .getPropertyValue('content');
var color = window.getComputedStyle(test, ':before')
  .getPropertyValue('color');
```



### 7. The StyleSheet interface

#### 7.1 Overview

The `StyleSheet` interface represents a stylesheet of the web page, including stylesheets loaded by the ` <link> ` element and stylesheets embedded by the `  <style>  ` element.

**The `styleSheets` property of the `document` object** can return all the `StyleSheet` instances of the current page (i.e. **all the stylesheets**). It is an **array-like object**.

```js
var sheets = document.styleSheets;
var sheet = document.styleSheets[0];
sheet instanceof StyleSheet // true
```

For a stylesheet embedded by the `  <style>  ` element, there is **another way to obtain the `StyleSheet` instance**, namely the `sheet` property of that node element.

```js
// HTML code is <style id="myStyle"></style>
var myStyleSheet = document.getElementById('myStyle').sheet;// another way to obtain the styleSheet
myStyleSheet instanceof StyleSheet // true
```

Strictly speaking, the `StyleSheet` interface includes not only web page stylesheets but also XML document stylesheets. So **it has a subclass `CSSStyleSheet`** representing the CSS stylesheet of a web page. The stylesheet instance we obtain in a web page is actually an instance of `CSSStyleSheet`. This **subinterface inherits all the properties and methods of `StyleSheet`** and **defines several of its own properties**; the two interfaces are introduced together below.



#### 7.2 Instance properties

A `StyleSheet` instance has the following properties.

##### (1) StyleSheet.disabled (whether the stylesheet is disabled; readable and writable)

`StyleSheet.disabled` returns a boolean indicating whether the stylesheet is disabled. Manually setting the `disabled` property to `true` is equivalent to setting this stylesheet as an `alternate stylesheet` in the ` <link> ` element, i.e. the stylesheet will not take effect.

Note that the `disabled` property can only be set in JavaScript, not in HTML.

```html
<link rel="alternate stylesheet" href="style.css">
<!--setting alternate stylesheet means this stylesheet is disabled-->

<style>
    body{font-size: 16px;}
</style>

 <script>
    console.log(document.styleSheets)

     /*
     StyleSheetList {}
		0: CSSStyleSheet {ownerRule: null, type: "text/css", href: 		"file:///C:/Users/dell/Desktop/css.css", ownerNode: link, parentStyleSheet: null, …}
		1: CSSStyleSheet {ownerRule: null, cssRules: CSSRuleList, rules: CSSRuleList, type: "text/css", href: null, …}
		length: 2
     */

     console.log(document.styleSheets[0].disabled) // false not disabled? can't tell whether the stylesheet is set as alternate?
    document.styleSheets[0].disabled = true
    console.log(document.styleSheets[0].disabled) // true disabled
 </script>
```



##### (2) Stylesheet.href (returns the URL of the stylesheet)

`Stylesheet.href` **returns the URL of the stylesheet**. For an embedded stylesheet, this property returns `null`. This property is **read-only**.

```js
document.styleSheets[0].href
```



##### (3) StyleSheet.media (returns a media array-like object)

The `StyleSheet.media` property **returns an array-like object (a `MediaList` instance) whose members are strings representing the applicable media.** It indicates whether the current stylesheet is used for the screen (screen), for printing (print), for handheld devices (handheld), or for all media (all). This property is read-only, with a default value of `screen`.

```js
document.styleSheets[0].media.mediaText
// "all"
```

The `appendMedium` method of a `MediaList` instance is used to add a medium, and the `deleteMedium` method is used to remove a medium.

```js
document.styleSheets[0].media.appendMedium('handheld');
document.styleSheets[0].media.deleteMedium('print');
```



##### (4) StyleSheet.title (returns the stylesheet's `title` attribute)

The `StyleSheet.title` property returns the stylesheet's `title` attribute.



##### (5) StyleSheet.type (returns the stylesheet's `type` attribute)

The `StyleSheet.type` property returns the stylesheet's `type` attribute, which is usually `text/css`.

```js
document.styleSheets[0].type  // "text/css"
```



##### (6) StyleSheet.parentStyleSheet (returns the parent stylesheet; only has a value when the current stylesheet was loaded via @import, otherwise null)

The CSS `@import` command allows loading other stylesheets within a stylesheet. The `StyleSheet.parentStyleSheet` property **returns the stylesheet that contains the current stylesheet**. If the current stylesheet is a top-level stylesheet, this property returns `null`.

```js
if (stylesheet.parentStyleSheet) {
  sheet = stylesheet.parentStyleSheet;
} else {
  sheet = stylesheet;
}
```



##### (7) StyleSheet.ownerNode (returns the DOM node where the stylesheet is located)

The `StyleSheet.ownerNode` property **returns the DOM node where the `StyleSheet` object is located, usually ` <link> ` or `  <style>  `**. For stylesheets referenced by other stylesheets, this property is `null`.

```js
// HTML code is
// <link rel="StyleSheet" href="example.css" type="text/css" />
document.styleSheets[0].ownerNode // [object HTMLLinkElement]
```



##### (8) CSSStyleSheet.cssRules (returns an array-like object containing all CSS rules)

The `CSSStyleSheet.cssRules` property **points to an array-like object (a `CSSRuleList` instance) in which each member is a CSS rule of the current stylesheet.** Using the `cssText` property of a rule, you can get the string corresponding to that CSS rule.

```js
var sheet = document.querySelector('#styleElement').sheet;

sheet.cssRules[0].cssText
// "body { background-color: red; margin: 20px; }"

sheet.cssRules[1].cssText
// "p { line-height: 1.4em; color: blue; }"
```

Each CSS rule also has a `style` property pointing to an object used to **read and write specific CSS commands**.

```js
cssStyleSheet.cssRules[0].style.color = 'red';
cssStyleSheet.cssRules[1].style.color = 'purple';
```



##### (9) CSSStyleSheet.ownerRule (the rule's owner)

Some stylesheets are imported via the `@import` rule; their `ownerRule` property returns a `CSSRule` instance representing that `@import` rule line. If the current stylesheet was not imported via `@import`, the `ownerRule` property returns `null`.



#### 7.3 Instance methods

##### (1) CSSStyleSheet.insertRule() (insert a CSS rule)

The `CSSStyleSheet.insertRule` method is used to **insert a new CSS rule into the current stylesheet**.

```js
var sheet = document.querySelector('#styleElement').sheet;
sheet.insertRule('#block { color: white }', 0);
sheet.insertRule('p { color: red }', 1);
```

This method can accept two arguments. The first argument is a string representing the CSS rule; there can only be one rule here, otherwise it throws an error. The second argument is the position in the stylesheet at which to insert the rule (starting from 0); this argument is optional, with a default of 0 (i.e. by default it is inserted at the head of the stylesheet). Note that if the insertion position is greater than the number of existing rules, it throws an error.

The return value of this method is the position index of the newly inserted rule.

Note that browsers impose many [restrictions](https://drafts.csswg.org/cssom/#insert-a-css-rule) on scripts inserting rules into a stylesheet. So this method is best used inside a `try...catch`.



##### (2) CSSStyleSheet.deleteRule() (remove a CSS rule)

The `CSSStyleSheet.deleteRule` method is used to remove a rule from a stylesheet; its argument is the position of that rule in the `cssRules` object. This method has no return value.

```js
document.styleSheets[0].deleteRule(1);
```



### 8. Example: adding a stylesheet

There are two ways to add a stylesheet to a web page. One is to add an internal stylesheet, i.e. add a `  <style>  ` node to the document.

```js
// way one
var style = document.createElement('style');
style.setAttribute('media', 'screen');
style.innerHTML = 'body{color:red}';
document.head.appendChild(style);

// way two
var style = (function () {
  var style = document.createElement('style');
  document.head.appendChild(style);
  return style;
})();
style.sheet.insertRule('.foo{color:red;}', 0); // insert a rule into the stylesheet
```

The other way is to add an external stylesheet, i.e. add a ` <link> ` node to the document and then point its `href` attribute to the URL of the external stylesheet.

```js
var linkElm = document.createElement('link');
linkElm.setAttribute('rel', 'stylesheet');
linkElm.setAttribute('type', 'text/css');
linkElm.setAttribute('href', 'reset-min.css');

document.head.appendChild(linkElm);
```



### 9. The CSSRuleList interface

The CSSRuleList interface is **an array-like object representing a group of CSS rules, where all the members are CSSRule instances**.

A CSSRuleList instance is generally obtained through the `StyleSheet.cssRules` property.

```js
// HTML code is as follows
// <style id="myStyle">
//   h1 { color: red; }
//   p { color: blue; }
// </style>
var myStyleSheet = document.getElementById('myStyle').sheet;
var crl = myStyleSheet.cssRules;
crl instanceof CSSRuleList // true
```

In a CSSRuleList instance, each rule (a CSSRule instance) can be obtained through `rules.item(index)` or `rules[index]`. The number of CSS rules is obtained through `rules.length`. Continuing with the example above.

```js
crl[0] instanceof CSSRule // true
crl.length // 2
```

Note that adding and removing rules cannot be done on the CSSRuleList instance; they must be done on its parent StyleSheet instance via `StyleSheet.insertRule()` and `StyleSheet.deleteRule()`.



### 10. The CSSRule interface

#### 10.1 Overview

A CSS rule consists of two parts: the CSS selector and the style declaration. Below is a typical CSS rule.

```css
.myClass {
  color: red;
  background-color: yellow;
}
```

JavaScript **manipulates CSS rules through the CSSRule interface**. A CSSRule instance is generally obtained through the CSSRuleList interface (`StyleSheet.cssRules`).

```js
// HTML code is as follows
// <style id="myStyle">
//   .myClass {
//     color: red;
//     background-color: yellow;
//   }
// </style>
var myStyleSheet = document.getElementById('myStyle').sheet;
var ruleList = myStyleSheet.cssRules;
var rule = ruleList[0];
rule instanceof CSSRule // true
```



#### 10.2 Properties of a CSSRule instance

##### (1) CSSRule.cssText

The `CSSRule.cssText` property **returns the text of the current rule**, continuing with the example above.

```js
rule.cssText
// ".myClass { color: red; background-color: yellow; }"
```

If the rule loads (`@import`) another stylesheet, the `cssText` property returns `@import 'url'`.



##### (2) CSSRule.parentStyleSheet

The `CSSRule.parentStyleSheet` property **returns the stylesheet object (a StyleSheet instance) that the current rule belongs to**, continuing with the example above.

```js
rule.parentStyleSheet === myStyleSheet // true
```



##### (3) CSSRule.parentRule

The `CSSRule.parentRule` property **returns the parent rule that contains the current rule; if there is no parent rule (i.e. the current rule is a top-level rule), it returns `null`.**

The most common case of a parent rule is when the current rule is contained within an `@media` rule code block.

```js
// HTML code is as follows
// <style id="myStyle">
//   @supports (display: flex) {
//     @media screen and (min-width: 900px) {
//       article {
//         display: flex;
//       }
//     }
//  }
// </style>
var myStyleSheet = document.getElementById('myStyle').sheet;
var ruleList = myStyleSheet.cssRules;

var rule0 = ruleList[0];
rule0.cssText
// "@supports (display: flex) {
//    @media screen and (min-width: 900px) {
//      article { display: flex; }
//    }
// }"

// since this rule has other rules nested inside it,
// it has a cssRules property, and that property is a CSSRuleList instance
rule0.cssRules instanceof CSSRuleList // true

var rule1 = rule0.cssRules[0];
rule1.cssText
// "@media screen and (min-width: 900px) {
//   article { display: flex; }
// }"

var rule2 = rule1.cssRules[0];
rule2.cssText
// "article { display: flex; }"

rule1.parentRule === rule0 // true
rule2.parentRule === rule1 // true
```



##### (4) CSSRule.type

The `CSSRule.type` property **returns an integer indicating the type of the current rule**.

The most common types are as follows.

- 1: a regular style rule (a CSSStyleRule instance)
- 3: an `@import` rule
- 4: an `@media` rule (a CSSMediaRule instance)
- 5: an `@font-face` rule



#### 10.3 The CSSStyleRule interface

**If a CSS rule is a regular style rule (not containing special CSS commands), then in addition to the CSSRule interface, it also implements the CSSStyleRule interface.**

The CSSStyleRule interface has the following two properties.

##### (1) CSSStyleRule.selectorText

The `CSSStyleRule.selectorText` property **returns the selector of the current rule**.

```js
var stylesheet = document.styleSheets[0];
stylesheet.cssRules[0].selectorText // ".myClass"
```

Note that this property is **writable**.

##### (2) CSSStyleRule.style

The `CSSStyleRule.style` property **returns an object (a CSSStyleDeclaration instance) representing the style declaration of the current rule, i.e. the part inside the braces after the selector.**

```js
// HTML code is
// <style id="myStyle">
//   p { color: red; }
// </style>
var styleSheet = document.getElementById('myStyle').sheet;
styleSheet.cssRules[0].style instanceof CSSStyleDeclaration
// true
```

The `cssText` property of a CSSStyleDeclaration instance can return all the style declarations, formatted as a string.

```js
styleSheet.cssRules[0].style.cssText
// "color: red;"
styleSheet.cssRules[0].selectorText
// "p"
```



#### 10.4 The CSSMediaRule interface

**If a CSS rule is an `@media` code block, then in addition to the CSSRule interface, it also implements the CSSMediaRule interface.**

This interface mainly provides the `media` property and the `conditionText` property. The former returns an object (a MediaList instance) representing the `@media` rule, and the latter returns the condition under which the `@media` rule takes effect.

```js
// HTML code is as follows
// <style id="myStyle">
//   @media screen and (min-width: 900px) {
//     article { display: flex; }
//   }
// </style>
var styleSheet = document.getElementById('myStyle').sheet;
styleSheet.cssRules[0] instanceof CSSMediaRule
// true

styleSheet.cssRules[0].media
//  {
//    0: "screen and (min-width: 900px)",
//    appendMedium: function,
//    deleteMedium: function,
//    item: function,
//    length: 1,
//    mediaText: "screen and (min-width: 900px)"
// }

styleSheet.cssRules[0].conditionText
// "screen and (min-width: 900px)"
```



### 11. window.matchMedia()

#### 11.1 Basic usage

The `window.matchMedia` method is **used to convert a CSS [`MediaQuery`](https://developer.mozilla.org/en-US/docs/DOM/Using_media_queries_from_code) condition statement into a MediaQueryList instance**.

```js
var mdl = window.matchMedia('(min-width: 400px)');
mdl instanceof MediaQueryList // true
```

In the code above, the variable `mdl` is an instance of mediaQueryList.

Note that if the argument is not a valid `MediaQuery` condition statement, `window.matchMedia` does not throw an error and still returns a MediaQueryList instance.

```js
window.matchMedia('bad string') instanceof MediaQueryList // true
```



#### 11.2 Instance properties of the MediaQueryList interface

A MediaQueryList instance has three properties.

##### (1) MediaQueryList.media

The `MediaQueryList.media` property **returns a string representing the corresponding MediaQuery condition statement**.

```js
var mql = window.matchMedia('(min-width: 400px)');
mql.media // "(min-width: 400px)"
```



##### (2) MediaQueryList.matches

The `MediaQueryList.matches` property **returns a boolean indicating whether the current page matches the specified MediaQuery condition statement**.

```js
if (window.matchMedia('(min-width: 400px)').matches) {
  /* the current viewport is not smaller than 400 pixels */
} else {
  /* the current viewport is smaller than 400 pixels */
}
```

The following example loads the corresponding CSS stylesheet depending on whether the `mediaQuery` matches the current environment.

```js
var result = window.matchMedia("(max-width: 700px)");

if (result.matches){
  var linkElm = document.createElement('link');
  linkElm.setAttribute('rel', 'stylesheet');
  linkElm.setAttribute('type', 'text/css');
  linkElm.setAttribute('href', 'small.css');

  document.head.appendChild(linkElm);
}
```



##### (3) MediaQueryList.onchange

**If the matching environment of the MediaQuery condition statement changes, a `change` event is triggered**. The `MediaQueryList.onchange` property is used to specify the listener function for the `change` event. The argument to this function is the `change` event object (a MediaQueryListEvent instance), which is similar to the MediaQueryList instance and also has `media` and `matches` properties.

```js
var mql = window.matchMedia('(max-width: 600px)');

mql.onchange = function(e) {
  if (e.matches) {
    /* the viewport is no larger than 600 pixels */
  } else {
    /* the viewport is larger than 600 pixels */
  }
}
```

In the code above, after the `change` event occurs, there are two possibilities. One is that the display width changed from above 700 pixels to below, and the other is that it changed from below 700 pixels to above, so inside the listener function you need to determine which case it currently is.



#### 11.3 Instance methods of the MediaQueryList interface

A MediaQueryList instance has two methods, `MediaQueryList.addListener()` and `MediaQueryList.removeListener()`, **used to add or remove listener functions for the `change` event**.

```js
var mql = window.matchMedia('(max-width: 600px)');

// specify a listener function
mql.addListener(mqCallback);

// remove a listener function
mql.removeListener(mqCallback);

function mqCallback(e) {
  if (e.matches) {
    /* the viewport is no larger than 600 pixels */
  } else {
    /* the viewport is larger than 600 pixels */
  }
}
```

Note that the `MediaQueryList.removeListener()` method cannot remove the listener function specified by the `MediaQueryList.onchange` property.





### 12. Chapter summary

The instances (interfaces) in this chapter are:

**The CSSStyleDeclaration interface** (CSS style declaration): used to manipulate an element's style.

**The StyleSheet interface (stylesheet)**: represents a stylesheet of the web page, including stylesheets loaded by the ` <link> ` element and stylesheets embedded by the `  <style>  ` element.

**The CSSRuleList interface (CSS rule list)**: an array-like object representing a group of CSS rules, where all the members are CSSRule instances.

**The CSSRule interface (CSS rule)**: JavaScript manipulates CSS rules through the CSSRule interface. A CSSRule instance is generally obtained through the CSSRuleList interface (`StyleSheet.cssRules`).

**The CSSStyleRule interface**: if a CSS rule is a regular style rule (not containing special CSS commands), then in addition to the CSSRule interface, it also implements the CSSStyleRule interface.

**The CSSMediaRule interface**: if a CSS rule is an `@media` code block, then in addition to the CSSRule interface, it also implements the CSSMediaRule interface.





## 9. The Mutation Observer API (monitoring DOM changes)

### 1. Overview

The Mutation Observer API is **used to monitor DOM changes**. Any change in the DOM—such as adding or removing nodes, changing attributes, or changing text content—can be notified through this API.

Conceptually, it is very close to events; **you can think of it as: a Mutation Observer event is triggered whenever the DOM changes. However, it differs from events in one essential way: events are triggered synchronously, meaning a DOM change immediately triggers the corresponding event; a Mutation Observer, on the other hand, is triggered asynchronously—a DOM change does not trigger it immediately, but only after all current DOM operations have finished.**

This design is meant to cope with the fact that DOM changes are frequent. For example, if you insert 1000 `  <p>  ` elements consecutively into the document, it would trigger 1000 insertion events consecutively and execute the callback of each event, which is very likely to cause the browser to stutter. A Mutation Observer is completely different: it is only triggered after all 1000 paragraphs have been inserted, and it is triggered only once.

The Mutation Observer has the following characteristics.

- **It runs only after all script tasks have completed (i.e. it is triggered asynchronously).**
- **It packages DOM change records into an array for processing, rather than handling each DOM change one at a time.**
- **It can observe all types of DOM changes, or it can be specified to observe only a particular type of change.**



### 2. The MutationObserver constructor (how to use it)

To use it, **first use the `MutationObserver` constructor to create a new observer instance, specifying the callback function for this instance at the same time.**

```js
var observer = new MutationObserver(callback);
```

The callback function in the code above is called after each DOM change. This callback accepts two arguments: the first is the change array, and the second is the observer instance. Below is an example.

```js
var observer = new MutationObserver(function (mutations, observer) {
  mutations.forEach(function(mutation) {
    console.log(mutation);
  });
});
```



### 3. Instance methods of the MutationObserver

#### 3.1 observe()

The `observe` method is **used to start monitoring; it accepts two arguments.**

- First argument: the DOM node to observe
- Second argument: a configuration object specifying the particular changes to observe

```js
var article = document.querySelector('article');

var  options = { // the configuration object for the observer's second argument, specifying the particular changes to observe
  'childList': true, // child node changes
  'attributes':true, // attribute changes
  'characterData': true // changes to node content or node text
} ;

observer.observe(article, options);
```

In the code above, the `observe` method accepts two arguments: the first is the DOM element to observe, `article`, and the second is the types of changes to observe (child node changes and attribute changes).

The types of DOM changes the observer can observe (i.e. the `options` object in the code above) are as follows.

- **childList**: changes to child nodes (additions, removals, or modifications).
- **attributes**: changes to attributes.
- **characterData**: changes to node content or node text.

To observe a particular type of change, specify its value as `true` in the `option` object. Note that you must specify one or more of `childList`, `attributes`, and `characterData`; if none are specified, it throws an error.

Besides the change types, the `options` object can also set the following properties:

- `subtree`: a boolean indicating whether to apply the observer to all descendant nodes of the node.
- `attributeOldValue`: a boolean indicating whether to record the attribute value before the change when observing `attributes` changes.
- `characterDataOldValue`: a boolean indicating whether to record the value before the change when observing `characterData` changes.
- `attributeFilter`: an array indicating the specific attributes to observe (such as `['class','src']`).

```js
// start monitoring changes to the document's root node (i.e. the <html> tag)
mutationObserver.observe(document.documentElement, {
  attributes: true, // attribute changes
  characterData: true, // changes to node content or node text
  childList: true, // child node changes
  subtree: true, // whether to apply the observer to all descendant nodes of the node
  attributeOldValue: true, // whether to record the attribute value before the change when observing attributes changes
  characterDataOldValue: true // whether to record the value before the change when observing characterData changes
});
```

Adding an observer to a node is like using the `addEventListener` method: adding the same observer multiple times has no effect, and the callback is still triggered only once. If you specify a different `options` object, the later-added one takes precedence, similar to overwriting.

The following example observes newly added child nodes.

```js
var insertedNodes = [];
var observer = new MutationObserver(function(mutations) {
  mutations.forEach(function(mutation) {
    for (var i = 0; i < mutation.addedNodes.length; i++) {
      insertedNodes.push(mutation.addedNodes[i]);
    }
  });
  console.log(insertedNodes);
});
observer.observe(document, { childList: true, subtree: true });
```



#### 3.2 disconnect(), takeRecords()

The `disconnect` method is **used to stop observing. After this method is called, further DOM changes do not trigger the observer**.

```js
observer.disconnect();
```

The `takeRecords` method is **used to clear the change records, i.e. to stop processing unprocessed changes. This method returns an array of change records**.

```js
observer.takeRecords();
```

Below is an example.

```js
// save all changes not yet processed by the observer
var changes = mutationObserver.takeRecords();

// stop observing
mutationObserver.disconnect();
```



### 4. The MutationRecord object (change record)

**Each time the DOM changes, a change record (a MutationRecord instance) is generated**. This instance contains all the information related to the change. What the Mutation Observer processes is an array composed of individual `MutationRecord` instances.

The `MutationRecord` object contains the relevant DOM information and has the following properties:

- `type`: the type of change observed (`attributes`, `characterData`, or `childList`).
- `target`: the DOM node where the change occurred.
- `addedNodes`: the added DOM nodes.
- `removedNodes`: the removed DOM nodes.
- `previousSibling`: the previous sibling node, or `null` if there is none.
- `nextSibling`: the next sibling node, or `null` if there is none.
- `attributeName`: the attribute that changed. If `attributeFilter` is set, only the pre-specified attributes are returned.
- `oldValue`: the value before the change. This property is only valid for `attribute` and `characterData` changes; for a `childList` change, it returns `null`.



### 5. Application examples

#### 5.1 Changes to child elements

The following example shows how to read change records.

```js
var callback = function (records){
  console.log(records) // the records argument is a MutationRecord instance
  records.map(function(record){
    console.log('Mutation type: ' + record.type);
    console.log('Mutation target: ' + record.target);
  });
};
/*
    [MutationRecord]
    0: MutationRecord
    addedNodes: NodeList [text]
    attributeName: null
    attributeNamespace: null
    nextSibling: null
    oldValue: null
    previousSibling: script
    removedNodes: NodeList []
    target: body
    type: "childList"
    __proto__: MutationRecord
    length: 1
    __proto__: Array(0)
    Mutation type: childList
    test.html:23 Mutation target: [object HTMLBodyElement]
*/
var mo = new MutationObserver(callback);

var option = {
  'childList': true,
  'subtree': true
};

mo.observe(document.body, option);

document.body.append(document.createElement('div').textContent = '123')
```

The observer in the code above observes changes to all descendant nodes of ` <body> ` (`childList` means observing child nodes, and `subtree` means observing descendant nodes). The callback function displays the type and target node of all changes in the console.



#### 5.2 Changes to attributes

The following example shows how to track changes to attributes.

```js
var callback = function (records) {
  records.map(function (record) {
    console.log('Previous attribute value: ' + record.oldValue);
  });
};

var mo = new MutationObserver(callback);

var element = document.getElementById('#my_element');

var options = {
  'attributes': true,
  'attributeOldValue': true
}

mo.observe(element, options);
```

The code above first sets up tracking of attribute changes (`'attributes': true`), then sets up recording of the value before the change. When a change actually occurs, the value before the change is displayed in the console.



#### 5.3 Replacing the DOMContentLoaded event

When a page loads, the creation of DOM nodes produces change records, so as long as you observe DOM changes, you can trigger the relevant events at the earliest possible moment, eliminating the need to use the `DOMContentLoaded` event.

```js
var observer = new MutationObserver(callback);
observer.observe(document.documentElement, {
  childList: true,
  subtree: true
});
```

In the code above, it monitors changes to the child nodes of `document.documentElement` (i.e. the page's ` <html> ` HTML node), and the `subtree` property specifies that the monitoring also includes descendant nodes. Therefore, as soon as any page element is created, it can be immediately detected.



###### Wrapping a function that monitors DOM creation

The code below uses the `MutationObserver` object to **wrap a function that monitors DOM creation**.

```js
(function(win){
  'use strict';

  var listeners = [];
  var doc = win.document;
  var MutationObserver = win.MutationObserver || win.WebKitMutationObserver;
  var observer;

  function ready(selector, fn){
    // store the selector and callback function
    listeners.push({
      selector: selector,
      fn: fn
    });
    if(!observer){
      // monitor changes to document
      observer = new MutationObserver(check);
      observer.observe(doc.documentElement, {
        childList: true,
        subtree: true
      });
    }
    // check whether the node is already in the DOM
    check();
  }

  function check(){
  // check for matches against the stored nodes
    for(var i = 0; i < listeners.length; i++){
      var listener = listeners[i];
      // check whether the specified node has any matches
      var elements = doc.querySelectorAll(listener.selector);
      for(var j = 0; j < elements.length; j++){
        var element = elements[j];
        // ensure the callback is called only once for this element
        if(!element.ready){
          element.ready = true;
          // call the callback function on this node
          listener.fn.call(element, element);
        }
      }
    }
  }

  // expose ready externally
  win.ready = ready;

})(this);

// usage
ready('.foo', function(element){
  // ...
});
```
