# Object-Oriented Programming

# I. Instance Objects and the new Command

# 1. What is an Object

Object-Oriented Programming (OOP) is currently the mainstream programming paradigm. It abstracts the complex relationships of the real world into individual objects, and then simulates the real world through the division of labor and cooperation between objects.

Each object is a functional center with a clear division of responsibilities, capable of completing tasks such as receiving information, processing data, and sending information. Objects can be reused, and through inheritance mechanisms, they can be customized. Therefore, object-oriented programming has the characteristics of flexibility, code reusability, and high modularity, making it easy to maintain and develop. Compared with traditional procedural programming consisting of a series of functions or instructions, it is more suitable for large-scale software projects involving team collaboration.

So, what exactly is an “object”? Let's understand it from two perspectives.

(1) An object is an abstraction of a single entity.

A book, a car, or a person can all be objects; a database, a web page, or a connection to a remote server can also be objects. When entities are abstracted into objects, the relationships between entities become relationships between objects, thus allowing us to simulate real-world situations and program against objects.

(2) An object is a container that encapsulates properties and methods.

Properties represent the state of an object, and methods represent the behavior of an object (completing certain tasks). For example, we can abstract an animal into an animal object, using “properties” to record what kind of animal it is, and using “methods” to represent the animal's behaviors (running, hunting, resting, etc.).

var person = { // object
    name: 'Zhang San', // property
    interest: function(){ // method (behavior)
        return this.name + ' is drinking'
    }
}
person.interest() // Zhang San is drinking

// Real world: There is a person named Zhang San, who is drinking.
// Object simulation: A person is abstracted into an object person. The object is a container that encapsulates the property name as Zhang San, and his drinking is encapsulated as the method interest

# 2. Constructor Functions

The first step in object-oriented programming is to create objects. As mentioned earlier, an object is an abstraction of a single entity. A template is usually needed to represent the common characteristics of a class of entities, and then objects are generated based on this template.

Typical object-oriented programming languages (such as C++ and Java) have the concept of “classes.” A “class” is a template for objects, and an object is an instance of a “class.” However, JavaScript's object system is not based on “classes,” but rather based on constructor functions and prototype chains.

JavaScript uses constructor functions as templates for objects.

A “constructor function” is a function specifically designed to create instance objects.

It serves as the template for objects, describing the basic structure of instance objects.

A single constructor function can generate multiple instance objects, all sharing the same structure.

A constructor function is just an ordinary function, but with its own characteristics and usage.

var Vehicle = function () {
  this.price = 1000;
};
1
2
3

In the code above, Vehicle is a constructor function. To distinguish it from ordinary functions, the first letter of a constructor function's name is usually capitalized.

Constructor functions have two characteristics:

  • The this keyword is used inside the function body, representing the instance object to be created.
  • The new command must be used when creating objects.

Let's first introduce the new command.

# 3. The new Command

# Basic Usage

The purpose of the new command is to execute a constructor function and return an instance object.

var Vehicle = function () {
  this.price = 1000; // this points to the instance object v
};

var v = new Vehicle();  // new executes a constructor function and returns an instance object to v
v.price // 1000

To ensure that a constructor function must be used together with the new command, one solution is to use strict mode inside the constructor function, i.e., add use strict on the first line. # How the new Command Works

When using the new command, the function following it executes the steps below in order.

  1. Create an empty object to serve as the instance object that will be returned.

  2. Point the prototype of this empty object to the constructor function's prototype property.

  3. Assign this empty object to the this keyword inside the function.

  4. Begin executing the code inside the constructor function. (In the code, this points to the empty object (the instance object).)

  5. Return the instance object (or a custom object).

In other words, inside a constructor function, this refers to a newly generated empty object, and all operations on this happen on this empty object. The reason a constructor function is called a “constructor function” is that the purpose of the function is to operate on an empty object (i.e., the this object) and “construct” it into the desired shape.

If there is a return statement inside the constructor function followed by an object, the new command returns the object specified by the return statement; otherwise, it ignores the return statement and returns the this object.

var Vehicle = function () {
  this.price = 1000;
  return 1000; // 1000 is not an object, so it is ignored, and the this object is returned; if it were return {}, then {} would be returned

};

(new Vehicle()) === 1000
// false

The simplified internal flow of the new command can be represented by the code below.

// constructor function
function Person(name,age){
    this.name = name
    this.age = age
}

// custom _new
function _new() {
  // convert the arguments object into an array
  var args = [].slice.call(arguments);
  // take out the constructor function
  var constructor = args.shift();
  // create an empty object that inherits the constructor function's prototype property
  var context = Object.create(constructor.prototype);
  // execute the constructor function
  var result = constructor.apply(context, args);
  // if the returned result is an object, return it directly; otherwise return the context object
  return (typeof result === 'object' && result != null) ? result : context;
}

// custom _new2
function _new2(/* constructor function */ constructor, /* constructor arguments */ params) {
  // create an empty object that inherits the constructor function's prototype property
  var context = Object.create(constructor.prototype);
  // execute the constructor function
  var result = constructor.apply(context, params);
  // if the returned result is an object, return it directly; otherwise return the context object
  return (typeof result === 'object' && result != null) ? result : context;
  // (if the user customizes a returned object inside the constructor function, use that object; otherwise return context)
}


// return an instance via the custom _new
var actor = _new(Person, 'Zhang San', 28);
actor.name // Zhang San

// return an instance via the custom _new2
var actor2 = _new2(Person, ['Li Si', 29]);
actor2.name // Li Si

// return an instance via the new command
var actor3 = new Person('Wang Wu',30)
actor3.name // Wang Wu

# 4. Creating Instance Objects with Object.create()

A constructor function serves as a template that can generate instance objects. But sometimes you cannot get the constructor function, only an existing object. We want to use this existing object as a template to generate a new instance object, and in this case we can use the Object.create() method.

// this example has no constructor function, only an object

  age: 38,

var person2 = Object.create(person1);

# II. The this Keyword

# 1. Meaning

The this keyword is a very important syntactic point. It is no exaggeration to say that without understanding its meaning, most development tasks cannot be completed.

As mentioned in the previous chapter, this can be used inside a constructor function to represent the instance object. Besides that, this can also be used in other situations. But no matter what the situation is, this has one thing in common: it always returns an object.

Simply put, this is the object that a property or method “currently” belongs to.

this.property // this represents the object that the property currently belongs to

# 2. Essence

The reason the JavaScript language has the design of this is related to the data structures in memory.

var obj = { foo:  5 };
1

The code above assigns an object to the variable obj. The JavaScript engine first generates an object { foo: 5 } in memory, and then assigns the memory address of this object to the variable obj. In other words, the variable obj is an address (reference). Later, if you want to read obj.foo, the engine first gets the memory address from obj, then reads the original object from that address and returns its foo property.

The original object is stored as a dictionary structure, where each property name corresponds to a property descriptor object. For example, the foo property in the example above is actually stored in the following form.

{
  foo: {
    [[value]]: 5
    [[writable]]: true
    [[enumerable]]: true
    [[configurable]]: true
  }
}

# Summary: The Difference Between call(), apply(), and bind()

All three methods change the reference of this inside a function.

Their differences are:

call() — the first argument is the object that this inside the function will point to, and the remaining arguments are the arguments passed into the function.

apply() — the first argument is the object that this inside the function will point to, and the second argument is an array, where each item is an argument passed into the function.

bind() — takes the same arguments as call(); the difference is that bind() returns a new function and does not execute immediately.

# (3) The Prototype Chain

JavaScript stipulates that all objects have their own prototype object (prototype). On the one hand, any object can serve as the prototype of other objects; on the other hand, since the prototype object is also an object, it also has its own prototype. Therefore, a “prototype chain” (prototype chain) is formed: object to prototype, then to the prototype's prototype…

If you trace upward level by level, the prototype of all objects can ultimately be traced back to Object.prototype, i.e., the prototype property of the Object constructor function. In other words, all objects inherit the properties of Object.prototype. This is the reason all objects have the valueOf and toString methods, because these are inherited from Object.prototype.

So, does the Object.prototype object have its prototype? The answer is that the prototype of Object.prototype is null. null has no properties or methods, nor its own prototype. Therefore, the end of the prototype chain is null.

Object.getPrototypeOf(Object.prototype) // the Object.getPrototypeOf method returns the prototype of an object

The code above shows that the prototype of the Object.prototype object is null; since null has no properties, the prototype chain ends here. The Object.getPrototypeOf method returns the prototype of the argument object; see later for a detailed introduction.

When reading a certain property of an object, the JavaScript engine first looks for the object's own property; if not found, it goes to its prototype to look; if still not found, it goes to the prototype's prototype to look. If it still cannot be found all the way up to the topmost Object.prototype, it returns undefined. If both the object itself and its prototype define a property with the same name, then the object's own property is read first, which is called “overriding”.

Note that going up level by level to look for a certain property across the entire prototype chain has an impact on performance. The higher up the prototype object that the property being looked for is, the greater the impact on performance. If you look for a property that does not exist, the entire prototype chain will be traversed.

# 10. Copying Objects

To copy an object, you need to do the following two things.

  • Ensure that the copied object has the same prototype as the original object.
  • Ensure that the copied object has the same instance properties as the original object.

Below is an object-copying function implemented based on the two points above.

function copyObject(orig) { // object-copying function
    // create a new object whose prototype points to the prototype of the old object
  var copy = Object.create(Object.getPrototypeOf(orig));
  copyOwnPropertiesFrom(copy, orig);
  return copy;
}

function copyOwnPropertiesFrom(target, source) { // copy the instance properties of the old object
  Object
    .getOwnPropertyNames(source)
    .forEach(function (propKey) {
      // get the property descriptor object of each property
      var desc = Object.getOwnPropertyDescriptor(source, propKey);
      // define the property, defining the propKey property on the target object with the property descriptor object being desc
      Object.defineProperty(target, propKey, desc);
    });
  return target;
}

# V. Strict Mode

In addition to the normal running mode, JavaScript has a second running mode: strict mode. As the name implies, this mode adopts a more strict JavaScript syntax.

The same code may have different running results in normal mode and strict mode. Some statements that can run in normal mode will not be able to run in strict mode.

# 1. Design Purpose

The early JavaScript language had many unreasonable design aspects, but in order to maintain compatibility with previous code, the old syntax could not be changed, so new syntax could only be continually added to guide programmers to use the new syntax.

Strict mode entered the standard in ES5, mainly for the following purposes.

  • Explicitly prohibit some unreasonable and lax syntax, reducing some of the quirky behaviors of the JavaScript language.
  • Add more error-throwing situations, eliminating some unsafe aspects of code execution and ensuring the safety of code execution.
  • Improve compiler efficiency and increase running speed.
  • Lay the groundwork for the syntax of future new versions of JavaScript.

In short, strict mode reflects a more reasonable, safer, and more rigorous direction for the development of JavaScript.

# 2. How to Enable It

'use strict';
